# 12.6 `Vector` 第 3 版：动态属性存取（`__getattr__`）与不可变性（`__setattr__` / `__slots__`）

`Vector` 作为“数学对象”，除了索引 `v[0]` / `v[1]` 之外，用户通常也希望能写出更直觉的形式：

- `v.x`, `v.y`, `v.z`, `v.t`

这一节的核心是两件事：

- 用 `__getattr__` **把单字母属性映射到分量**
- 用 `__setattr__`（并结合 `__slots__`）**禁止赋值/禁止新增属性**，保证不可变序列的语义不被破坏

---

## 一、为什么用 `__getattr__`？

属性查找的大致流程是：

1. 先查实例属性（如果有 `__dict__` 或 `__slots__` 中的存储槽）
2. 再查类属性（含描述符、方法、property 等）
3. **都找不到才会调用 `__getattr__`**

这意味着 `__getattr__` 非常适合做“按需提供的动态属性”：只有当常规查找失败时才介入。

---

## 二、用 `__match_args__` 驱动属性映射

我们定义：

```python
__match_args__ = ('x', 'y', 'z', 't')
```

这样同一份“名称列表”可以复用在两个地方：

- **动态属性**：`v.x` → 分量 0，`v.y` → 分量 1 ...
- **模式匹配**（可选）：`match v: case Vector(x, y): ...`（后续章节会更系统讨论）

`__getattr__` 的核心逻辑是：

- `name` 在 `__match_args__` 里 → 找到位置 `pos`
- `pos` 在向量维度范围内 → 返回对应分量
- 否则抛出标准 `AttributeError`

---

## 三、为什么必须拦截赋值？（`v.x = 10` 的矛盾）

如果你只实现 `__getattr__`，但允许赋值：

- `v.x = 10` 会创建/写入实例属性 `x`
- 之后访问 `v.x` 会直接命中实例属性，不会再走 `__getattr__`
- 这会让 “`v.x` 显示为 10” 但内部分量完全没变，产生语义矛盾

因此对“不可变向量”，正确策略是：

- **禁止**对 `x/y/z/t` 赋值
- （可选强化）禁止对任意小写单字母赋值，避免用户误以为这些是合法分量

---

## 四、`__setattr__` + `__slots__`：更像内置不可变类型

### `__slots__`

```python
__slots__ = ('_components',)
```

效果：

- 实例不再有 `__dict__`（不能动态新增任意属性）
- 只允许 `_components` 这个存储槽存在

### `__setattr__`

我们做三类处理：

- 初始化阶段允许设置 `_components`（只允许一次）
- 禁止设置 `x/y/z/t`：报错 `readonly attribute 'x'`
- 禁止设置任意小写单字母：报错信息更明确（“a 到 z 都别设”）

配套可运行代码见 `vector_v3_dynamic_attrs_demo.py`。
