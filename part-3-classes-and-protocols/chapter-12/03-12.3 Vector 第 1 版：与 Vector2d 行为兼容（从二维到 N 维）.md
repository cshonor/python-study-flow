# 12.3 `Vector` 第 1 版：与 `Vector2d` 行为兼容（从二维到 N 维）

这一节完成 N 维 `Vector` 的“第 1 版实现”。重点不是一次性把序列协议做到极致，而是先把第 11 章 `Vector2d` 的核心语义**平移**到 N 维世界，并确保二维场景下行为对齐。

> 关键词：**组合（composition）**、**兼容性**、**通用性**、**可读性（repr 截断）**

---

## 一、核心设计目标

- **兼容性**：二维时，`Vector([x, y])` 在表现与核心协议上尽量贴近 `Vector2d(x, y)`（表示/bytes/相等性/模长/真值等）。
- **通用性**：支持任意维度 N，而不是固定 `x, y` 参数。
- **序列风格构造**：`__init__(components)` 接受一个可迭代对象，符合 `list/tuple` 等内置序列的构造方式。
- **repr 可读性**：高维向量 `repr` 自动截断，避免调试输出“爆屏”。

---

## 二、为什么不继承 `Vector2d`？

书中选择独立实现 `Vector`，而不是继承 `Vector2d`，主要是两点：

- **构造函数签名不兼容**：`Vector2d(x, y)` 是固定位置参数；而 N 维 `Vector(components)` 必须接受可迭代对象。
- **教学目标**：第 12 章主线是“序列协议的实现”，把 `Vector` 独立出来更容易把协议构建过程讲清楚，避免把继承细节混在一起。

---

## 三、实现要点（逐个协议对齐 `Vector2d`）

### 1）内部存储：`array('d')`

- 用 `array.array` 存储浮点分量，比 `list` 更紧凑，并且元素类型更明确。
- `typecode = 'd'` 与 `Vector2d` 保持一致（双精度浮点）。

### 2）`__iter__`：让 `Vector` 像“序列”一样可遍历

`__iter__` 直接返回内部数组的迭代器：

- 支持 `for` 循环
- 支持拆包（`x, y, *rest = v`）
- 让很多通用算法可直接复用（例如 `tuple(v)`、`sum(...)`、`zip(v1, v2)`）

### 3）`__repr__` / `__str__`：调试友好 + 用户友好

- `__repr__` 用 `reprlib.repr` 截断高维内容，例如 `Vector([0.0, 1.0, 2.0, 3.0, 4.0, ...])`
- `__str__` 通常更“面向用户”，这里输出 `tuple(self)` 的字符串形式

### 4）`__bytes__` / `frombytes`：二进制序列化对齐

与 `Vector2d` 同一路线：

- `bytes(v)` 的第 1 个字节写入 `typecode`
- 后续字节是内部 `array` 的原始 bytes
- `frombytes` 用 `memoryview(...).cast(typecode)` 零拷贝解析，再直接把 `memv` 作为可迭代对象传回构造函数，天然支持任意维度

### 5）`__eq__`：任意维度相等性

最直接的通用实现之一是把双方都转成 `tuple` 比较：

- 复用 `__iter__`
- 对 N 维直接成立
- 与 `Vector2d` 的“按分量比较”一致

### 6）`__abs__` / `__bool__`：模长与真值

- `abs(v)`：用 \(L_2\) 范数（欧几里得模长）
- `bool(v)`：遵循向量语义——非零向量为 `True`，零向量为 `False`

在 Python 3.8+ 中，`math.hypot` 支持 N 维参数，因此可以写成 `math.hypot(*self)`。

---

## 四、这版 `Vector` 与“生产”之间的边界

书里强调得很务实：如果你真的要做大量向量运算（NLP、检索、科学计算），应使用 **NumPy / SciPy**（以及相关生态）。

这里的 `Vector` 主要用于：

- 学习 Python 数据模型
- 学习“协议驱动的设计”（鸭子类型）
- 训练把一个自定义类型做得像标准库对象一样自然
