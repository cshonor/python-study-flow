# 16.3 一元运算符重载

## 一元运算符与特殊方法

| 语法 | 特殊方法 | 常见含义 |
|------|----------|----------|
| `-x` | `__neg__` | 取负（如向量各分量变号） |
| `+x` | `__pos__` | 一元正号（通常与拷贝/规范化有关） |
| `~x` | `__invert__` | 按位取反（整数）；自定义类型需语义自洽 |
| `abs(x)` | `__abs__` | 绝对值或模长（如二维向量长度） |

---

## 实现时的通用约定

1. **只接收 `self`**：一元特殊方法签名里只有实例自身（外加类型检查里可能用到的其他信息，但协议上就是 `self`）。

2. **优先“像内置类型一样”**：  
   - 对**不可变**值对象：`-v`、`abs(v)` 等应返回**新实例**（`__pos__` 若语义是“拷贝”，可返回新实例；若实例完全不可变且 `+v` 与恒等一致，也可返回 `self`，见下节例外）。  
   - 避免在 `__neg__` 等里**原地修改** `self`，除非类型明确设计为可变且文档说明如此（多数数值/向量 API 不这么做）。

3. **返回类型**：协变地保持“仍是该类型”，便于链式运算与类型检查（如 `Vector` 取负后仍是 `Vector`）。

## 何时 `x != +x`？（书中强调的两类例外）

### 1. `decimal.Decimal` 与上下文的精度

`+x` 会按**当前算术上下文**（`decimal.getcontext().prec` 等）参与构造/舍入。若上下文在 `x` 创建之后改变，可能出现 **`one_third == +one_third` 为假** 的情况（取决于实现细节与精度变化方式）。这打破“正号只是恒等”的直觉，却是**上下文敏感数值**的真实行为。

### 2. `collections.Counter`

对 `Counter` 实例，`+ct` 在语义上等价于 `ct + Counter()`：会**去掉非正计数**，得到只含正计数的新 `Counter`，因此一般 **`ct != +ct`**（当存在 0 或负计数时）。

## 实战关联（量化 / 自定义结构）

- **向量、坐标、信号**：`__neg__` / `__abs__` 表达方向与模长，公式直观。  
- **行情/策略对象**：仅在语义清晰时为一元运算绑定含义（例如“取反”表示反向头寸）；避免牵强附会。  
- **静态类型**：为这些方法写好返回注解，便于 `mypy` 等检查调用侧类型。
