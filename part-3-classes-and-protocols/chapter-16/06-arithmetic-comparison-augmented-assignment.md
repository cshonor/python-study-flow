# 16.7–16.9 收尾总览：算术、比较、增量赋值

## 16.7 中缀算术运算符与特殊方法

下表是**算术 / 按位**中缀运算符与**正向 / 反向 / 就地**方法的常见对应关系（详见 [数据模型](https://docs.python.org/3/reference/datamodel.html)）。

| 运算符 | 正向 | 反向 | 就地 |
|--------|------|------|------|
| `+` | `__add__` | `__radd__` | `__iadd__` |
| `-` | `__sub__` | `__rsub__` | `__isub__` |
| `*` | `__mul__` | `__rmul__` | `__imul__` |
| `/` | `__truediv__` | `__rtruediv__` | `__itruediv__` |
| `//` | `__floordiv__` | `__rfloordiv__` | `__ifloordiv__` |
| `%` | `__mod__` | `__rmod__` | `__imod__` |
| `divmod()` | `__divmod__` | `__rdivmod__` | — |
| `**` / `pow()` | `__pow__` | `__rpow__` | `__ipow__` |
| `@` | `__matmul__` | `__rmatmul__` | `__imatmul__` |
| `&` | `__and__` | `__rand__` | `__iand__` |
| `\|` | `__or__` | `__ror__` | `__ior__` |
| `^` | `__xor__` | `__rxor__` | `__ixor__` |
| `<<` | `__lshift__` | `__rlshift__` | `__ilshift__` |
| `>>` | `__rshift__` | `__rrshift__` | `__irshift__` |

说明：`divmod(a, b)` 通过 `__divmod__` / `__rdivmod__` 分派；语言层面**没有**与 `+=` 同类的 `divmod` 就地运算符，故上表就地列为 `—`。

### 分派要点

1. **`a op b`**：先试 `type(a)` 的正向方法；若未定义或返回 **`NotImplemented`**，再试 `type(b)` 的反向方法。
2. **交换律**（如 `+`、`*`、点积意义下的 `@`）：常写 **`__radd__ = __add__`** 等；**减、除**不可照搬，需单独实现 `__rsub__` / `__rtruediv__` 等。
3. **就地方法 `__i*__`**：用于 **`+=`、`*=`** 等；若未实现，解释器对**不可变风格**类型会退化为 **`a = a op b`**（见 16.9）。

## 16.8 比较运算符

### 与算术分派的差异

- **`==` / `!=`**：`a == b` 调用 `a.__eq__(b)`；若得到 `NotImplemented`，再试 `b.__eq__(a)`；若仍为 `NotImplemented`，Python 3 会**回退到 `id` 比较**（等价于 `a is b` 的布尔含义参与相等判定流程中的最后手段）。
- **`!=`**：若未定义 `__ne__`，Python 会**用 `__eq__` 的结果取反**；自定义 `__ne__` 时需与 `__eq__` 语义一致。
- **`<` `>` `<=` `>=`**：采用**反射**成对调用，例如 `a > b` 在 `a.__gt__(b)` 不可用时会尝试 **`b.__lt__(a)`** 等；若无法处理，最终 **`TypeError`**，**没有**像 `==` 那样的 `id` 回退。

### `Vector.__eq__` 的严谨写法

避免 `Vector([1, 2]) == (1, 2)` 因“逐项 zip 相等”而意外为真：应对 **`isinstance(other, Vector)`** 做类型判断，否则 **`return NotImplemented`**，把解释权交给对方类型或通用相等逻辑。

本仓库 `vector_add_operator_demo.py` 中的 `__eq__` 已按此方式实现。

### 比较方法速查

| 表达式 | 主要尝试顺序（简化） |
|--------|----------------------|
| `a == b` | `a.__eq__(b)` → `b.__eq__(a)` → 按规则比较 identity |
| `a != b` | `a.__ne__(b)` 或 `not (a == b)`（默认） |
| `a < b` | `a.__lt__(b)` → `b.__gt__(a)` → … |
| `a > b` | `a.__gt__(b)` → `b.__lt__(a)` → … |
| `a <= b` | `a.__le__(b)` → `b.__ge__(a)` → … |
| `a >= b` | `a.__ge__(b)` → `b.__le__(a)` → … |

## 16.9 增量赋值（`+=`、`*=`、`@=` …）

- **`a += b`** 对应 **`__iadd__`**（若存在且返回 `self` 等）；若**没有** `__iadd__`，则退化为 **`a = a + b`**，即调用 **`__add__`** 并把结果重新绑定到 `a`。
- **不可变语义的值对象**（如书中 `Vector`：运算总返回新实例）：通常**不实现** `__iadd__`，这样 **`v += x`** 会生成**新 `Vector`**，`id(v)` 改变；原引用仍指向旧对象。
- **可变对象**（如 `list`）：实现 `__iadd__` 时**原地修改**，一般 **`id` 不变**。
