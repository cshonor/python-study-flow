# 16.2 运算符重载入门：范围与限制

## 核心定义

运算符重载让**自定义对象**支持一元运算符（如 `-`、`+`、`~`）、中缀运算符（如 `+`、`*`、`|` 等），使表达式更贴近数学或领域习惯、可读性更好。

广义上，Python 里 `()`（调用）、`.`（属性）、`[]`（索引/切片）等也可视为“运算符”，但**本章主要讨论一元运算符与中缀运算符**及其特殊方法。

---

## Python 对运算符重载的三条硬限制

为避免滥用，语言层面对可扩展性做了约束：

1. **不能改写内置类型的运算符语义**  
   无法为 `int`、`list` 等内置类型替换或猴子补丁其 `__add__` 等行为（内置类型在 C 层实现，且不开放这种修改）。

2. **不能发明新符号，只能绑定现有运算符**  
   只能实现已有运算符对应的特殊方法，不能新增 `@%` 这类自定义中缀符号。

3. **部分语法无法重载**  
   `is`、`and`、`or`、`not` 不是通过对普通对象调用特殊方法实现的，**不能重载**。  
   按位运算 `&`、`|`、`~` 等则可通过 `__and__`、`__or__`、`__invert__` 等重载（对支持这些运算的类型而言）。

## 与后续小节的关系

- **16.3**：一元运算符 → `__neg__`、`__pos__`、`__invert__`、`__abs__`

掌握上述限制，有助于判断“这件事该不该用运算符表达”以及“能否用特殊方法实现”。
