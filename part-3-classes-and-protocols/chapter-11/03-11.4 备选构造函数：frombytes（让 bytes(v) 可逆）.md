# 11.4 备选构造函数：`frombytes`（让 `bytes(v)` 可逆）

在 11.3 我们已经让 `Vector2d` 支持：

```python
octets = bytes(v)
```

这一节的目标是实现反向操作：**从字节序列重建对象**。

在 Python 里，一个类只有一个 `__init__`，但我们经常需要“多种创建方式”（从极坐标、从字节流、从字符串……）。这类需求的标准解法是：

> **用 `@classmethod` 实现备选构造函数（alternative constructor）**。

配套实现已整合进 `02_vector2d_repr_demo.py`，并提供了独立断言 demo：`03_vector2d_frombytes_demo.py`。

---

## 一、核心实现（与 `__bytes__` 对称）

```python
@classmethod
def frombytes(cls, octets: bytes) -> Vector2d:
    typecode = chr(octets[0])
    memv = memoryview(octets[1:]).cast(typecode)
    return cls(*memv)
```

逐步拆解：

1. **读取 typecode**：`octets[0]` 是 `__bytes__` 写入的首字节（例如 `'d'`）。  
2. **零拷贝解析 payload**：`memoryview(octets[1:])` 直接建立内存视图，不复制数据。  
3. **按 typecode 解释二进制**：`.cast(typecode)` 把字节解释为对应的数值数组（例如 double）。  
4. **用 `cls` 构造实例**：`return cls(*memv)`，避免硬编码类名，支持子类继承。

---

## 二、为什么用 `classmethod`（不是 `staticmethod`）？

- `classmethod` 的第一个参数是 `cls`，能根据“实际调用的类”创建实例：
  - `Vector2d.frombytes(...)` 返回 `Vector2d`
  - 子类 `SubVector2d.frombytes(...)` 返回 `SubVector2d`
- `staticmethod` 没有 `cls`，需要硬编码类名或手动传入类，扩展性差。

---

## 三、使用示例（最重要的等式）

```python
v1 = Vector2d(3, 4)
v2 = Vector2d.frombytes(bytes(v1))
assert v1 == v2
```

只要你的 `__bytes__` 与 `frombytes` 使用同一套协议（typecode + payload），这个等式就成立。
