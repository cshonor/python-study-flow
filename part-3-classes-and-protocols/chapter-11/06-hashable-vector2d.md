# 11.7 可哈希的 `Vector2d`：让它能进 `set` / 当 `dict` 键

要让对象可哈希（hashable），需要同时满足三个条件：

1. 实现 `__eq__`
2. 实现 `__hash__`
3. **对象在生命周期内不可变**（参与比较/哈希的状态不能变）

配套脚本：`vector2d_hash_demo.py`。

---

## 一、为什么“可变对象”不能安全哈希

`set` / `dict` 依赖哈希值把对象放入桶中：

- 如果对象放进去之后哈希值变了  
  → 它会“留在旧桶里”，导致查找失败、集合行为异常。

因此：**哈希必须基于不可变状态**。

---

## 二、`Vector2d` 的做法：只读属性 + `__hash__` 与 `__eq__` 对齐

在本仓库的 `Vector2d` demo 里：

- `x` / `y` 是只读 `@property`
- 实例使用 `__slots__` 限制属性集合
- `__setattr__` 阻止初始化后修改坐标（“尽量不可变”）
- `__eq__` 用 `(x, y)` 值比较
- `__hash__` 用 `hash((x, y))` 计算（官方推荐思路：元组哈希）

核心一致性规则：

> 如果 `a == b` 为真，那么必须保证 `hash(a) == hash(b)`。

因此 `__hash__` 的输入应覆盖 `__eq__` 判等所使用的全部字段。

---

## 三、验证：能进 `set`，能当 `dict` 键

```python
v = Vector2d(3, 4)
s = {v}
d = {v: "ok"}
assert v in s
assert d[v] == "ok"
```

配套 demo 会同时验证“修改坐标会失败”，避免哈希漂移。
