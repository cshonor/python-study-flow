# 11.12 覆盖类属性：用 `typecode` 做“默认配置”，用实例/子类做扩展

本节用 `Vector2d.typecode` 说明一个非常实用的机制：

- **类属性**提供“所有实例共享的默认值”（默认配置）
- **实例属性**可以覆盖同名类属性（个性化配置）
- **子类覆盖类属性**是更 Pythonic 的扩展方式（不改父类，全局改变子类默认行为）

配套脚本：`09_vector2d_typecode_override_demo.py`（基于 `09_vector2d_v3.py`）。  

---

## 一、类属性是“默认配置”

`typecode` 定义在类上：

```python
class Vector2d:
    typecode = "d"
```

当实例读取 `self.typecode` 时：

1. 先在实例上找同名属性（实例字典/slots）
2. 找不到再去类上找（类属性）

因此：只要实例没有自己的 `typecode`，它看到的就是类属性的默认值。

---

## 二、实例覆盖类属性：只影响这个实例

```python
v = Vector2d(1.1, 2.2)
v.typecode = "f"
```

这句不会修改 `Vector2d.typecode`，而是给 `v` 增加一个**实例属性** `typecode`，从此：

- `v.typecode` 读到 `"f"`
- 其他实例仍读到类属性 `"d"`

对 `Vector2d` 来说，`typecode` 影响 `bytes(v)` 的序列化精度：

- `"d"`：1 字节 typecode + 2×8 字节浮点（典型 17 字节）
- `"f"`：1 字节 typecode + 2×4 字节浮点（典型 9 字节）

---

## 三、子类覆盖类属性：更推荐的“配置扩展”

与其在运行中全局改 `Vector2d.typecode`，更常见的做法是定义子类：

```python
class ShortVector2d(Vector2d):
    typecode = "f"
```

效果：

- `ShortVector2d` 继承全部行为，只改变默认 `typecode`
- 不影响 `Vector2d` 与其它子类

这是“对扩展开放、对修改关闭”的一个非常自然的落地方式。

---

## 四、一个最佳实践：`__repr__` 不硬编码类名

如果 `__repr__` 用 `type(self).__name__`，子类会自动得到正确的类名显示：

- `Vector2d(...)`
- `ShortVector2d(...)`

这让“只覆盖类属性的子类”非常轻量：不用重写 `__repr__` 等方法。
