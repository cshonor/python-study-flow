# 11.11 使用 `__slots__` 节省空间：为什么省、怎么用、怎么继承

`__slots__` 的目标非常明确：**当你要创建大量实例**时，用更紧凑的结构保存实例属性，从而减少每个实例的额外开销。

配套脚本：

- `slots_inheritance_demo.py`：继承规则与 `__dict__`/动态属性演示
- `slots_memory_demo.py`：用 `tracemalloc` 做小规模内存对比（可在普通机器上跑）

---

## 一、为什么默认的 `__dict__` 很占空间

默认情况下，每个实例通常都有一个 `__dict__` 来存放实例属性，这带来两类开销：

- **结构开销**：`dict` 本身为了支持快速查找/动态扩容，有较高的内存成本；
- **动态性开销**：允许任意新增属性，必须维持哈希表结构。

当实例数量上到十万/百万级，这部分开销会成为主要内存消耗。

---

## 二、`__slots__` 的做法：把“允许的属性集合”固定下来

在类里声明：

```python
class Pixel:
    __slots__ = ("x", "y")
```

效果（常见表现）：

- 实例通常**没有 `__dict__`**（`p.__dict__` 会 `AttributeError`）
- 只能设置 slots 中声明的属性名；拼写错/新增属性会报错

这就是省内存的核心：**不为每个实例再分配一个 dict**。

---

## 三、继承规则（最容易踩坑）

关键事实：`__slots__` **不会自动“阻止子类拥有 `__dict__`**。

- 子类**不声明 `__slots__`**：通常会重新获得 `__dict__`（能动态加属性）
- 子类**声明 `__slots__`**：可以继续保持无 `__dict__`，并可新增 slots

示例见 `slots_inheritance_demo.py`。

---

## 四、弱引用：需要 `__weakref__`（否则不能被 weakref）

如果一个类定义了 `__slots__`，但 slots 中不包含 `__weakref__`：

- 实例可能不能作为弱引用目标（`weakref.ref(obj)` 报 `TypeError`）

如果你需要弱引用支持，显式加入：

```python
__slots__ = ("x", "y", "__weakref__")
```

---

## 五、与 `cached_property` 的取舍

`cached_property` 通常把缓存值写入实例的 `__dict__`。

- 如果你的类完全没有 `__dict__`（纯 slots），`cached_property` 往往不能工作（或需要额外设计）。
- 一种折中是把 `__dict__` 加入 `__slots__`，但这会抵消大部分“省内存”的意义：

```python
__slots__ = ("x", "y", "__dict__")
```

结论：`__slots__` 是有代价的优化，适合**大规模实例 + 属性集合稳定**的类。

---

## 六、把 `__slots__` 用在 `Vector2d`

对于 `Vector2d` 这种“属性固定、实例可能很多”的数据对象，`__slots__` 常常很合适。

本仓库的 `vector2d_repr_demo.py` 版本已经采用 slots，并且把坐标存放在私有属性里（便于保持不可变性/可哈希性）。
