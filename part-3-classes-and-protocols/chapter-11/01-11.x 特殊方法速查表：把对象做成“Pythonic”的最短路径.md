# 11.x 特殊方法速查表：把对象做成“Pythonic”的最短路径

判断一个对象是否 Pythonic，往往不是看它有没有一堆方法，而是看它是否正确实现了 Python 的“协议”（protocol）——也就是一组约定的特殊方法，让内置语法与标准库能自然地工作。

本页按**能力/协议**整理常用特殊方法，并给出 `Vector2d` 的实现清单（用于本章主线案例）。

## 一、表示与调试（representation）

### 1.1 速查表（先记一行）

| 目标 | 推荐实现 | 说明 |
|---|---|---|
| 交互式/调试输出 | `__repr__` | 无歧义、**尽量**可重建；常写成 `ClassName(x=..., y=...)` |
| 面向用户的字符串 | `__str__` | `str(obj)`、`print(obj)`；未实现则回退 **`__repr__`** |
| 字节序列 | `__bytes__` | `bytes(obj)`；自定义二进制协议 / 与 **`frombytes`** 配对 |

---

### 1.2 `__repr__`：开发者 / 调试用「官方表示」

- **目标**：给 **REPL、日志、断点** 看的东西；要**无歧义**，并**尽量**满足 **`eval(repr(obj)) == obj`**（做不到时也要让人一眼看懂状态）。  
- **常见触发**：**`repr(obj)`**、容器里元素的展示、交互式里直接写 **`obj`**。  
- **习惯写法**：**`类名(...)`** + 用 **`!r`** 打印各字段，避免 **`str`** 混入歧义。

```python
class Point:
    def __init__(self, x: float, y: float) -> None:
        self.x = x
        self.y = y

    def __repr__(self) -> str:
        return f"Point(x={self.x!r}, y={self.y!r})"


p = Point(1, 2)
print(repr(p))  # Point(x=1, y=2)
```

---

### 1.3 `__str__`：用户友好字符串

- **目标**：给人读的**短描述**；**不必**能 **`eval`**。  
- **常见触发**：**`print(obj)`**、**`str(obj)`**；未定义 **`__str__`** 时用 **`__repr__`**。  
- **`f"{obj}"`**：走 **`__format__`**；未自定义时，默认实现常会落到与 **`str(obj)`** 类似的展示（细节见 **`02-11.x … __format__`**）。

```python
class Point:
    def __init__(self, x: float, y: float) -> None:
        self.x = x
        self.y = y

    def __str__(self) -> str:
        return f"坐标({self.x}, {self.y})"


p = Point(1, 2)
print(p)  # 坐标(1, 2)
```

---

### 1.4 `__bytes__`：字节序列

- **目标**：定义 **`bytes(obj)`** 的二进制外形，便于**自定义协议**、**网络**、与 **`frombytes`** 对称。  
- **习惯**：**明确 `encoding`**（如 **`utf-8`**），返回 **`bytes`** 实例。

```python
class Point:
    def __init__(self, x: float, y: float) -> None:
        self.x = x
        self.y = y

    def __bytes__(self) -> bytes:
        return f"{self.x},{self.y}".encode("utf-8")


p = Point(1, 2)
print(bytes(p))  # b'1,2'
```

---

### 1.5 三者对比

| 方法 | 用途 | 受众 | 典型形态 |
|---|---|---|---|
| **`__repr__`** | 调试 / 重建 | 开发者 | **`Point(x=1, y=2)`** |
| **`__str__`** | 展示 / 阅读 | 终端用户 | **`坐标(1, 2)`** |
| **`__bytes__`** | 二进制传输 | 程序 / 协议 | **`b'1,2'`** |

---

### 1.6 推荐模板（三个方法写在一个类里）

```python
class MyClass:
    def __init__(self, a: object, b: object) -> None:
        self.a = a
        self.b = b

    def __repr__(self) -> str:
        return f"MyClass(a={self.a!r}, b={self.b!r})"

    def __str__(self) -> str:
        return f"MyClass 实例：a={self.a}, b={self.b}"

    def __bytes__(self) -> bytes:
        return f"{self.a},{self.b}".encode("utf-8")
```

---

### 1.7 常见避坑

1. **`__repr__` 与循环引用**：互相引用的对象在 **`repr`** 里可能无限递归；可用 **`...`**、**`id(...)`**、或 **`reprlib.recursive_repr`** 等策略截断。  
2. **`__bytes__` 与编码**：**`str.encode`** 在非法字符上会失败；协议里应约定**允许的字符集**或先做**转义**。  
3. **容器显示的是元素的 `__repr__`**：例如 **`print([p])`** 里 **`p`** 走 **`__repr__`**，不是 **`__str__`**。

`Vector2d`（本章主线）常见：**`__repr__`**、**`__str__`**、**`__bytes__`**；完整实现见 **`02-11.x …`** 与 **`09_vector2d_v3.py`**。

## 二、类型转换与数值接口（coercion / numeric）

| 目标 | 特殊方法 | 示例触发 |
|---|---|---|
| 转 `complex` | `__complex__` | `complex(obj)` |
| 转 `float` | `__float__` | `float(obj)` |
| 转 `int` | `__int__` | `int(obj)` |
| 绝对值/模长 | `__abs__` | `abs(obj)` |
| 真值测试 | `__bool__` | `if obj:` / `bool(obj)`（不实现则用 `__len__`） |

`Vector2d` 常见：`__abs__`、`__bool__`、（可选）`__complex__`。

---

## 三、可迭代与序列协议（iteration / sequence）

| 能力 | 特殊方法 | 说明 |
|---|---|---|
| 可迭代 | `__iter__` | 让 `for x in obj`、`tuple(obj)` 工作 |
| 取长度 | `__len__` | 让 `len(obj)` 工作（也影响 `bool(obj)` 的默认行为） |
| 下标访问 | `__getitem__` | 序列/映射协议关键入口 |
| 成员测试 | `__contains__` | `x in obj`（不实现会退化为迭代搜索） |

`Vector2d` 通常实现 `__iter__`（例如 yield x, y），不一定做完整序列。

---

## 四、比较与哈希（equality / hashing）

| 目标 | 特殊方法 | 注意点 |
|---|---|---|
| 相等性 | `__eq__` | 定义语义：坐标相等？容忍浮点误差？（本书通常用严格相等） |
| 可哈希 | `__hash__` | **只对不可变对象**实现；若定义了 `__eq__`，想可哈希需确保哈希与相等性一致 |

经验法则：

- 若对象**可变**，不要实现 `__hash__`（否则放进 set/dict 会出大问题）。
- 若对象不可变（例如 `frozen=True` 或只读属性），可实现 `__hash__`。

`Vector2d` 常见：只读属性 + `__hash__`（或 `@dataclass(frozen=True)` 的等价语义，前提是哈希语义匹配需求）。

---

## 五、格式化（formatting）

| 目标 | 特殊方法 | 说明 |
|---|---|---|
| `format(obj, spec)` | `__format__` | 支持 `format()` / f-string 格式规约（例如极坐标/直角坐标展示） |

`Vector2d` 常见：支持自定义 `format_spec`（比如 `".2f"` 或自定义前缀）。  

---

## 六、属性访问与只读约束（attribute access）

| 目标 | 手段 | 说明 |
|---|---|---|
| 只读属性 | `@property` | 外部读，内部控写（或完全不提供 setter） |
| 计算属性 | `@property` | 例如 `angle`、`magnitude` 等 |

本章 `Vector2d` 常见：`x` / `y` 只读，`angle` 计算属性。

---

## 七、内存优化（slots）

| 目标 | 机制 | 代价/限制 |
|---|---|---|
| 节省实例内存 | `__slots__` | 失去动态添加属性；与多继承/弱引用等特性有交互成本 |

建议：先写对语义与协议，再考虑 `__slots__`，并用基准/内存工具验证收益。

---

## 八、`Vector2d` 实现清单（本章主线）

最常见的一组（按学习顺序）：

1. **数据与只读接口**：`x`、`y`（property 或不可变数据结构）
2. **可迭代**：`__iter__`（支持 `tuple(v)` / 解包）
3. **表示**：`__repr__`、`__str__`
4. **数值语义**：`__abs__`（模长）、`__bool__`
5. **转换/序列化**：`__bytes__`（可选搭配类方法 `frombytes`）
6. **比较/哈希**：`__eq__`、`__hash__`（确保不可变 + 语义一致）
7. **格式化**：`__format__`（可选：极坐标/直角坐标双格式）
8. **内存优化**：`__slots__`（最后再做）

---

## 九、两条实践原则（避免“为了 dunder 而 dunder”）

1. **先定义语义，再选协议**：例如 `__eq__` 到底是严格坐标相等还是容忍误差？一旦决定就要让 `__hash__` 与之匹配。
2. **只实现你需要的那部分协议**：实现 `__iter__` 并不等于你必须实现完整序列接口；Pythonic 不等于“全都实现一遍”。
