# 可迭代对象 vs 迭代器：协议、底层驱动与陷阱（17.4）

## 新手大白话：可迭代对象 vs 迭代器

### 1）两个东西到底是什么？

**可迭代对象（iterable）** —— **能用来「造出」迭代器的东西。**  
例如：`list`、`str`、`tuple`，以及你写的 **`Sentence`**（见 **17.2 / 17.5**）。

- **常见关键面**：有 **`__iter__`**（或走 **[17.3](<./03-iter() 与 Python 迭代的底层规则（17.3）.md>)** 里那种 **`__getitem__` 回退**）。

```python
my_list = [1, 2, 3]  # 可迭代对象：每次 iter(my_list) 都能拿一个新迭代器
```

**迭代器（iterator）** —— **真正负责「一个一个往外吐」的那个对象。**

- **常见关键面**：有 **`__next__`**；并且 **`__iter__` 返回自己**（见下条硬规则）。

```python
it = iter(my_list)  # it 是迭代器：负责 next，直到耗尽
```

---

### 2）最核心硬规则（背这一条）

**迭代器一定满足：**

```python
iter(it) is it  # 对「真·迭代器」应为 True
```

意思是：**迭代器自己也算「可迭代」**，但 `for` 再向它要迭代器时，**不会给你换一个新的**，还是它自己 —— 这和「列表每次 `iter` 都是新迭代器」正相反。

---

### 3）`for` 循环底层在干嘛？

你写：

```python
for x in obj:
    ...
```

Python 实际做的是：

```python
it = iter(obj)
while True:
    try:
        x = next(it)
    except StopIteration:
        break
    ...
```

一句话：**`for` 只认「能不能从 `obj` 弄到迭代器」**；弄到手之后，就只跟 **`next`** / **`StopIteration`** 打交道。

---

### 4）最大陷阱：迭代器是「一次性筷子」

```python
nums = [1, 2, 3]
it = iter(nums)

print(list(it))  # [1, 2, 3]
print(list(it))  # []  ← 同一个 it 已经耗尽，不能“自动 rewind”
```

**可迭代对象**（如上面的 **`nums`**）为什么还能反复玩？

- 因为每次 **`iter(nums)`** 都会**新建**一个迭代器；旧 iterator 空了不影响你再要一个新的。

---

### 5）超好用判断：`iter(x) is x` 吗？

```python
x = ...  # 你手里的对象

if iter(x) is x:
    print("多半是迭代器：一次性，用完就空")
else:
    print("多半是可迭代对象：iter(x) 会给你新的迭代器")
```

（极端自定义类若乱写 `__iter__`，也可能骗人；**工程里以协议是否健全为准**，学习阶段这条够用了。）

---

### 6）三句背会就通透

1. **可迭代对象**：负责 **`iter(...)` 造迭代器** → 同一份数据常能**反复遍历**。  
2. **迭代器**：负责 **`next`** 推进状态 → **一次性**，耗尽就停。  
3. **真迭代器**：**`iter(it) is it`**。

---

### 7）十行以内：一眼看懂「谁是一次性」

```python
nums = [1, 2, 3]
it = iter(nums)

print("list is iterator?     ", iter(nums) is nums)   # False
print("iterator is itself?   ", iter(it) is it)        # True

print("first pass: ", list(it))   # [1, 2, 3]
print("second pass:", list(it))   # []

print("fresh from list:", list(iter(nums)))  # 每次 iter(nums) 都是新迭代器 → 又有 [1,2,3]
```

## 两个概念先对齐

| 概念 | 一句话 | 关键方法 |
|------|--------|----------|
| **可迭代对象（iterable）** | 能被 `iter(x)` 取出迭代器的对象 | `__iter__`（或兼容的 `__getitem__`） |
| **迭代器（iterator）** | 能被 `next(it)` 反复取值直到耗尽的对象 | `__next__` + `__iter__` |

迭代器的一个硬规则：

- **`it.__iter__()` 必须返回 `it` 自身**（保证“迭代器也是可迭代的”）

---

## `for` 循环的底层等价形式

```python
it = iter(obj)
while True:
    try:
        x = next(it)
    except StopIteration:
        break
    ...
```

要点：

- `for` 不关心 `obj` 是什么类型，只要 `iter(obj)` 能拿到迭代器即可（鸭子类型）
- 结束信号不是返回 `None`，而是抛 `StopIteration`

---

## 迭代器的核心特性（最容易踩坑的点）

- **一次性**：迭代器走完就耗尽，不能“重置”
- **可重复遍历的是 iterable**：容器/序列每次 `iter(x)` 会给你一个新的迭代器
- 判断一个对象是否是迭代器的典型方式：
  - `iter(x) is x` 往往为真（对迭代器）

---

## `Sentence`：显式迭代器版本（对比 17.2）

17.2 用的是“仅 `__getitem__` 的回退迭代”。在 17.4 里我们补一个更标准的写法：

- `Sentence.__iter__` 返回一个 **独立的迭代器对象**
- 迭代器对象实现 `__next__`（无元素时抛 `StopIteration`）与 `__iter__`
