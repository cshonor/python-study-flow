# 17.13 经典协程与类型提示

**经典协程（generator-based coroutine）**：用**普通生成器函数** + **`.send()`** / **`yield` 表达式** 做「让出—恢复」的控制流；在 **`async` / `await`** 成为主流之前，这是 Python 里实现「可暂停计算」的重要形态。

本节只做**导引**：把**类型上**的 **`Generator[Yield, Send, Return]`** 与**运行时**的 **`.send` / `yield`** 对上号，并明确与 **`asyncio`** 原生协程的边界。深入调度与库设计请以官方 asyncio 文档与现代异步教材为准。

---

## 新手先看

1. **你现在写新业务**：优先 **`async def` / `await`**；不要新上大段「纯经典协程」框架。  
2. **你要读老代码 / 懂生成器**：经典协程是历史与原理的一部分，**和 `yield from`（见 [10 …](<./10-归约函数与 yield from：把 iterable 合拢、把生成器拆分（17.10–17.11）.md>)）同一脉络**。  
3. **类型提示**：经典协程对象常用 **`Generator[Yielded, Sent, Return]`** 第二参 **`Sent`** 表示 **`.send` 进来的类型**。

---

## 一、经典协程长什么样（最小形状）

```python
from collections.abc import Generator


def echo() -> Generator[str, str, None]:
    msg = yield "ready"  # 第一次 next() 后停在 yield；之后 send 的值赋给 msg
    while True:
        msg = yield f"echo: {msg}"


it = echo()
print(next(it))       # ready
print(it.send("hi"))  # echo: hi
```

要点：

- **第一次**通常要先 **`next(it)`**（或 **`send(None)`**），让执行流走到第一个 **`yield` 右侧** 并挂起。  
- 之后 **`it.send(x)`** 会把 **`x`** 作为 **`yield` 表达式的值** 赋给左侧变量。  
- **类型上**：**`Generator[str, str, None]`** ≈ 「yield 出 **`str`**；send 进来 **`str`**；若正常结束，`return` 的值类型为 **`None`**」。

---

## 二、`Generator[Yield, Send, Return]` 三参数怎么记

| 位置 | 含义 | 直觉 |
|------|------|------|
| **第 1 个 `Yield`** | **`yield expr` 产出的类型** | 外面 `next` / `for` 看到的 |
| **第 2 个 `Send`** | **`.send(x)` 里 `x` 的类型** | 传回生成器内部左侧变量 |
| **第 3 个 `Return`** | **生成器 `return v` 的 `v` 类型**（经 `StopIteration`） | 很多协程无限循环则 **`Never`/`NoReturn`** 或 **`None`** |

若生成器**从不**被 **`send` 非 `None`**，第二参常写 **`None`**（与 **12** 节里「只迭代、不驱动协程」的生成器一致）。

---

## 三、与 `async` / `await` 原生协程的区别（边界）

| | 经典协程（本节选型） | `asyncio` 原生协程 |
|--|----------------------|---------------------|
| **定义** | `def` + `yield` / `.send` | `async def` + `await` |
| **主要用途（今天）** | 遗留代码、教学、库内部技巧 | 并发 I/O、任务、事件循环 |
| **类型常见写法** | `Generator[...]`、`Iterator[...]` | `Coroutine[Any, Any, T]`、`AsyncIterator[T]` 等 |

**结论**：新代码的「协程」一词，默认指 **`async def`**；读到 **`yield` + `.send`** 时，再切到**经典协程**心智模型。

---

## 四、类型检查器能帮你什么、帮不了什么

- **能帮**：**`send` 类型与 `Yield`/`Send` 不一致**、漏 **`next`/`send(None)`** 导致的死锁式误用（部分场景靠人工规范）。  
- **很难帮**：复杂调度、与事件循环交织的运行时死锁；要靠测试与日志。

## 六、与 **12** 的关系

- **12**：偏「**数据面**」——`Iterable` / `Iterator` / `Sequence` 入参与返回。  
- **13**：偏「**控制面**」——**`Send`** 非 `None` 时，生成器已是**协程式**驱动；注解用 **`Generator[..., Send, ...]`** 把 **`send`** 的类型说清楚。
