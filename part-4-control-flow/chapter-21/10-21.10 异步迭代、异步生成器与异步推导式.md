# 21.10 异步迭代、异步生成器与异步推导式

## 一句话定位

异步迭代把第 17 章的“迭代协议”搬到了 async 世界：

- `for` → `async for`
- `__iter__`/`__next__` → `__aiter__`/`__anext__`

区别在于：**取下一个元素的动作变成了 awaitable**，需要事件循环调度。

---

## 异步可迭代对象 vs 异步迭代器

| 对象 | 必须实现 | 返回/语义 |
|------|----------|----------|
| **异步可迭代对象** | `__aiter__(self)` | 返回一个 **异步迭代器** |
| **异步迭代器** | `__anext__(self)` | 返回 awaitable；耗尽时抛 `StopAsyncIteration` |

对应同步协议：

- `__iter__` 返回迭代器
- `__next__` 返回下一个元素；耗尽时抛 `StopIteration`

---

## `async for` 在做什么

可以把它想象成伪代码：

```python
aiter = obj.__aiter__()
while True:
    try:
        item = await aiter.__anext__()
    except StopAsyncIteration:
        break
    ...
```

关键点：

- `__aiter__` 通常是普通方法（不 `await`）
- `__anext__` 必须能被 `await`（因为拿下一项往往要等 I/O）

---

## 异步生成器（async generator）

`async def` 里出现 `yield`：

- 函数调用返回异步生成器对象
- 它本身就是异步迭代器（可被 `async for` 消费）

优点：写异步数据流非常短。

---

## 异步生成器表达式与异步推导式

### 异步生成器表达式

```python
(expr async for x in aiter if cond)
```

### 异步列表/集合/字典推导式

```python
[expr async for x in aiter if cond]
{expr async for x in aiter}
{k: v async for x in aiter}
```

要点：这些都只能出现在 `async def` 里（因为内部需要 `await` 语义）。
