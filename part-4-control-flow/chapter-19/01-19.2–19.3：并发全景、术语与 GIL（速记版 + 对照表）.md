# 19.2–19.3：并发全景、术语与 GIL（速记版 + 对照表）

## 19.2 并发编程的难点：不是“启动”，是“收尾”

并发最容易低估的一点是：**启动线程/进程很容易，但收集结果、传播异常、正确退出很难**。

| 普通函数调用 | 并发执行单元（线程/进程/协程） |
|--------------|-------------------------------|
| 调用 → 阻塞 → 返回值直接拿到 | 提交任务 → 立刻返回“句柄”(future/task) → 需要额外机制等待、取回结果 |
| 异常可直接 `try/except` | 异常通常发生在另一个执行单元里，必须通过 future/队列等“信道”再传回来 |

当你把执行单元做成“worker 循环”（持续消费队列任务）时，又会新增一批必须处理的问题：

- **停止协议**：怎么通知 worker 退出（sentinel / cancel / join）？
- **资源释放**：退出时如何 flush、close、清理？
- **背压与争用**：任务太多时如何限流？锁/CPU 争用怎么处理？

## 19.3 术语统一（后面所有讨论都靠它）

| 术语 | 定义（够用版） |
|------|----------------|
| **并发（concurrency）** | 处理多个待办任务；可以在单核交替推进，也可以在多核并行推进 |
| **并行（parallelism）** | 真正同时执行多个计算任务；依赖多核/多 CPU/GPU/集群 |
| **执行单元** | 并发执行代码的对象：**进程**、**线程**、**协程** |
| **队列（queue）** | FIFO 通道，用于执行单元间交换数据/控制消息（也常用于背压） |
| **锁（lock）** | 同步原语，保护共享状态；避免数据竞争与破坏 |
| **争用（contention）** | 多个执行单元争夺共享资源（CPU、锁、IO 带宽）造成等待 |

## 三大执行单元：进程 / 线程 / 协程（对照）

| 执行单元 | 启动/切换成本 | 内存模型 | 适合 | 主要代价/限制 |
|----------|---------------|----------|------|---------------|
| **进程** | 高 | 隔离，默认不共享 | **CPU 密集**并行 | IPC 成本高，启动重 |
| **线程** | 中 | 同进程共享内存 | **I/O 密集** | 需要锁；CPython 有 **GIL** |
| **协程** | 低 | 同线程共享内存 | **高并发 I/O** | 任意一次阻塞会拖住整个事件循环；CPU 密集不合适 |

## GIL（全局解释器锁）：10 点速记

以下讨论默认指 **CPython**：

1. 一个 Python 解释器实例通常对应一个进程；可用 `multiprocessing`/`subprocess` 起新进程。
2. 进程内可以起多个线程（`threading`/`ThreadPoolExecutor`）。
3. **GIL**：同一时刻只有一个线程能执行 Python 字节码（与 CPU 核心数无关）。
4. 解释器会周期性让出 GIL（默认约 5ms，可用 `sys.getswitchinterval()` / `sys.setswitchinterval()` 调整）。
5. Python 代码不能显式“控制 GIL”，但很多 **C 扩展/系统调用**会释放 GIL。
6. 标准库里的大量 **I/O**（磁盘/网络/`sleep` 等）会释放 GIL。
7. C 扩展可以在不持有 GIL 的情况下跑原生线程，但这类线程不能随意操作 Python 对象。
8. 对 **I/O 密集**任务：线程通常有效（I/O 等待时释放 GIL，让别的线程跑）。
9. 对 **CPU 密集**任务：线程通常无加速，甚至更慢（GIL 争用 + 切换成本）。
10. 想让 **CPU 密集 Python 代码**在多核上跑起来：通常用 **多进程**（或把核心计算下沉到释放 GIL 的原生库/扩展）。

补充：

- GIL 是 CPython 的实现权衡，不是 Python 语言规范。
- 协程常见最佳实践是“单线程事件循环 + 大量协程”，避免线程级竞争，但仍要避免阻塞调用。

---

## 选型（最短可用版）

| 任务类型 | 优先方案 | 直觉理由 |
|----------|----------|----------|
| I/O 密集 | 线程 / 协程 | I/O 等待会释放 GIL；协程开销更低、并发量更大 |
| CPU 密集 | 多进程 | 避开 GIL，吃满多核 |
| 超高并发 I/O | 协程（`asyncio` 等） | 单线程可承载海量连接与等待 |
