# 9.7 `nonlocal` 与名字解析：闭包里为什么 `+=` 会炸（以及完整查找规则）

9.6 你已经看到闭包能“带着环境一起走”。但很多人第一次写“优化版 averager”会立刻撞上：

`UnboundLocalError: local variable 'count' referenced before assignment`

这一节把原因讲清楚：不是闭包不工作，而是**名字绑定规则**决定了你写的 `count += 1` 会触发“局部变量判定”。`nonlocal` 的作用就是把这个判定改回来。

配套脚本：`07_nonlocal_name_resolution_demo.py`（含带状态装饰器例子）。  

---

## 一、问题背景：从 list 方案到 count/total 方案

list 方案（可变对象，修改内容不需要重新绑定名字）：

```python
def make_averager():
    series = []

    def averager(new_value):
        series.append(new_value)     # 修改 list 内容，不是给 series 赋值
        return sum(series) / len(series)

    return averager
```

优化方案（不可变数值，需要重新绑定名字）：

```python
def make_averager():
    count = 0
    total = 0.0

    def averager(new_value):
        count += 1          # 等价于：count = count + 1
        total += new_value  # 等价于：total = total + new_value
        return total / count

    return averager
```

这段会报错，因为 `+=` 本质上包含**赋值**，而“函数体内出现赋值”会触发局部变量判定。

---

## 二、核心规则：赋值会让名字变成局部（除非声明 `global` / `nonlocal`）

Python 在**编译函数体**时就决定了每个名字是：

- 局部（local）
- 非局部（enclosing / closure free var）
- 全局（global）

规则可以记成一句话：

> **只要在函数体里给某名字赋值，它就会被判定为局部变量**，除非你显式声明 `global` 或 `nonlocal`。

因此在 `averager` 里出现 `count += 1`，`count` 就被判定为 local；执行 `count + 1` 时需要先读取 local `count`，但它还没绑定，于是 `UnboundLocalError`。

---

## 三、`nonlocal`：把名字绑定到“最近的外层函数作用域”

修复版：

```python
def make_averager():
    count = 0
    total = 0.0

    def averager(new_value):
        nonlocal count, total
        count += 1
        total += new_value
        return total / count

    return averager
```

`nonlocal` 的含义不是“让变量变全局”，而是：

- 在 **E（Enclosed）** 层级里寻找名字（最近一层外层函数作用域）；
- 绑定到那个名字，并允许在内层函数里对它重新绑定。

注意：`nonlocal` **不会**向上搜索模块全局；如果目标名字只存在于模块级，那就应该用 `global`。

---

## 四、`global` vs `nonlocal`（一句话区分）

| 关键字 | 绑定到哪里 | 典型用途 |
|---|---|---|
| `global x` | 当前模块的全局作用域 | 在函数里修改模块级变量 |
| `nonlocal x` | 最近的外层函数局部作用域 | 在闭包里修改外层函数变量 |

---

## 五、完整名字查找逻辑（LEGB 的工程版）

当你在函数里写到名字 `x`，决定顺序大致是：

1. **如果 `x` 声明了 `global`**：从模块全局取（并在赋值时写回模块全局）。
2. **如果 `x` 声明了 `nonlocal`**：从最近外层函数作用域取（并写回该作用域的绑定）。
3. **如果 `x` 是参数**，或函数体内对 `x` 有赋值：`x` 是 **local**（`LOAD_FAST`）。
4. 否则按 LEGB 查找：Enclosed → Global → Builtins（读取时会体现为 `LOAD_DEREF` / `LOAD_GLOBAL` 等指令）。

## 六、一个实战：用 `nonlocal` 写“带状态装饰器”（调用次数统计）

装饰器返回的 `wrapper` 本质就是闭包；如果你要在 wrapper 里更新计数器，就需要 `nonlocal`：

```python
from functools import wraps

def counted(func):
    calls = 0

    @wraps(func)
    def wrapper(*args, **kwargs):
        nonlocal calls
        calls += 1
        return func(*args, **kwargs)

    return wrapper
```

这类模式在工程里很常见：统计、限流、采样、熔断计数……都依赖“闭包 + nonlocal”。
