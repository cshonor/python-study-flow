# 9.2 装饰器基础知识：`@` 到底做了什么（以及它什么时候执行）

## 新手大白话：什么是装饰器？

**1）它是什么（一句话）**  
**装饰器 = 给函数加功能，但尽量不去改原函数里面的业务代码。**  
可以把它想成：给手机**套壳、贴膜**——手机还是那台手机，外面多了一层能力（保护、手感）。给函数则是多一层：**日志、计时、缓存、权限校验**等。

**2）必须记住的一句**  
**装饰器 = 语法糖 + 函数替换。**  
写成 `@decorate` 放在 `def` 上一行，**定义结束时**发生的事，等价于：

```python
target = decorate(target)
```

也就是：**名字还是 `target`，绑定的对象常常已经换成「增强版」的新函数（或别的可调用对象）了。**

**2.5）`@` 后面写的到底是什么？（帮你点破疑惑）**

- **`@xxx` 里的 `xxx` 不是保留关键字**，就是**一个普通的名字**：多数时候是你自己写的**装饰器函数名**，也可以是**类名**（类里用 `__call__` 当包装）、或库里已经定义好的名字（如 `functools.lru_cache`）。  
- **无参装饰器**：`@xxx` 等价于 **`def` 写完以后立刻执行** `func = xxx(func)` —— 也就是「**把下面的函数丢给 `xxx` 包一层，再用返回值盖住原来的名字**」。  
- **有参装饰器**（如 `@lru_cache(maxsize=128)`）是再多一层工厂，见 **9.10**；这里先记住：**`@` 右边始终是「可调用 / 返回可调用」这一套链条上的名字**。

**超简易执行流程（套一层在脑子里走一遍）**

1. 解释器先把你写的 **`def func...` 体**编译出一个**函数对象**。  
2. 立刻调用 **`xxx(那个函数对象)`**，得到**返回值**（通常是另一个函数或可调对象）。  
3. 把名字 **`func` 重新绑定**到这个返回值上。  
4. 以后写 **`func(...)`**，跑的是**包装后的**逻辑（里面再决定要不要调用「原来的那份」）。

**3）一眼能懂的例子**

```python
def my_decorator(func):
    def wrapper():
        print("函数运行前")
        func()
        print("函数运行后")
    return wrapper


@my_decorator
def hello():
    print("Hello")


hello()
```

运行时会依次打印：`函数运行前` → `Hello` → `函数运行后`。  
**`hello` 函数体里仍然只有一行业务打印**；前后多出来的两行，是装饰器包上去的。

**4）新手必背四条**

1. 装饰器的目标通常是：**少动原业务代码**，把横切逻辑往外抽。  
2. **`@` 只是快捷写法**；心里要会展开成 **`f = decorate(f)`**。  
3. **本质经常是「旧函数进，新函数出」**（新函数里再择机调用旧函数）。  
4. 最适合干的事：**日志、计时、缓存、权限**这类「很多函数都要来一遍」的重复劳动。

**5）可复制：最简「日志」装饰器模板（带 `wraps`，避免丢 `__name__`）**

```python
import functools


def log_calls(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print(f"[LOG] {func.__name__} args={args!r} kwargs={kwargs!r}")
        return func(*args, **kwargs)
    return wrapper


@log_calls
def add(a, b):
    return a + b


print(add(2, 3))
```

更完整的计时版、参数打印版见 **`08_clock_decorator_demo.py`** 与 **`08-9.8 …md`**。

## 新手：`self` 和装饰器别混（三条铁律 + 分界线）

### 三条铁律

1. **只要 `def` 不写在 `class` 里面**（模块顶层的普通函数、装饰器外层函数、工具函数）→ **永远没有 `self` 这回事**，也不需要写。  
2. **只有「写在 `class` 里的普通实例方法」**→ 约定第一个形参叫 **`self`**（名字可以改，但不建议），表示「将来绑定的那个实例」。  
3. **`@` 装饰器本质就是高阶函数**：`@deco` + `def f` 等价于 **`f = deco(f)`** —— 把**下面的函数对象**当参数传进 `deco`。

```python
def deco(f):  # f 就是被装饰的那个函数
    return f


@deco
def hello():
    pass


# 等价于：hello = deco(hello)
```

### 你容易混的分界线（一刀切开）

| 情况 | 要不要 `self` |
|------|----------------|
| **`class` 外面的 `def`**（含装饰器工厂外层） | **不要** |
| **`class` 里普通实例方法** | **要**（第一个参数） |
| **`@staticmethod`** | **不要** `self` |
| **`@classmethod`** | 第一个参数写 **`cls`**，不是 `self` |

### 一句大白话收尾

- **装饰器函数**：跟「类」无绑定关系时，**不是方法**，**不写 `self`**。  
- **`self` 只跟「类的实例方法」绑定**；搞不清就看：`def` 上面是不是在缩进在 `class` 里、有没有 `@staticmethod` / `@classmethod`。

这一节用最少的概念，讲清楚装饰器的本质和工作方式。你只要抓住一句话：

> **装饰器 = “语法糖 + 函数替换”。**

也就是说，`@` 只是让代码更好看；真正发生的事情是：**把原函数作为参数交给装饰器，再用装饰器返回的新可调用对象覆盖原函数名**。

---

## 一、核心定义

装饰器本质上是一个**接收函数作为参数的可调用对象**。它会对原函数做处理（增强/替换），然后返回一个**新的可调用对象**。

装饰器语法糖：

```python
@decorate
def target():
    print("running target()")
```

等价于：

```python
def target():
    print("running target()")

target = decorate(target)
```

关键点在最后一行：被装饰后，名字 `target` 会绑定到 `decorate(target)` 的返回值上——它**可能还是原函数**，也可能是一个**全新的函数/对象**。

---

## 二、关键示例：装饰发生了什么

看这个最小装饰器：

```python
def deco(func):
    def inner():
        print("running inner()")
    return inner

@deco
def target():
    print("running target()")
```

你可以按 3 步理解它：

### 2.1 装饰过程
`@deco` 会在函数定义完成后立刻执行一次装饰，相当于：

```python
target = deco(target)
```

### 2.2 调用结果
因此当你调用 `target()` 时：

- 你以为在跑原来的 `target`
- 但实际跑的是 `inner()`

输出会是：

```text
running inner()
```

### 2.3 对象验证
如果你打印 `target`（例如 `print(target)`），你看到的将是 `deco` 内部定义出来的 `inner` 函数对象，而不再是原来的 `target`。

## 三、装饰器的三大核心性质

### 3.1 装饰器本身是“可调用对象”
装饰器可以是：

- 函数
- 类（类本身可调用）
- 或任何实现了 `__call__` 的对象

总之，只要能写成 `decorate(target)`，它就能当装饰器用。

### 3.2 它会“替换”被装饰的函数
装饰器返回什么，名字就指向什么：

- 你可以返回原函数（仅做登记/注册等）
- 也可以返回一个新函数（最常见：在新函数里调用原函数并增强行为）
- 甚至返回完全不同的可调用对象（但这会改变调用语义，要谨慎）

### 3.3 它在“模块加载时”立即执行
装饰器的处理逻辑发生在**导入模块阶段**（准确说：当 Python 执行到函数定义这行代码时），而不是等到你调用 `target()` 才装饰。

因此很多“注册式装饰器”的副作用会出现在 **import 的那一刻**：一导入就完成登记/注册。

---

## 四、一句话总结

装饰器的核心是 **“语法糖 + 函数替换”**：`@` 只是简化写法，本质是把原函数作为参数传给装饰器，再用返回的新可调用对象覆盖原函数名，从而实现无侵入式增强功能。
