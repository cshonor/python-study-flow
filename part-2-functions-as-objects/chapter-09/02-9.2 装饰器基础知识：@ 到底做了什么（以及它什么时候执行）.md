# 9.2 装饰器基础知识：`@` 到底做了什么（以及它什么时候执行）

这一节用最少的概念，讲清楚装饰器的本质和工作方式。你只要抓住一句话：

> **装饰器 = “语法糖 + 函数替换”。**

也就是说，`@` 只是让代码更好看；真正发生的事情是：**把原函数作为参数交给装饰器，再用装饰器返回的新可调用对象覆盖原函数名**。

---

## 一、核心定义

装饰器本质上是一个**接收函数作为参数的可调用对象**。它会对原函数做处理（增强/替换），然后返回一个**新的可调用对象**。

装饰器语法糖：

```python
@decorate
def target():
    print("running target()")
```

等价于：

```python
def target():
    print("running target()")

target = decorate(target)
```

关键点在最后一行：被装饰后，名字 `target` 会绑定到 `decorate(target)` 的返回值上——它**可能还是原函数**，也可能是一个**全新的函数/对象**。

---

## 二、关键示例：装饰发生了什么

看这个最小装饰器：

```python
def deco(func):
    def inner():
        print("running inner()")
    return inner

@deco
def target():
    print("running target()")
```

你可以按 3 步理解它：

### 2.1 装饰过程
`@deco` 会在函数定义完成后立刻执行一次装饰，相当于：

```python
target = deco(target)
```

### 2.2 调用结果
因此当你调用 `target()` 时：

- 你以为在跑原来的 `target`
- 但实际跑的是 `inner()`

输出会是：

```text
running inner()
```

### 2.3 对象验证
如果你打印 `target`（例如 `print(target)`），你看到的将是 `deco` 内部定义出来的 `inner` 函数对象，而不再是原来的 `target`。

---

## 三、装饰器的三大核心性质

### 3.1 装饰器本身是“可调用对象”
装饰器可以是：

- 函数
- 类（类本身可调用）
- 或任何实现了 `__call__` 的对象

总之，只要能写成 `decorate(target)`，它就能当装饰器用。

### 3.2 它会“替换”被装饰的函数
装饰器返回什么，名字就指向什么：

- 你可以返回原函数（仅做登记/注册等）
- 也可以返回一个新函数（最常见：在新函数里调用原函数并增强行为）
- 甚至返回完全不同的可调用对象（但这会改变调用语义，要谨慎）

### 3.3 它在“模块加载时”立即执行
装饰器的处理逻辑发生在**导入模块阶段**（准确说：当 Python 执行到函数定义这行代码时），而不是等到你调用 `target()` 才装饰。

因此很多“注册式装饰器”的副作用会出现在 **import 的那一刻**：一导入就完成登记/注册。

---

## 四、一句话总结

装饰器的核心是 **“语法糖 + 函数替换”**：`@` 只是简化写法，本质是把原函数作为参数传给装饰器，再用返回的新可调用对象覆盖原函数名，从而实现无侵入式增强功能。
