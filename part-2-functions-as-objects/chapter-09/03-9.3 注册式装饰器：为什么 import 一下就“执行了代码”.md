# 9.3 注册式装饰器：为什么 `import` 一下就“执行了代码”

## 先别管「装饰器」三个字

你就想一件事：

**`import` 一个 `.py` 文件，等于 Python 帮你把这个文件从第一行读到最后一行，能执行的就先执行一遍。**

所以：**不是 import 特别神、特别快**，而是它本来就会「先跑一遍文件最外面的代码」。

---

## 那「最外面的代码」到底指什么？

指：**没有缩进到函数里面的那些行**。

- 写在函数**里面**的代码：默认**不会**因为你 import 就跑起来。  
- 写在函数**外面**的代码：import 的时候**会**按顺序跑。

---

## 本节的例子在讲什么？（对照 `03_registration.py`）

文件里有一个 `register`，它里面会 `print`，还会往列表 `registry` 里塞函数。

`f1`、`f2` 头上写了 `@register`。

### import 的时候，**已经发生了**什么？

1. **最外面**先建了一个空列表 `registry`。  
2. 最外面的 **`def register`**：只是在**准备**一个叫 `register` 的函数（**这时还不会跑** `register` 里面的 `print`，要等有人**调用** `register(...)`）。  
3. Python 先「造出」`f1` 这个函数（**同样：只是把函数准备好，不会立刻去跑 `f1` 里面的 `print`**）。  
4. 接着因为头上有 `@register`，Python **马上帮你多执行一步**：相当于写了 **`f1 = register(f1)`**。  
5. 这一步一执行，`register` **里面的** `print` 和 `append` 就真的跑了——所以你会看到 **`running register ...`**，列表里也会立刻有 `f1`。  
6. `f2` 同理，再来一遍。  
7. 后面的 **`def f3`、`def main`**：同样只是先把函数准备好，**不会**自动去调用它们。

**一句话：** import 时你看到的那些「登记」打印，来自 **`register` 这个函数被调用了**，不是来自 `f1()`。

### import 的时候，**还没发生**什么？

1. **`f1` 里面的** `print("running f1")`：**没跑**。要等你以后写 **`f1()`** 才会跑。  
2. **`f2` 里面的** `print("running f2")`：**没跑**。要等你以后写 **`f2()`** 才会跑。  
3. **`f3` 里面的** `print("running f3")`：**没跑**（而且 `f3` 头上没有 `@register`，所以也不会自动登记）。  
4. **`main` 里面的任何东西**：**没跑**。要有人**调用** `main()` 才会跑。  
5. **`if __name__ == "__main__":` 这一块**：别人 **import** 你时，一般**整块都不进**；只有你**自己双击/命令行运行这个文件**当脚本时，才会进。

---

## `@register` 你可以当成一句「自动多写的代码」

看见：

```python
@register
def f1():
    print("running f1")
```

你就把它想成：Python 在定义完 `f1` 之后，**立刻多帮你执行了一行**：

```python
f1 = register(f1)
```

所以：**`register` 里面的代码，会在定义阶段就跑**；**`f1` 里面的代码，要等 `f1()` 才跑**。

---

## 和「自己运行这个文件」差在哪？

- **自己运行**：`python 03_registration.py`  
  - 上面 import 时会跑的那一套，**照样会跑**。  
  - 另外还会进 `if __name__ == "__main__":`，于是会 **`main()`**，你才会看到 `running main()`、`running f1` 那些。

- **别人 import 你**：  
  - 上面那一套**照样会跑**（所以 `register` 的 print 还会出现）。  
  - 但 **`main()` 不会自动帮你跑**，除非你另外写代码去调用它。

---

## 完整代码（与仓库脚本一致；带类型标注）

细节以仓库里的 `03_registration.py` 为准（含类型提示与 `main`）。

```python
from collections.abc import Callable

registry: list[Callable[[], None]] = []


def register(func: Callable[[], None]) -> Callable[[], None]:
    print(f"running register {func}")
    registry.append(func)
    return func


@register
def f1() -> None:
    print("running f1")


@register
def f2() -> None:
    print("running f2")


def f3() -> None:
    print("running f3")


def main() -> None:
    print("running main()")
    print("registry ->", registry)
    f1()
    f2()
    f3()


if __name__ == "__main__":
    main()
```

---

## 最后只背一句（新手够用）

**import = 先跑文件最外面的代码；`@` 会让「登记函数」在定义时多跑一趟；函数里面的打印，要等你自己调用函数才跑。**
