# 9.3 注册式装饰器：为什么 `import` 一下就“执行了代码”

## 一句话

**`import` 会从上到下执行模块顶层代码**；执行到 `@register` 时，等价于立刻做 **`f1 = register(f1)`**，所以装饰器里的 `print` / `append` 会马上出现——**不是**等你以后才 `f1()`。

## 三句话（时机别混）

1. **`register(f1)`**：在 **`def f1` 写完、装饰器绑定那一刻**就执行（`print`、`registry.append` 都在这里跑）。  
2. **`f1` 函数体里的代码**：只有 **`f1()`** 被调用时才执行。  
3. **`if __name__ == "__main__":` 里的代码**：只有 **直接 `python 这个文件.py`** 时跑；**被别人 `import` 时不进这个块**。

## 等价写法（背这个就够）

```python
@register
def f1():
    ...
```

在解释器眼里，就是 **`def f1` 创建好函数对象之后，立刻执行**：

```python
f1 = register(f1)
```

所以「注册式」装饰器常用来：**模块被加载一次，就把要登记的函数都登记好**。

## 示例代码（与 `03_registration.py` 一致）

```python
from collections.abc import Callable

registry: list[Callable[[], None]] = []


def register(func: Callable[[], None]) -> Callable[[], None]:
    print(f"running register {func}")
    registry.append(func)
    return func


@register
def f1() -> None:
    print("running f1")


@register
def f2() -> None:
    print("running f2")


def f3() -> None:
    print("running f3")


def main() -> None:
    print("running main()")
    print("registry ->", registry)
    f1()
    f2()
    f3()


if __name__ == "__main__":
    main()
```

## 两种用法，各会看到什么

| 你怎么用 | 导入阶段 | 之后 |
|----------|----------|------|
| **`python .../03_registration.py`** | 两次 `running register ...`（`f1`、`f2`） | 进入 `main()`：`running main()`、`registry -> [...]`，再 `running f1` / `f2` / `f3` |
| **在别的代码里 `import` 这个模块** | 同样会先跑完两次 `running register ...` | **不会**自动跑 `main()`；只有你显式调用 `f1()` 等时，才打印函数体里的内容 |

模块名随你的 `PYTHONPATH` / 包结构而定，不必死记；抓住 **`import` = 执行顶层（含 `@`）** 即可。

## 小结

**加载模块时**：装饰器那一趟已经跑完（登记、副作用都在这里）。**`f1()`**：只负责跑「当前名字 `f1` 绑定的那个可调用对象」的函数体（本例里 `register` 返回原函数，所以体还是 `f1`；若返回包装函数，调的就是包装后的逻辑）。
