# 9.5 变量作用域：为什么“函数里一赋值就变局部”（用 `dis` 看证据）

## 先记一条铁律（和运行到哪一行无关）

**只要函数体里对某个名字出现了绑定（最常见是 `=` 赋值），整个函数里这个名字都会被当成局部变量**（除非你写了 `global` / `nonlocal`）。

这不是“执行到那一行才决定”的，而是 Python 在**把函数编译成字节码**的时候，就把这张表定好了。所以你才会看到：**上一行的 `print(b)` 也在读「局部 b」**，而局部 `b` 还没创建 → **`UnboundLocalError`**。

> 常见坑：**`+=`、`*=` 这类增强赋值**，也算对名字做绑定，同样会把整个函数里的该名字标成局部。

配套脚本：`05_scope_dis_demo.py`（可直接看 `dis.dis(f1)` / `dis.dis(f2)`）。

---

## 一、两个函数：长得差不多，命运不一样

```python
a = 10
b = 20


def f1():
    print(a)  # 整个函数里没有给 a 赋值 → a 按全局读


def f2():
    print(b)  # 你以为读的是全局 b…
    b = 9     # …但只要出现这一句，整个 f2 里的 b 全是「局部 b」
```

- **`f1()`**：编译器眼里 **`a` 没有绑定**，读的是 **global**。  
- **`f2()`**：编译器眼里 **`b` 有绑定**，**全程 local**，包括上面的 `print(b)`。

报错长这样（背关键词就行）：

`UnboundLocalError: local variable 'b' referenced before assignment`

---

## 二、f1 和 f2 一眼对照（人话版）

| 函数 | 函数体里有没有给这个名字「绑定」？ | 这个名字在函数里当什么？ |
|------|--------------------------------------|--------------------------|
| `f1` | 没有给 `a` 赋值 | **`a` → 全局** |
| `f2` | 有 `b = 9` | **`b` → 全程局部**（`print(b)` 也在读局部） |

---

## 三、用 `dis` 看铁证（最直观）

```python
import dis

dis.dis(f1)
dis.dis(f2)
```

你通常会看到类似差异：

- **`f1`**：读 `a` 时用 **`LOAD_GLOBAL`**（从全局名字空间取）。  
- **`f2`**：读 `b` 时用 **`LOAD_FAST`**（从局部快取槽取）——说明编译阶段已经把 **`b` 标成局部**。

（具体输出随 Python 版本略有出入，但**指令类别**的稳定对照就是上面这一对。）

---

## 三·附、用 `dis` 看字节码时，还容易踩的坑（和作用域同一套逻辑）

### 核心前提（先看懂这句）

`dis` 展示的是：**源码已经被编译成字节码之后**的样子。  
**某个名字在函数里是全局还是局部（还是闭包 cell），是在编译阶段就定好的**——不是执行到那一行才「临时决定」。这就是 **`UnboundLocalError`** 和 **`LOAD_GLOBAL` / `LOAD_FAST`** 对得上的根源。

### 1）函数里一旦出现绑定，整段函数里该名都是局部（你已会）

和正文「铁律」同一件事：字节码里读局部常见是 **`LOAD_FAST`**；只读全局常见是 **`LOAD_GLOBAL`**。  
写全局则是 **`STORE_GLOBAL`**，写局部是 **`STORE_FAST`**——别只盯 `LOAD_*`，**存**到哪一类名字空间也要分清楚。

加了 **`global x`** 之后：读 `x` 走 **`LOAD_GLOBAL`**，给 `x` 赋值走 **`STORE_GLOBAL`**（不再走 `FAST`）。  
加了 **`nonlocal x`** 之后：往往走 **`LOAD_DEREF` / `STORE_DEREF`**（闭包 cell），和全局又不是一回事。

### 2）`dis` 最好对着「函数 / 方法 /代码对象」看，不要指望拆「零散半行」

像 `dis.dis(print(a))` 这种，往往**不是你想象的那种教学视图**；新手最稳的是：**把逻辑写进 `def`，再 `dis.dis(函数名)`**。  
配套脚本 **`05_scope_dis_demo.py`** 已经打好 `f1` / `f2`，直接跑就能对照本文。

### 3）分支、循环、嵌套函数：会多出跳转指令

`if` / `for` / `while`、嵌套 `def` 会让字节码出现 **`JUMP_*`** 等跳转；读的时候别当成「从上到下一行行执行」的线性小说。

### 4）`dis` 里带的**源码行号**，是物理行提示，不是「逻辑时刻表」

指令顺序是编译器排的；**别只靠行号脑补执行顺序**，以指令流为准。

### 5）想看「多一点点上下文」：加 `show_codelines`（版本要够新）

Python **3.11+** 起，`dis.dis` 支持把源码行和指令对齐得更清楚，例如：

```python
import dis

dis.dis(f2, show_codelines=True)
```

更花哨的选项以你本机 **`help(dis.dis)`** 为准（不同小版本参数会增减，**别死记不存在的参数名**）。

### 6）推导式、`lambda` 往往是**另一份**代码对象

它们有自己的 `code` 对象；作用域规则和「外层函数」不一定同一张脸。想看清就 **`dis.dis(那个 lambda)`** 或把推导式赋给名字再 `dis`，别和外层混在一块猜。

### 7）交互环境里改了函数定义，记得重新 `dis`

`dis` 看的是**当前内存里这份函数对象**；你在 REPL 里重新定义了 `f2`，就要**重新** `dis.dis(f2)`，避免对着旧字节码说话。

### 8）不同 Python 大版本，指令名/细节可能变

不必背每一条 opcode；本课抓 **`LOAD_GLOBAL` / `LOAD_FAST` / `LOAD_DEREF`** 和对应的 **`STORE_*`** 这几类就够用了。

### 三句收工

1. **`dis` 看的是编译结果，作用域在编译期就定了。**  
2. **有绑定 → 局部读写走 `FAST`；`global` / `nonlocal` 会改写成别的 `LOAD/STORE` 路线。**  
3. **包成函数再 `dis`；行号辅助看，别当执行顺序本身。**

## 四、怎么修？

### 方法 1：就是要改全局 → 写 `global`

```python
def f2_fixed():
    global b
    print(b)
    b = 9
```

语义：**在这个函数里，`b` 这两个字母指全局那个名字**。

### 方法 2：工程上更常见 → 别靠可变全局

用**参数传入**、**类的属性**、**闭包**等，把状态收进你能看见的作用域里，可读性和测试都会好很多。

### 和外层函数有关时 → 用 `nonlocal`（本节不展开，只记名字）

要改的是**外层函数里的变量**（不是模块全局），用 **`nonlocal`**，道理和 `global` 类似：**先声明再读写**，别和全局混了。

## 五、增强赋值也算绑定（别漏）

```python
n = 0


def bump_wrong():
    print(n)
    n += 1  # 等价于对 n 做绑定 → 整个函数里 n 都是局部


def bump_ok():
    global n
    print(n)
    n += 1
```

## 六、口诀（记一辈子）

**函数内部一赋值，全程变成局部名；**  
**编译阶段就定死，运行顺序不顶用。**
