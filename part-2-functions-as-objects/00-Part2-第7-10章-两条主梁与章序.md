# Part 2（第 7–10 章）两条主梁 + 章序一条线

对应《流畅的 Python》（**第二版**）**Part 2** 与仓库目录 `part-2-functions-as-objects/`。  
新手先抓住**两个底层问题**的答案，再按章往下读，不容易散。

---

## 一、这一大块在讲什么？

**一句话**：不再把函数只当成「定义完就调一下」的语法块，而是把 **函数当成和 int、list 一样的实体**——能绑定名字、能传参、能返回、能嵌套、能**包住**外层变量、能用**另一个函数去包装**它。

这是 **高阶函数、闭包、装饰器、用函数替代部分设计模式** 的共同地基。

---

## 二、问题一：函数**什么时候**被创建？

### 新手版（人话）

1. 源码里的 **`def` 不会**在「你保存文件的一瞬间」就全部变成函数。  
2. Python **按执行流**跑：模块导入、脚本运行，都是**从上到下**（再配合函数调用栈）。  
3. **执行流真正跑到某条 `def`（或包含 `lambda` 的表达式）时**，解释器才**创建**对应的函数对象，并把名字绑上去。  
4. **没执行到** → 对当前作用域来说这个函数**还不存在**，去调就会 **`NameError`**。

**一句背**：**`def` 是普通可执行语句；走到它，函数才「出生」。**

### 和书上怎么对？

即第 7 章说的：**函数是运行时创建的一等对象**。

**展开与反例**（先调用后 `def`、`if` 里从不 `def`）：`chapter-07/02-示例 7-1 7-2：函数是对象、可赋值、可传给 map（高阶函数）.md` **§1.25**。

## 三、问题二：创建出来的函数，在 **CPython** 底层是什么？

### 标准答案（够用版）

> **语言规范**里：用户函数是 Python 的 **`function` 类型**（`types.FunctionType`）。下面说的是 **CPython 实现**，换解释器可能不同。

1. 你用 **`def` / `lambda` 写的普通用户函数**，在 CPython 里通常都走**同一种 C 级布局**，源码侧常对应 **`PyFunctionObject`**：**一份模板、多个实例**。  
2. **外壳/槽位一致**，**每个实例里填的数据不同**：名字、`__doc__`、字节码（`__code__`）、参数与默认值、注解、闭包单元等。  
3. **内置**如 `len`、`print` → 另一路（如 **`PyCFunctionObject`**），Python 里常见 **`builtin_function_or_method`**，**不是**上面的 **`PyFunctionObject`**。  
4. **类体里的 `def`** 常见仍是用户 **`function`**；从实例上取 **`obj.meth`** 往往是 **`method` 绑定方法**（函数 + `self`），别和「类命名空间里那份裸函数」混成同一个对象。

**一句背（记得加「CPython」）**：**自定义 `def`/`lambda` 在 CPython 里共用 `PyFunctionObject` 这一套；内置走别的 C 实现。**

**展开**：同篇 **§1.5**：`chapter-07/02-示例 7-1 7-2：函数是对象、可赋值、可传给 map（高阶函数）.md`。

## 四、第 7–10 章怎么串成一条线？（与**第二版**目录一致）

下面按**本书第二版**实际章题写（与仓库 `chapter-07` … `chapter-10` 对齐）。  
**高阶函数、`map`、多种参数形态**主要落在 **第 7 章**；**第 8 章**不是「再讲一轮传函数」，而是**类型提示**。

| 章 | 主题（第二版） | 在「函数即对象」这条线上的位置 |
|----|----------------|--------------------------------|
| **7** | 函数是一等对象 | **地基**：对象观、`map`/高阶、`lambda`、九种 callable、`__call__`、工具链 |
| **8** | 函数中的类型提示 | 在**同一套可调对象/签名**上，加 **PEP 484 注解**与渐进类型（mypy/pyright 边界） |
| **9** | 装饰器与闭包 | **嵌套 + 闭包**（函数携带外层绑定）；**装饰器**（用函数包装函数） |
| **10** | 用一等函数实现设计模式 | **策略、命令**等：用**函数/闭包**替代部分类层级 |

**整段 Part 2 反复用的还是那两条：**

1. 函数是 **`def` 执行到才创建**的运行时对象。  
2. （CPython）用户自定义函数实例多是 **`PyFunctionObject` 这一路**；内置另说。

---

## 五、极简背诵（两条）

1. **函数不是「写在文件里就自动存在」**——**执行到 `def`（或 `lambda` 所在表达式）才现场创建**。  
2. **（CPython）`def`/`lambda` 用户函数共用 `PyFunctionObject` 结构模板**；内置 `len`/`print` 等走 **`PyCFunctionObject` 等**另一套。
