## 零、大白话背诵版（与 **8.6 / 8.7**、文末速查表一致）

### 8.6 仅限位置与变长注解

#### 1）`/` 仅限位置

**`/` 前面的形参**：调用时**只能按位置传**，**不能**写 **`名=值`**。

```python
def f(a: int, /, b: str) -> None: ...
```

- **合法**：**`f(1, "x")`**  
- **非法**：**`f(a=1, b="x")`**（`/` 左侧禁止关键字实参）

**用处**：参数名可当实现细节，以后改名时**不鼓励**调用方依赖关键字（API 更稳）。

#### 2）`*args: T`

**含义**：多出来的**每一个**位置实参，类型都应是 **`T`**；概念上体内当 **`tuple[T, ...]`** 用。

```python
def func(*args: str) -> None: ...
```

#### 3）`**kwargs: T`

**含义**：多出来的关键字实参，**键**在类型上视为 **`str`**，**每个值**都应是 **`T`**；概念上体内当 **`dict[str, T]`**。

```python
def func(**kwargs: int) -> None: ...
```

#### 4）混合模板（工作常见）

```python
def tag(
    name: str,
    /,
    *content: str,
    class_: str | None = None,
    **attrs: str,
) -> str:
    ...
```

（与 **`07_tag_type_hints_demo.py`** 中实现一致；**`class_`** 避开保留字 **`class`**。）

#### 5）变长「多形状」怎么办

- 单一注解兜不住时，常用 **`Any`** 折中，或**改 API**（显式参数、单个 **`dict`** / **`Mapping`**）。  
- **`TypedDict`**：适合「**键集合固定、值类型按键变化**」的 **`kwargs`** 形状（见 **8.5**、**TypedDict** 文档）。

### 8.7 类型系统局限性（三观）

1. **静态检查 ≠ 程序正确**：只抓**类型形状**与大量静态错误，**不管**业务对错、算法、边界。  
2. **误报（False Positive）**：收窄注解、**`Protocol`**、升级工具；必要时**单行** **`# type: ignore[具体码]`** 并写原因。  
3. **漏报（False Negative）**：滥用 **`Any`**、动态 **`setattr`**、元编程、未校验的外部数据等。  
4. **实践**：**类型**挡低级静态问题，**测试**挡行为与盲区；渐进式类型先铺**公共 API / 热路径**，不追求一上来 100%。

---

### 一句话口诀

- **`/`** 左边：**只能位置**，别写关键字。  
- **`*args: T`**：**每个位置实参**都是 **`T`**。  
- **`**kwargs: T`**：**每个值**都是 **`T`**（键当 **`str`**）。  
- **类型只查形状**，**不能代替测试**。

## 8.6 仅限位置参数、变长参数

### 1. 典型形态（以 `tag` 为例）

```python
def tag(
    name: str,
    /,
    *content: str,
    class_: str | None = None,
    **attrs: str,
) -> str:
    ...
```

- **`/` 之前**：**仅限位置**——不能写 `tag(name="img")`，只能 `tag("img", ...)`。
- **`*content: str`**：捕获的“多余位置参数”在类型上视为**同构元组** `tuple[str, ...]`；注解表示**每一个**可变位置实参都应是 `str`。
- **`**attrs: str`**：关键字参数的值在类型上为 **`dict[str, str]`**（键始终是 `str`）；注解表示**每一个**关键字实参的值都应是 `str`。
- **`class_`**：普通命名参数（在 `*` 之后可用关键字），因 `class` 是关键字而写成 `class_`。

### 2. Python 版本

- **3.8+**：推荐使用 **`/`**（PEP 570），语义清晰。
- **更早版本**：无 `/` 时，可用 **双下划线前缀** `__name` 等形式配合检查器约定（以你使用的 mypy 文档为准）；新代码优先 3.8+ 语法。

### 3. 变长参数需要“多种类型”时

- 注解只能给**一种**元素/值类型时：常用折中是 **`Any`**，或把 API 改成接收**一个**映射/序列而不是裸 `**kwargs`。
- `**kwargs: str | float` 表示**每个值**都是 `str` 或 `float`（而不是“有的键是 str 有的是 float”那种混合结构——那种更适合显式参数或 `TypedDict`）。

## 8.7 类型系统的局限性：静态检查 ≠ 正确性证明

### 1. 结论

mypy / pyright **不能保证**程序在语义上 100% 正确；类型提示是**高信噪比的早期防线**，不是替代品。

### 2. 误报（False Positive）

检查器认为有错，但你在该约束下知道没问题。

- **应对**：收窄注解、补 `Protocol`、升级工具版本；必要时对**单行**使用 `# type: ignore[code]` 并写清原因（避免全局忽略）。

### 3. 漏报（False Negative）

程序有错，检查器没吭声。

- 常见来源：滥用 **`Any`**、动态 `setattr`、元编程、外部数据未校验等。
- **应对**：测试（单元 / 集成 / 属性测试）、边界输入、契约（schema）与运行时校验（按领域需要）。

### 4. 实践原则

- **渐进式类型**：先覆盖热路径与公共 API，不追求一上来 100% 覆盖率。
- **类型 + 测试**：类型负责“形状与大量静态错误”，测试负责“行为与工具盲区”。

## 速查表

| 语法 | 含义 | 体内常见类型（概念上） |
|------|------|-------------------------|
| `x: T, /` | 仅限位置 | `T` |
| `*args: T` | 可变位置，**每个**为 `T` | `tuple[T, ...]` |
| `**kw: T` | 可变关键字，**每个值**为 `T` | `dict[str, T]` |
| `__x`（旧式） | 仅限位置（兼容向） | 依检查器 |
