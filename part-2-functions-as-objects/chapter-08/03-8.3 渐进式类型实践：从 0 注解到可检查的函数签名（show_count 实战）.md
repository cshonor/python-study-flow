# 8.3 渐进式类型实践：从 0 注解到可检查的函数签名（`show_count` 实战）

## 第一步：先看「没写类型」的长什么样

```python
def show_count(count, word):
    if count == 1:
        return f"1 {word}"
    count_str = str(count) if count else "no"
    return f"{count_str} {word}s"
```

它运行起来没问题，例如：

- `show_count(99, "bird")` → `"99 birds"`  
- `show_count(1, "bird")` → `"1 bird"`  
- `show_count(0, "bird")` → `"no birds"`

### 你可能会问：那我跑 mypy，它怎么好像不说话？

在**默认、偏宽松**的设置下，mypy 对「没写类型的形参」常常先当成很宽的类型（效果上接近 **`Any`**）。  
**渐进式**的意思里，有一层就是：**没写就先不逼你**——所以你会觉得「明明没注解，也不报错」。

如果你希望团队里 **「新函数必须写全签名」**，就要靠下一节的两个开关（或更严的 `strict`），把「渐进」变成**你们自己定的规矩**。

## 第二步：让 mypy「认真起来」（两个最好记的开关）

官方说明见 [Command line](https://mypy.readthedocs.io/en/stable/command_line.html)、[Config file](https://mypy.readthedocs.io/en/stable/config_file.html)。

### 开关一：`--disallow-untyped-defs`

大意：**不许函数「参数和返回都裸着」**。该写的类型要写全。

```python
# 在开了这个开关时，容易看到类似：缺少返回类型 / 参数未注解
def add(a, b):
    return a + b


def add_ok(a: int, b: int) -> int:
    return a + b
```

### 开关二：`--disallow-incomplete-defs`

大意：**不许写一半**。例如只标了部分参数，或标了参数却**没写返回类型**。

```python
# 容易报：注解不完整
def bad1(a: int, b) -> int:
    return a + int(b)


def bad2(a: int, b: int):
    return a + b
```

### 命令行一把梭（常用组合）

```bash
mypy --disallow-untyped-defs --disallow-incomplete-defs your_module.py
```

### 写进项目（复制粘贴友好）

**`pyproject.toml`（推荐）**：

```toml
[tool.mypy]
python_version = "3.10"
disallow_untyped_defs = true
disallow_incomplete_defs = true
warn_return_any = true
show_error_codes = true
```

**或 `mypy.ini`**：

```ini
[mypy]
python_version = 3.10
disallow_untyped_defs = True
disallow_incomplete_defs = True
warn_return_any = True
show_error_codes = True
```

仓库里还有一份带 **`tests.*` 放宽** 等示例的片段：**`mypy_strict_pyproject_snippet.toml`**，和这里对齐，合并时注意别覆盖你项目里已有的 `[tool.mypy]`。

### 关于 `strict = true`

```toml
[tool.mypy]
strict = true
```

这会**一次性打开一大包**严格选项（具体以你安装的 mypy 版本文档为准）。

- **全新小项目**：可以试试从 strict 起步。  
- **老项目**：更稳的是**先只开上面两个核心开关**，按目录慢慢加，避免一天冒出几千条报错。

### 实战小建议（三行）

1. 先 **`disallow_untyped_defs` + `disallow_incomplete_defs`**，让**新代码**签名完整。  
2. 尽量**按包 / 目录**收紧，而不是全仓库一刀切。  
3. 测试目录常常需要放宽（示例见 `mypy_strict_pyproject_snippet.toml` 里的 `overrides`）。

## 第三步：给 `show_count` 一点一点穿上「类型外衣」

### 3.1 先只写「返回什么」

```python
def show_count(count, word) -> str:
    ...
```

含义很简单：**我保证这个函数最后吐出来的是字符串**。  
在比较严的模式下，mypy 还会催你：参数也写一写。

### 3.2 再把参数写清楚

```python
def show_count(count: int, word: str) -> str:
    ...
```

到这里，调用方如果写成 `show_count("a lot", "bird")`，**不用跑程序**，检查器就有机会提前喊停。

## 第四步：需求变复杂——不规则复数

想要：`show_count(3, "mouse", "mice")` → `"3 mice"`。

### 一个「能跑，但意思有点糊」的写法

```python
def show_count(count: int, singular: str, plural: str = "") -> str:
    if count == 1:
        return f"1 {singular}"
    count_str = str(count) if count else "no"
    if not plural:
        plural = singular + "s"
    return f"{count_str} {plural}"
```

问题不在运行，而在**表达**：`plural=""` 既像「用户传了空字符串」，又像「用户没传」，读起来费劲。

### 更干净的习惯：用 `None` 表示「没提供」

- **能不能省略这个参数**：由**有没有默认值**决定（`= None` 才能省略）。  
- **允不允许真的是 `None` 这个值**：由**类型里写不写 `Optional` / `| None`** 决定。

```python
from typing import Optional

def show_count(count: int, singular: str, plural: Optional[str] = None) -> str:
    if count == 1:
        return f"1 {singular}"
    count_str = str(count) if count else "no"
    if plural is None:
        plural = singular + "s"
    return f"{count_str} {plural}"
```

（Python 3.10+ 也可以写成 `plural: str | None = None`，和 `Optional[str]` 是一类意思。）

---

## 新手最常打错的一类：`color=str` 不是类型

**错**（把「默认值」当成了「类型」）：

```python
def hex2rgb(color=str) -> tuple[int, int, int]:
    ...
```

**对**：

```python
def hex2rgb(color: str) -> tuple[int, int, int]:
    ...
```

记法：**类型跟在参数名后面，用冒号 `:`**；**默认值跟在类型后面，用等号 `=`**。
