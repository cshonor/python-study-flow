# 8.3 渐进式类型实践：从 0 注解到可检查的函数签名（`show_count` 实战）

8.2 讲的是“渐进式类型”的哲学：可选、可逐步、主要给工具看。  
8.3 则是把这套哲学落到最常见的工作流：**从一个完全无注解的函数开始，逐步加上类型提示，并用静态检查器把问题提前暴露出来**。

本节用一个很小但很真实的函数 `show_count` 做贯穿案例：根据数量返回单复数文本。

配套脚本：`03_show_count_demo.py`（包含最终版实现 + 断言测试）。  

---

## 一、起点：没有任何类型提示的函数（示例 8-1）

```python
def show_count(count, word):
    if count == 1:
        return f"1 {word}"
    count_str = str(count) if count else "no"
    return f"{count_str} {word}s"
```

它能用：

- `show_count(99, "bird")` → `"99 birds"`
- `show_count(1, "bird")` → `"1 bird"`
- `show_count(0, "bird")` → `"no birds"`

### 1.1 mypy 默认为什么“看起来什么都没做”

mypy 默认策略是：**只对“它看得懂类型”的部分严格**。未注解的函数体里，形参往往被当成 **`Any`** 处理（渐进式：**不拦、不报**），所以你会感觉「明明没写类型，mypy 也不吵」——这在大型项目里容易变成**类型黑洞**。收紧办法见 **§二**。

---

## 二、让 mypy 变严格：把“渐进”变成“可控”

默认 **mypy** 偏**渐进式**：没注解的地方常常按 **`Any`** 放行，不逼你一次写全。要在团队里把约束**落地、可强制**，用**命令行**或 **`pyproject.toml` / `mypy.ini`** 逐步收紧（官方文档：[Command line](https://mypy.readthedocs.io/en/stable/command_line.html)、[Config file](https://mypy.readthedocs.io/en/stable/config_file.html)）。

### 2.1 两个最常用开关

#### `--disallow-untyped-defs`

**禁止「完全无注解」的函数**：参数与返回值都应有类型（对 mypy 正在检查的那批定义生效）。

```python
# 报错：Function is missing a return type annotation（以及参数未注解等，依版本/配置措辞略有不同）
def add(a, b):
    return a + b


def add_ok(a: int, b: int) -> int:
    return a + b
```

#### `--disallow-incomplete-defs`

**禁止「写一半」**：例如只标了部分参数、或标了参数却**没写返回类型**。

```python
# 报错：类型注解不完整
def bad1(a: int, b) -> int:
    return a + int(b)


def bad2(a: int, b: int):
    return a + b
```

**常见组合**（团队 CI 里很常用）：

```bash
mypy --disallow-untyped-defs --disallow-incomplete-defs your_module.py
```

### 2.2 写进配置文件（项目级）

**`pyproject.toml`（推荐）**：

```toml
[tool.mypy]
python_version = "3.10"
disallow_untyped_defs = true
disallow_incomplete_defs = true
warn_return_any = true
show_error_codes = true
```

**或 `mypy.ini`**：

```ini
[mypy]
python_version = 3.10
disallow_untyped_defs = True
disallow_incomplete_defs = True
warn_return_any = True
show_error_codes = True
```

仓库内可复制片段（含 **`tests.*` 放宽** 示例）：**`mypy_strict_pyproject_snippet.toml`**（与本节 **§二** 对齐，按需合并进你自己的 **`pyproject.toml`**）。

### 2.3 `strict = true`（一键拉满，慎用）

```toml
[tool.mypy]
strict = true
```

等价于打开**一整包**严格选项（含 **`disallow_untyped_defs`**、**`disallow_incomplete_defs`**、**`disallow_untyped_calls`**、**`disallow_any_generics`** 等，以你安装的 mypy 版本文档为准）。

- **新项目**：可以考虑从 **`strict = true`** 起步。  
- **老项目**：更稳的是 **§2.1 两个核心开关先开**，再按模块/目录逐步加码，避免一次性被几千条报错淹没。

### 2.4 实战落地建议

1. **先开** **`disallow_untyped_defs` + `disallow_incomplete_defs`**，先让**新代码**完整注解。  
2. **按包 / 目录**收紧，而不是全仓库一刀切（配合 **`files` / `packages`** 或 CI 里只对变更路径跑）。  
3. **测试目录放宽**（示例，路径按项目改）：

```toml
[[tool.mypy.overrides]]
module = "tests.*"
disallow_untyped_defs = false
```

### 2.5 一句话背

- **`disallow-untyped-defs`**：**不许**「参数和返回都裸着」。  
- **`disallow-incomplete-defs`**：**不许**「只写一半注解」。  
- **组合** ≈ **强制完整函数签名**（在 mypy 检查范围内），把渐进式类型变成**可执行的团队规则**；**仍不能替代测试**（见 **8.7**）。

## 三、第一步：只加返回值类型（让工具知道“你想返回什么”）

```python
def show_count(count, word) -> str:
    ...
```

这一步的意义：

- 你先把“输出契约”固定住；
- mypy 会进一步提醒你：参数也需要类型（在严格模式下）。

---

## 四、第二步：补全参数类型（把边界写清楚）

```python
def show_count(count: int, word: str) -> str:
    ...
```

此时静态检查器能做的事就多了：

- 调用处如果写 `show_count("a lot", "bird")`，它可以提前报错；
- IDE 也能更好地补全和提示。

---

## 五、需求升级：加入不规则复数（第三个参数）

我们希望支持：

- `show_count(3, "mouse", "mice")` → `"3 mice"`

最直接的扩展是加入 `plural` 参数。

### 5.1 一个常见“写得能跑，但类型不理想”的版本

```python
def show_count(count: int, singular: str, plural: str = "") -> str:
    if count == 1:
        return f"1 {singular}"
    count_str = str(count) if count else "no"
    if not plural:
        plural = singular + "s"
    return f"{count_str} {plural}"
```

这个版本的缺点不是“运行”，而是“表达意图”：

- `plural=""` 既表示“用户传了空字符串”，也表示“用户没传”
- 语义不够干净

---

## 六、用 `None` 表达“没提供”：`Optional[str]`（或 `str | None`）

这是 Python 类型提示里非常常用的模式：

- **可选参数**：用 `= None` 表示“没传”
- **可选类型**：用 `Optional[T]`（或 3.10+ 的 `T | None`）表示“可能为 None”

最终推荐版：

```python
from typing import Optional

def show_count(count: int, singular: str, plural: Optional[str] = None) -> str:
    if count == 1:
        return f"1 {singular}"
    count_str = str(count) if count else "no"
    if plural is None:
        plural = singular + "s"
    return f"{count_str} {plural}"
```

这里有两条新手常混淆的规则：

- **“参数可不可以省略”** 由 **是否有默认值**决定（`= None` 让它可省略）
- **“参数允许不允许为 None”** 由 **类型注解**决定（`Optional[str]` 允许为 `None`）

---

## 七、最常见语法错误：把默认值当成注解

错误写法：

```python
def hex2rgb(color=str) -> tuple[int, int, int]:
    ...
```

这里 `color=str` 是“默认值是 `str` 这个对象”，不是“类型是 str”。  
正确写法应该是：

```python
def hex2rgb(color: str) -> tuple[int, int, int]:
    ...
```
