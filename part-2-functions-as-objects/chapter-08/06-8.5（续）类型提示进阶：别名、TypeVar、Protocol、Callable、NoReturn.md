## 零·附、新手速查：六块核心（大白话 + 可直接复制运行）

下面与 **§一～§六** 同一套知识点，用**更口语**的句子 + **一整段可粘贴**的示例串起来；想展开定义与边界再往下读 **「零、大白话压缩版」** 与正文各节。

#### 1）类型别名（`TypeAlias`）：给复杂类型起短名

**作用**：注解里少写长串，**读起来像目录**；**不改变运行时行为**。

```python
from typing import TypeAlias

FromTo: TypeAlias = tuple[str, str]


def swap(xy: FromTo) -> FromTo:
    return (xy[1], xy[0])
```

#### 2）`Iterable` vs `Sequence`（必背口诀）

- **`Iterable[T]`**：只要能 `for` 遍历（最宽），生成器、集合等都可以。  
- **`Sequence[T]`**：能遍历，还能 **`len()`** 和 **`[下标]`**（更严），如 `list` / `tuple` / `str`。

**口诀**：**只遍历用 `Iterable`，要长度/索引用 `Sequence`。**

```python
from collections.abc import Iterable, Sequence


def print_all(items: Iterable[str]) -> None:
    for item in items:
        print(item)


def first_item(items: Sequence[str]) -> str:
    return items[0]
```

#### 3）`TypeVar`（泛型）：把「输入类型」和「输出类型」绑在一起

**作用**：写「通用小工具」时，尽量不用 `Any`，让 **进 `list` 里是什么，取出来还是什么**。

```python
from typing import TypeVar

T = TypeVar("T")


def pick(items: list[T]) -> T:
    return items[0]


num = pick([1, 2, 3])
txt = pick(["a", "b"])
```

**进阶：只允许少数几种类型**（联合上界的一种写法）：

```python
NumberT = TypeVar("NumberT", float, int)


def double(x: NumberT) -> NumberT:
    return x * 2
```

#### 4）`Protocol`（静态鸭子类型）：看「有没有方法」，不看「有没有继承」

**作用**：不强迫对方继承某个基类；**只要长得够用**（该有方法都有），mypy / Pyright 就认。

下面用**最好记的名字：动物、猫、狗**，新手版手把手过一遍；书里也常见 **`Quackable` + `Duck`（嘎嘎叫）**，**事理完全一样**。

##### 4·A）先定义一个协议（当成「动物标准」）

```python
from typing import Protocol


class Animal(Protocol):
    # 只规定：必须会叫；这里用 `...` 表示「只描述形状，不写实现」
    def cry(self) -> str: ...
```

人话：`Animal` 写在 **`Protocol` 后面**，主要是给**静态检查**用的「**行为合同**」——**规定：只要会 `cry()` 就算过关**。（运行时默认不会像某些语言接口那样自动拦住「不会叫」的对象；要运行期检查可看 `typing.runtime_checkable`，此处不深。）

##### 4·B）写两个普通类：猫、狗（**都不**写 `class Dog(Animal)`）

```python
class Dog:
    def cry(self) -> str:
        return "汪汪叫"


class Cat:
    def cry(self) -> str:
        return "喵喵叫"
```

##### 4·C）函数参数用 `Animal` 做类型提示

```python
def hear_animal(ani: Animal) -> None:
    print(ani.cry())
```

##### 4·D）直接传狗、传猫

```python
hear_animal(Dog())
hear_animal(Cat())
```

**大白话总结（超好记）**

1. **`Animal(Protocol)`**：变成「**接口形状 / 行为标准**」——只要会 **`cry()`** 就算合格动物。  
2. **`Dog`、`Cat` 不用继承 `Animal`**：只要有**同样形状**的 **`cry()`** 方法。  
3. **参数写 `ani: Animal`**：**谁长得符合这个标准，谁就能传**，猫狗随便进。

**一句口诀**：**`Protocol` 写标准，不必显式继承；方法长得一样，就能往里传。**

##### 4·E）同一套理：嘎嘎叫版（对照用）

```python
from typing import Protocol


class Quackable(Protocol):
    def quack(self) -> None: ...


class Duck:
    def quack(self) -> None:
        print("嘎嘎")


def make_sound(animal: Quackable) -> None:
    animal.quack()


make_sound(Duck())
```

#### 5）`Callable`：给「函数当参数」写清签名

**格式**：`Callable[[第1参, 第2参, ...], 返回值]`。

```python
from collections.abc import Callable


def run(fn: Callable[[], None]) -> None:
    fn()


def hello() -> None:
    print("你好")


run(hello)
```

**参数个数不固定时**，占位用 `...`（表示「任意形参表」，由被传入的函数自己决定）：

```python
def execute(fn: Callable[..., int]) -> int:
    return fn(1, 2, 3)


def sum_three(a: int, b: int, c: int) -> int:
    return a + b + c


execute(sum_three)
```

#### 6）`NoReturn`：标「正常情况不会 `return`」

**典型两种**：一直抛错，或死循环（演示用）。

```python
from typing import NoReturn


def panic() -> NoReturn:
    raise RuntimeError("程序崩溃")


def infinite_loop() -> NoReturn:
    while True:
        pass
```

#### 常见错误提醒（扫一眼）

- 把 **`Iterable`** 当 **`Sequence` 用**：`Iterable` **不保证**能 `items[0]`，需要下标就改 **`Sequence`**（或先 `list(...)` 再索引）。  
- **`Callable`** 方括号里写的是**参数类型列表**，再跟返回值；漏一层方括号会语法错。  
- **`NoReturn`** 与 **`None`** 不同：**`None`** 是「正常返回、值是空」；**`NoReturn`** 是「从类型意义上讲不会正常返回」。  
- **`Protocol`**：运行时**不会**替你拦「没实现方法的类」；它主要给 **静态检查**用（要运行期检查可看 `typing.runtime_checkable`，本节不深展开）。

#### 新手速记（六条）

1. **`TypeAlias`**：长类型变短名。  
2. **`Iterable`**：能遍历；**`Sequence`**：能遍历 + 长度 + 索引。  
3. **`TypeVar`**：输入输出类型绑在一起。  
4. **`Protocol`**：看方法形状，不强迫继承。  
5. **`Callable`**：把回调的「参数 + 返回」写清楚。  
6. **`NoReturn`**：不会正常返回（抛错 / 死循环等）。

## 零、大白话压缩版（与 **§一～§七** 同内容；背 + 写）

### 本节核心一句话

下面六块工具，用来把注解写得**更短、更宽、更准、更可维护**：**别名**、**入参迭代选型**、**`TypeVar` 泛型**、**`Protocol` 静态鸭子**、**`Callable` 回调**、**`NoReturn` 永不返回**。

---

### 一、类型别名（`TypeAlias`）

**给复杂类型起外号**，只影响**注解可读性**，不改变运行逻辑。

```python
from typing import TypeAlias

FromTo: TypeAlias = tuple[str, str]
```

---

### 二、`Iterable` vs `Sequence`（必背）

- **只 `for` 遍历** → **`Iterable[T]`**（更宽）。  
- **要 `len` / `[]` 索引** → **`Sequence[T]`**。  
口诀：**能宽则宽**（与 **§二** 表一致）。

---

### 三、`TypeVar`（泛型）

**把「输入容器元素类型」和「输出」绑在同一字母上**，写通用函数又不丢信息。

```python
from typing import TypeVar

T = TypeVar("T")

def pick(items: list[T]) -> T:
    return items[0]
```

还有 **`bound=`**、**受限 `TypeVar`**、**`AnyStr`** 等，见 **§三**。

---

### 四、`Protocol`（静态鸭子类型）

**不看类名，看形状**：只要实现了约定的方法/属性，静态检查就认（不必 `class C(Animal)` 那种显式继承）。

```python
from typing import Protocol

class Quackable(Protocol):
    def quack(self) -> None: ...
```

新手建议先把 **「零·附」第 4 条里的「动物 / 猫狗」手把手段落** 跑通，再回来看这一屏的嘎嘎叫写法。

与 **8.4**、**第 13 章** 对照读。

---

### 五、`Callable`

**`Callable[[参数…], 返回值]`** 描述回调 / 高阶函数 / 装饰器；参数不确定时常用 **`Callable[..., R]`**（见 **§五**）。

```python
from collections.abc import Callable

def run(fn: Callable[[], None]) -> None:
    fn()
```

---

### 六、`NoReturn`

标注**永远不会正常返回**的路径（**总是抛错**、死循环、`os._exit` 等），帮助检查器做**控制流**推断（见 **§六**）。

```python
from typing import NoReturn

def panic() -> NoReturn:
    raise RuntimeError("stop")
```

---

### 七、六句口诀

1. **别名**：缩短长类型。  
2. **`Iterable` / `Sequence`**：入参**能宽则宽**。  
3. **`TypeVar`**：**进什么元素，出什么同型**（在签名里绑死）。  
4. **`Protocol`**：**静态鸭子**，按方法形状约束。  
5. **`Callable`**：**注解函数值**。  
6. **`NoReturn`**：**永不返回**。

---

### 八、一页浓缩示例（与脚本 **`cheat_sheet_snippets`** 同思路）

```python
from collections.abc import Callable
from typing import NoReturn, Protocol, TypeAlias, TypeVar

Point: TypeAlias = tuple[int, int]

T = TypeVar("T")


def first(items: list[T]) -> T:
    return items[0]


class Drawable(Protocol):
    def draw(self) -> None: ...


def paint(obj: Drawable) -> None:
    obj.draw()
```

## 一、类型别名（Type Alias）

- **作用**：给冗长类型起短名字，**只影响注解可读性**，不引入新运行时类型。
- **3.10+** 推荐显式标注，避免与普通赋值混淆：

```python
from typing import TypeAlias

FromTo: TypeAlias = tuple[str, str]
# 例如：Iterable[FromTo] 比 Iterable[tuple[str, str]] 更清晰
```

---

## 二、`Iterable` vs `Sequence`：入参怎么选

| 类型 | 能力 | 典型入参 |
|------|------|----------|
| `Iterable[T]` | 可迭代（`for`） | 列表、元组、生成器、自定义迭代器 |
| `Sequence[T]` | 有限、`len`、按索引访问 | 列表、元组、字符串等**序列** |

- **只需要遍历**：优先 `Iterable`（最宽）。
- **需要长度或索引**（例如按列排版、分块）：用 `Sequence`。
- **返回值**：尽量写具体类型（如 `list[T]`），少用裸 `Iterable`/`Sequence` 表示“我返回什么”，以免调用方不好用。

---

## 三、`TypeVar`：把“输入类型”和“输出类型”绑在一起

```python
from collections.abc import Sequence
from typing import TypeVar

T = TypeVar("T")

def sample(population: Sequence[T], size: int) -> list[T]:
    ...
```

- 传入 `Sequence[int]`，检查器会期望返回 `list[int]`；同理 `str` 等。

### 受限 `TypeVar`（固定几种）

```python
from decimal import Decimal
from fractions import Fraction
from typing import TypeVar

NumberT = TypeVar("NumberT", float, Decimal, Fraction)
```

### 有界 `TypeVar`（`bound=`）

```python
from collections.abc import Hashable
from typing import TypeVar

HashableT = TypeVar("HashableT", bound=Hashable)
```

### 标准库：`AnyStr`

```python
from typing import AnyStr  # TypeVar('AnyStr', str, bytes)

def join_lines(lines: list[AnyStr], sep: AnyStr) -> AnyStr: ...
```

常用于 **同一套 API 在 `str` 与 `bytes` 上保持类型一致**。

---

## 四、静态协议 `typing.Protocol`

- **含义**：只描述“需要哪些方法/属性”，**不要求**目标类显式继承该协议（结构子类型）。
- 与 ABC：**名义继承** vs **结构匹配**——二者可并存；协议特别适合“只依赖一两个操作”的函数。
- **新手入门**：先看上文 **「零·附」第 4 条（动物 / 猫狗手把手）**，再读本节的进阶例。

示例（可比较）：

```python
from collections.abc import Iterable
from typing import Any, Protocol, TypeVar

class SupportsLessThan(Protocol):
    def __lt__(self, other: Any) -> bool: ...

LT = TypeVar("LT", bound=SupportsLessThan)

def top(series: Iterable[LT], length: int) -> list[LT]:
    ordered = sorted(series, reverse=True)
    return ordered[:length]
```

## 五、`Callable`：注解“可调用对象”

- **形式**：`Callable[[Arg1, Arg2, ...], Return]`。
- **任意参数**：`Callable[..., R]`（参数列表用 `...`）。

```python
from collections.abc import Callable

def repl(prompt: str, input_fn: Callable[[str], str] = input) -> str:
    return input_fn(prompt)
```

适用于回调、高阶函数、装饰器（更复杂的型变在第 15 章等深入）。

---

## 六、`NoReturn`：永不正常返回

用于：**总是抛错**、**无限循环不返回**、`os._exit` 一类路径。只用于**返回值**位置。

```python
from typing import NoReturn

def fatal_error(msg: str) -> NoReturn:
    raise RuntimeError(msg)
```

检查器会假定调用点之后的代码在“能返回”的前提下不可达（配合控制流分析）。

---

## 七、速览表

| 工具 | 作用 |
|------|------|
| `TypeAlias` | 缩短复杂注解 |
| `TypeVar` / `bound` / 受限 | 泛型函数、输入输出类型关联 |
| `Protocol` | 静态鸭子类型、结构约束 |
| `Callable[[...], R]` | 回调与高阶函数 |
| `NoReturn` | 不返回的函数 |
| `Iterable` / `Sequence` | 宽入参：只迭代 vs 要长度/索引 |

---

## 八、实践原则（精简）

1. **别名**：只为真实存在的冗长签名服务，避免无意义缩写。
2. **`TypeVar`**：能用 **`bound=` 描述公共超类型或协议**时，往往比硬编码“几种类型”更稳、更易扩展。
3. **`Protocol`**：在“只依赖少量操作”时减少继承耦合；需要共享实现或生态约定时仍会用 ABC。
4. **`Callable[..., R]`**：回调参数常很杂时用 `...`，返回值尽量具体。
5. **`NoReturn`**：只标**确实不会正常返回**的函数，避免削弱控制流推断。
