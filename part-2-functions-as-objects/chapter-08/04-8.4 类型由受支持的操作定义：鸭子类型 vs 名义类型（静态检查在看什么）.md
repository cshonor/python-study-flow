# 8.4 类型由受支持的操作定义：鸭子类型 vs 名义类型（静态检查在看什么）

## 读完这一节，你只需要记住一个画面

- **Python 运行时**更像：**不管对方身份证上写什么职业，只要能干这个活，就让你干**。这叫**鸭子类型**（好记版：能 `quack` 就当鸭子）。  
- **mypy / Pyright 这类工具**更像：**先看你在纸上登记的职位，再查这个职位「规定该会哪些技能」**。这叫**名义类型**（先看名字 / 继承关系）。

两者不矛盾，只是**看的时机不一样**：一个在你**跑起来**的时候；一个在你**写代码 / 跑 CI** 的时候。

**整章地图**：[00 全套类型提示总览](<00-全套类型提示深度展开（Fluent Python 第8章学习路线）.md>)（你在第 **4** 节）。

**建议先跑脚本**：`04_duck_nominal_typing_demo.py`（里面有 `Bird` / `Duck` / 无注解的 `alert` / 带 `Protocol` 的 `alert_quackable`，打印结果一眼能对上下面文字）。

## 先从「没写类型」的函数看起

```python
def alert(birdie):
    birdie.quack()
```

- **运行时**：只要传进来的对象**真的有** `quack` 这个方法，就成功；没有就 `AttributeError`。  
- **静态检查**：对这种**完全没注解**的函数，在渐进式设置下常常**很宽松**，很多问题要等你运行才暴露。

这就是很多新手的第一印象：**Python 很灵活，类型工具好像也没那么神**——不是工具坏了，是你还没给它「检查用的说明书」（注解）。

---

## 再看「写了类型」之后，工具脑子怎么转

```python
class Bird:
    pass

class Duck(Bird):
    def quack(self) -> None:
        print("Quack!")

def alert_duck(birdie: Duck) -> None:
    birdie.quack()

def alert_bird(birdie: Bird) -> None:
    birdie.quack()
```

- **`alert_duck`**：纸上写「来的是 `Duck`」；`Duck` 身上**有** `quack`，mypy 通常**满意**。  
- **`alert_bird`**：纸上写「来的是 `Bird`」；`Bird` 这个类**没写** `quack`，mypy 通常**会报错**——哪怕你心里知道「这次运行时其实传进来的是 `Duck()`，而 `Duck` 会叫」。

**关键点**：静态检查更信**你在注解里登记的名字**，不会自动替你想「这次碰巧是子类」。

所以才有这句人话总结：

> **运行时看「手里这个对象到底是谁」；静态检查先看「你纸条上写的是谁」。**

## 那我想「又要灵活，又要工具帮看」怎么办？用 `Protocol`

有时你并不在乎类名叫什么，只在乎：**有没有这几个方法**。  
`typing.Protocol` 就是干这个的：**按「形状」检查，不强迫继承某个具体类名**。

```python
from typing import Protocol


class Quackable(Protocol):
    def quack(self) -> None: ...


def alert_quackable(birdie: Quackable) -> None:
    birdie.quack()
```

- **静态**：只要对象**结构上**有 `quack(self) -> None` 这一套，常常就能过检查（脚本里用 `Goose` 演示）。  
- **运行**：还是普通属性查找，和鸭子类型一样走 `birdie.quack()`。

可以把它想成：**给「鸭子」办了一张检查器认得的卡**——不是运行时多了一层魔法。

默认 **`Protocol`** 主要服务静态工具；如果还想在运行时用 `isinstance` 那一套，需要 **`@runtime_checkable`**，那是第 13 章会再展开的细节；这里先知道有这回事就行。

## 一个小例子：为什么「声明成序列」会跟「做乘法」打架

```python
def double(x):
    return x * 2
```

运行时，`x` 可能是 `int`、`str`、`list`……只要乘得动。

但如果你写成：

```python
from collections.abc import Sequence

def double(x: Sequence[int]) -> list[int]:
    return x * 2  # mypy 可能报错：Sequence 并不保证支持这种乘法
```

这里不是在骂你「程序一定跑不起来」，而是在说：**你告诉检查器的故事（Sequence）和它看到的操作（*）对不上号**。

记住方向就够：**声明的类型，要经得起你在函数体里对它做的操作**。

## 一眼对照表（想不起来就回来看）

|  | **鸭子类型（运行时）** | **名义 / 结构（静态工具）** |
|---|------------------------|------------------------------|
| **关心啥** | 对象上**有没有**那个方法 | 注解怎么写；`Protocol` 则看**形状** |
| **啥时候** | 跑到那一行才知道 | 写代码 / CI 时就能报 |

三句顺口溜：

1. **运行时**：默认很「鸭子」，能干活就行。  
2. **写了注解**：工具会按「名义 / 声明」较真。  
3. **`Protocol`**：想按形状约束，又不必搞继承链时的常用桥。
