# 7.5 Python 的 9 种可调用对象（Callable）：谁能用 `()`，以及怎么安全判断

这一节把“可调用对象”系统梳理一遍。你要记住的定义很简单：

> **只要能写成 `obj(...)` 的对象，就是可调用对象（callable）。**

判断方式也很简单：

> 用 `callable(obj)`，返回 `True` 就代表它能被 `()` 调用。

但“能调用”并不代表“调用后立刻得到业务数据”。有些可调用对象（生成器函数 / 协程函数 / 异步生成器函数）**调用后返回的是迭代器/协程对象**，需要进一步消费或 `await` 才会产生结果。

配套脚本：`05_callable_objects_demo.py`（把 9 类都跑出来，并打印 `callable` / `type` / `inspect` 辅助信息）。

## 大白话速通（一听就懂；与下面 **§零** 说的是同一件事）

### 1、什么叫「可调用」？

**大白话**：只要**语法上**允许你在它后面写 **`(...)`**，并且这次写法表示「去执行它的调用」，那它在这里就是**可调用对象**。  
**操作上**：先用 **`callable(obj)`** 粗筛——**`True` 一般表示能写 `obj(...)`**（调了会不会报错、参数对不对，另说）。

### 2、那到底是「怎么调用」的？

就一种**写法上的统一姿势**：**`xxx(...)`**。  
你写成 **`hello()`**、**`list()`**、**`lst.append(1)`** 里最后那段 **`(...)`**，都是在**调用**某个可调用对象（有时要先通过 **`.`** 把方法对象取出来，再对它加括号）。

### 3、举几个例子，一看就通

**① 普通函数**

```python
def hello():
    print("hi")


hello()
```

**② 类（类型对象）也是可调用的：调用就是在「造实例」**

```python
list()
str()
```

**③ 方法：没加括号只是拿到可调用对象，加了括号才是调用**

```python
lst = []
lst.append  # 绑定方法对象，本身往往也是 callable
lst.append(1)
```

**④ 自己写的类**：想让实例也能直接 `obj()`，就实现 `__call__`（9 类里第 6 类，7.6 细讲）。

```python
class A:
    def __call__(self):
        print("我被调用了")


obj = A()
obj()
```

### 4、底层直觉（别记死，记方向）

你写 **`obj()`**，解释器做的是「**走一遍调用协议**」。  
**对实现了实例级 `__call__` 的对象**，你可以直观理解成最终会执行到**与 `__call__` 相关**的那条路径（教学上常说成 **`obj()` ↔ `obj.__call__()`** 好记）。

**一句不抬杠的补充（面试别翻车）**：普通 **`def` 函数**、很多**内置函数/方法**、**类本身 `MyClass()`**，底层实现路径**各自不同**，**不等于**「所有 callable 都是你在类里手写了一个 `__call__`」。**九类表（§零·二）**里第 **6** 类才是「典型靠实例 `__call__`」那一档。

### 5、实例方法、`self`、三种方法（很多人卡在这里）

**先定结论：**

1. **类体里 `def` 出来的方法，对应拿到的函数/方法对象，本身往往是可调用对象**（**`callable(...)` 常为 `True`**）。  
2. **普通实例方法**从类上**裸调** **`Person.say_hi()`** 会报错，通常**不是**因为它「不可调用」，而是 **`say_hi` 还缺第一个位置参数 `self`**——解释器不知道把谁当成「当前实例」。

```python
class Person:
    def say_hi(self) -> None:
        print("你好")


print(callable(Person.say_hi))  # True：未绑定（或底层 descriptor），仍是可调用的“那种东西”
# Person.say_hi()  # TypeError：少了 self 实参

p = Person()
p.say_hi()              # 绑定方法：自动把 p 塞进第一个参数
Person.say_hi(p)      # 等价写法（手动补上 self）
```

**一句话**：实例方法**需要**一个实例来填 **`self`**；**`p.say_hi()`** 就是语法糖，帮你完成了 **`Person.say_hi(p)`**。

**三种方法对照（顺带记死）：**

| 种类 | 第一个参数习惯 | 谁帮你自动塞进去 | 常见调用姿势 |
|:---:|---|---|---|
| **实例方法** | `self` | **实例** `p` | **`p.meth()`**（或 **`C.meth(p)`**） |
| **`@classmethod`** | `cls` | **类** `C` | **`C.meth()`** 或 **`p.meth()`**（都传 `C`） |
| **`@staticmethod`** | 无强制 `self/cls` | **谁都不塞** | **`C.meth()`** / **`p.meth()`** 都行，等价普通函数挂在命名空间里 |

**超好记的一句**：方法对象**常常本来就是 callable**；**实例方法**只是**少个 `self` 实参**时不能直接按「零参数」去 **`()`**，要靠**实例**或**手动把实例当第一个参数传进去**。

### 6、三句极简总结（背）

- **可调用**：能写成 **`xxx(...)`** 去调它（先想 **`callable(x)`**）。  
- **调用方式**：语法上就是 **`(...)`** 这一套。  
- **`callable(obj)`**：查的是**「能不能按这套语法去调」**，**不保证**调一次就拿到你想要的「最终业务结果」（尤其生成器函数 / 协程函数 / 异步生成器函数，见 §零·二表末三行与 **§一**）。

## 零、完整版速查（面试 / 背诵；与 **§一～§三** 互补）

### 零·一、一句话定义

**可调用对象** = 能写成 **`obj(...)`** 的对象。  
**判断**：**`callable(obj)`** 为 **`True`** 表示**语法上**可以拿括号去调（是否该调、调了会不会抛错，还要看参数与运行时状态）。

### 零·二、九种可调用对象（《流畅的 Python》第二版 **7.5** 常用梳理）

**说明**：**`def` 与 `lambda` 同属「用户定义函数」**一类；下表为 **9 类**，第 1 行用 `def` 代表即可。

| 序号 | 种类 | 典型示例 | 调用 `(...)` 之后常见得到什么 |
|:---:|:---:|---|---|
| 1 | 用户定义函数 | `def f(): ...`、`lambda: ...` | 由函数体决定的**普通返回值** |
| 2 | 内置函数 | `len`、`sum`、`print` | 内置实现返回的值 / 行为 |
| 3 | 内置方法 | `{}.get`、`[].append`（绑定到实例后） | 依方法语义 |
| 4 | 方法 | `obj.meth`、`C.meth`（未绑定） | 依方法体 |
| 5 | **类**（类型对象） | `str`、`list`、`MyClass` | **新实例**（构造流程） |
| 6 | 实现了 **`__call__`** 的实例 | `adder(1, 2)` | **`__call__` 返回值** |
| 7 | 生成器函数 | `def g(): yield 1` | **生成器对象**（惰性，需迭代） |
| 8 | 协程函数 | `async def c(): ...` | **协程对象**（需 `await` / 事件循环） |
| 9 | 异步生成器函数 | `async def ag(): yield 1` | **异步生成器**（需 `async for`） |

### 零·三、面试高频点

1. **`callable(list)`、`callable(str)` 为 `True`**：类对象本身可调用，用来**构造实例**。  
2. **有 `__call__` 的实例可当「函数」用**：装饰器、命令对象等常见。  
3. **`callable(x)` 为真 ≠ 调一次就得到最终业务数据**：上表 **7～9** 三类得到**中间对象**，还要 **`next` / `for` / `await` / `async for`**。  
4. **粗筛用 `callable`，细分用 `inspect`**（见脚本输出：`isfunction`、`ismethod`、`isbuiltin`、`isclass` 等）。  
5. 在 CPython 里，**生成器函数 / 协程函数 / 异步生成器函数**的 `type(obj).__name__` **常常都是 `function`**，要靠 **`inspect.isgeneratorfunction`**、**`inspect.iscoroutinefunction`**、**`inspect.isasyncgenfunction`** 才能和普通的 `def` 区分开（**`05_callable_objects_demo.py`** 里可对照打印）。

### 零·四、九类各一例：可复制运行（与脚本分类一致）

下面 **`examples` 恰好 9 个**：`def` 代表用户函数一类；其余每类一个。运行后看 **`callable` 与 `type`**；最后三行演示「**调用后得到中间对象**」。

```python
class Widget:
    def meth(self) -> str:
        return "meth"

    def __call__(self) -> str:
        return "called"


def user_fn() -> None:
    pass


def gen_fn():
    yield 1


async def coro_fn():
    return 42


async def agen_fn():
    yield 1


w = Widget()
examples = [
    ("1 user-defined function", user_fn),
    ("2 builtin function", len),
    ("3 builtin method", {}.get),
    ("4 method", w.meth),
    ("5 class", Widget),
    ("6 __call__ instance", w),
    ("7 generator function", gen_fn),
    ("8 coroutine function", coro_fn),
    ("9 async generator function", agen_fn),
]

for label, o in examples:
    nm = getattr(o, "__name__", type(o).__name__)
    print(f"{label}: name={nm!r} callable={callable(o)!s} type={type(o).__name__}")

g = gen_fn()
print("gen_fn() ->", type(g).__name__)
c = coro_fn()
print("coro_fn() ->", type(c).__name__)
c.close()
```

更完整的 **`inspect` / `asyncio` 消费示例**见 **`05_callable_objects_demo.py`**。

### 零·五、三条结论（背）

1. **能写 `obj(...)` 的就是 callable；先想 `callable(obj)`。**  
2. **九类里最后三类**：调用得到**生成器 / 协程 / 异步生成器**，不是「一次括号就结束」。  
3. **类、带 `__call__` 的实例**也是 callable，别只想到 `def`。

## 一、先把两件事区分开：可调用对象 vs 调用结果

### 1.1 可调用对象（callable）
能写 `obj(...)`。

### 1.2 调用结果（result）
调用之后返回的对象是什么：

- 普通函数/方法/内置函数：通常直接返回业务数据；
- 生成器函数：返回生成器对象（你需要 `next(...)` 或 `list(...)` 才能得到数据）；
- `async def` 协程函数：返回协程对象（要 `await` 或交给事件循环执行）；
- 异步生成器函数：返回异步生成器对象（要 `async for` 消费）。

## 二、9 种可调用对象（Python 3.9+ 常见梳理）

下面的分类是“用法视角”的分类，重点是你写代码时怎么识别。

### 2.1 用户定义函数（`def` / `lambda`）
- 你写出来的普通函数对象。
- 特点：有 `__name__`、可写 docstring、支持闭包、可作为参数/返回值。

### 2.2 内置函数（built-in function）
- 由解释器/标准库 C 扩展提供的函数对象。
- 示例：`len`、`sum`、`print`、`math.sqrt`。

### 2.3 内置方法（built-in method）
- 内置类型的方法（很多也是 C 实现）。
- 示例：`dict.get`（绑定到实例后得到 built-in method）、`list.append`。

### 2.4 方法（method）
你在类体中 `def` 定义的方法，绑定到实例后成为“绑定方法”可调用。
- 例：`obj.method(...)`
- **实例方法**：**`callable(C.meth)` 常为 `True`**，但 **`C.meth()`** 往往 **`TypeError`（缺 `self`）**；应写 **`obj.meth()`** 或 **`C.meth(obj)`**。详见文首 **「大白话 · §5」**。  
- **细分**：实例方法、`@classmethod`、`@staticmethod`（三种都能 `callable`；差别在**第一个参数由谁自动绑定**，见 **大白话 §5** 表）

### 2.5 类（type object）
类本身也可调用：`MyClass(...)`。
- 调用过程大致是：`__new__` 创建实例 → `__init__` 初始化 → 返回实例。

### 2.6 实现了 `__call__` 的实例（user-defined callable instance）
只要类里定义了 `__call__`，它的实例就能像函数一样被 `()` 调用。
- 这是 7.6 要展开的重点（BingoCage）。

### 2.7 生成器函数（generator function）
函数体里包含 `yield` 的函数。**调用它不会执行函数体**，而是返回一个生成器对象（iterator-like）。  
示例：`def gen(): yield 1`

### 2.8 原生协程函数（native coroutine function）
用 `async def` 定义的函数。调用后返回协程对象，需要 `await` 或由事件循环调度。  
示例：`async def coro(): return 1`

### 2.9 异步生成器函数（async generator function）
用 `async def` 定义、并包含 `yield` 的函数。调用后返回异步生成器对象，需要 `async for` 消费。  
示例：`async def agen(): yield 1`

## 三、怎么安全判断“是不是可调用”

### 3.1 `callable(obj)`：最直接、最通用

```python
callable(len)       # True
callable(str)       # True（类可调用）
callable("Ni!")     # False
```

### 3.2 `callable` 为 True，不代表“调用就得到结果”

这三个也都是 callable，但调用结果不是“值”，而是“待消费对象”：

- 生成器函数 → 生成器对象
- 协程函数 → 协程对象（需 await）
- 异步生成器函数 → 异步生成器对象（需 async for）

因此实际写代码时，除了 `callable` 以外，你经常还要配合 `inspect` 做更细的分类（demo 里会展示）。
