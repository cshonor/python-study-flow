## 零、新手直白版（与书第 7 章「一等对象」对照）

《流畅的 Python（第二版）》**第 7 章**要建立的直觉：**函数在 Python 里是一等对象**；**示例 7-1**（函数有属性）、**示例 7-2**（赋值 + `map`）的展开与输出见下文 **§一、§二**。`def` 何时「存在」见 **§1.25**。

### 零·一、一等对象（first-class）常说的是哪四条？

（与书中「函数是一等公民」的讲法一致，措辞因译本略有出入。）

1. **运行时创建**——执行到 **`def`**（或包含 **`lambda`** 的表达式）才生成**函数对象**。  
2. **可以赋值给变量**——给函数起**别名**。  
3. **可以作为参数传给别的函数**——例如传给 **`map`**。  
4. **可以作为函数的返回值**——工厂、闭包等（后文章节展开）。

Python 里用 **`def` 定义的函数**满足这四条，故称**一等对象 / 一等公民**。

### 零·二、可赋值：多个名字，同一个函数对象

```python
def factorial(n):
    return 1 if n < 2 else n * factorial(n - 1)

fact = factorial
print(fact(5))        # 120
print(factorial(5))   # 120
# fact is factorial 通常为 True
```

`fact` 与 `factorial` 指向**同一**函数对象，和 `a = 1; b = a` 后两个名字指向同一个 `int` 是同一类事。详见 **§二·2.1**。

### 零·三、可传给 `map`：函数当参数（高阶函数）

**`map(函数, 可迭代对象)`**：对 iterable 里**每个元素**调用你传入的那个 **callable**；Python 3 里 **`map` 返回迭代器**，常用 **`list(...)`** 看全结果。

```python
def factorial(n):
    return 1 if n < 2 else n * factorial(n - 1)

result = map(factorial, range(6))
print(list(result))  # [1, 1, 2, 6, 24, 120]
```

**高阶函数**：接收「别的函数」当参数的函数；`map` 是典型。等价列表推导：**`[factorial(n) for n in range(6)]`**。详见 **§二·2.2**。

### 零·四、「运行时创建」= 执行到 `def` 才造函数

```python
print("=== 程序开始 ===")

# hello()  # 若在这里调用会 NameError：还没执行到 def，hello 尚未被创建

def hello():
    print("我被创建了！")

hello()
```

- **`def` 是普通可执行语句**，不是「整份源码在编译阶段就把所有函数先建好」。  
- **没执行到** `def` → 这个名字下**还没有**可用的函数对象。  
- **执行到** `def` → **当场**创建 **`function` 实例**并绑定到函数名。

更多反例（先调用后 `def`、`if` 里从不执行 `def`）见 **§1.25**。

### 零·五、一句话背

**Python 函数是一等对象：运行时创建、能赋值、能当参数传给 `map`、能当返回值；`def` 是执行到那一行才造函数的普通语句。**

## 一、示例 7-1：出处、代码、输出与概念

### 1.0 出处与背景

- **来源**：第 7 章「函数是一等对象」，**示例 7-1**（函数作为对象及其属性）。
- **书上要建立的结论**：

> 用 **`def` 绑定的名字**指向运行时的 **`function` 实例**；因此可以像普通对象一样访问 **`__doc__`、`__name__`、`__class__`** 等——这就是「**函数是数据 / 对象**」，也是后面高阶函数、闭包、装饰器的共同地基。

### 1.1 完整可运行代码（示例 7-1）

```python
def factorial(n: int) -> int:
    """returns n!"""
    return 1 if n < 2 else n * factorial(n - 1)

print(factorial.__doc__)    # 文档字符串
print(type(factorial))      # <class 'function'>
print(factorial.__class__)  # 一般与 type(factorial) 相同
print(factorial.__name__)   # 定义时的函数名
```

与脚本 **`demo_example_7_1_function_as_object`** 一致；脚本里还会多打 **`type(factorial) is factorial.__class__`** 以及 **`type(builtins.len)`**（见 **§1.4**）。

### 1.2 典型输出（CPython）

前几行（与书中直接 `print(__doc__)` 一致）：

```text
returns n!
<class 'function'>
<class 'function'>
factorial
```

脚本 **Ex 7-1** 段在同一块还会继续打印（示例）：

```text
True
<class 'builtin_function_or_method'>
```

### 1.25 `def` 写在文件里，为什么说「运行时才创建」？（新手常见困惑）

很多人直觉是：**`def` 提前写在源码里 → 程序一跑，所有函数应该已经都在了。**  
和书上「**执行到 `def` 这一行，才当场造出函数对象**」听起来矛盾，其实不矛盾。

**Python 真实的规则**：

- 模块/脚本也是**从上到下顺序执行**的。
- **`def` 和普通语句一样**：只有**执行流真的跑到这一行**，解释器才**创建**对应的 **`function` 对象**，并把名字（如 `hello`）**绑定**到它上面。
- **没跑到 `def` 之前**：这个名字要么不存在，要么还是别的绑定——**不能当「已经定义好的函数」来用**。

**例子 1：先调用、后 `def` → 直接报错**

```python
print("程序开始")

hello()  # NameError：此时还没执行到下面的 def，hello 尚未被创建

def hello():
    print("hello")
```

**例子 2：`def` 在分支里，分支没进 → 函数永远不会被创建**

```python
flag = False

if flag:
    def hello():
        print("我被创建了")

hello()  # NameError：if 块没执行，def 从未运行，hello 不存在
```

**为什么平时感觉不到**：大家习惯把 `def` 写在**文件靠前**的位置，解释器**很快**就会执行到，所以像「一上来就有函数」；底层仍是：**执行到 `def` 的那一瞬间才创建对象**。

```text
程序执行流：上一行 → 下一行 → … → 遇到 def → 创建 function 实例并绑定名字 → 继续往下
```

**四句话记住**（和上面是一回事，方便背诵）：

1. **`def` 不是**「源码里写好了就自动全部存在」的静态声明。  
2. **`def` 就是一行普通可执行语句**：跑到它才执行，没跑到就不执行。  
3. **执行到 `def` 的那一瞬间**，才**创建**函数对象并把名字绑上去；没执行到这一行，对当前作用域来说这个函数**尚不可用**（或根本未创建）。  
4. 平时把 `def` 放在文件**最上面**，解释器**很快**执行到，所以感觉像「一开始就有」——**感觉**和**底层机制**要分开想。

下面 **§1.3·1** 把同一句话收成「背要点」版。

---

### 1.3 关键概念（背要点）

#### 1. 运行时对象

- **`def` 是运行时执行的语句**：执行到该条 `def` 时创建 **`function` 实例**，并把函数名（如 `factorial`）**绑定**到这个对象上；不是「只占位的静态声明」。
- **本质**：函数与整数、字符串一样，都是**运行时存在的一等对象**（可绑定、可传递，见 **§二** 与开篇 **§零**）。

#### 2. `__doc__`（文档字符串）

- **是什么**：函数体**紧跟在 `def` 行之后**的三引号字符串（若写了的话），说明用途。
- **没有文档时**：`__doc__` 为 **`None`**。
- **怎么看**：交互里用 **`help(函数名)`** 可读整理后的说明。

#### 3. `type()` 与 `__class__`

- **`type(factorial)`**：对象的**类型**。
- **`factorial.__class__`**：对象**所属的类**。
- **用户定义函数（CPython）**：二者通常都是 **`<class 'function'>`**；且常见 **`type(factorial) is factorial.__class__`** 为 **`True`**。

#### 4. 核心结论（必背）

- Python 里，用户 **`def` 出来的函数**是 **`function` 类的实例** → 一等对象。
- **一等对象**在这里记四条就够：**运行时创建、可赋值、可传参、可返回**（开篇 **§零** 有展开）。

#### 5. 终极短句（面试）

**`def` 执行造 `function`，有 `__doc__`/`__name__`/`__class__`，能绑定能传递。**

### 1.4 延伸：内置函数的类型

```python
import builtins

print(type(builtins.len))  # 典型：<class 'builtin_function_or_method'>
```

| 类别 | 常见 `type(...)` |
|------|------------------|
| 用户定义函数（`def` / 多数 `lambda`） | `<class 'function'>` |
| 内置函数（如 `len`） | `<class 'builtin_function_or_method'>`（名称随实现可能略有差异） |

二者都是 **callable**，只是**类型不同**。

### 1.5 选读：CPython 底层是不是「同一种外壳、不同填充」？（`PyFunctionObject`）

> **语言规范**只要求：用户函数在 Python 里是 **`function` 类型**（`types.FunctionType`）。下面说的是 **CPython 的主流实现**，属于**实现细节**，换解释器或未来版本可能不同；写业务代码不必依赖，但有助于消掉「每个函数底层是不是各搞一套结构」的疑惑。

**一句话**：在 **CPython** 里，你用 **`def` 或 `lambda` 得到的普通用户函数**，底层通常都落在**同一种 C 级布局**上——源码里对应 **`PyFunctionObject`** 这一套「模板」；**每个函数一份实例**，里面装的 **`__name__`、`__doc__`、字节码（`__code__`）、参数与默认值、闭包单元……** 等数据**各不相同**，但**槽位/结构一致**。

**大白话比方**：大家都是同一个「人类」骨架（头、躯干、四肢）；**长相、名字、经历**不同。对应过来：**`PyFunctionObject` 像统一骨架**，**每个函数对象**是**填了不同内容**的那一份。

**你心里那句确认**：

- 是不是「**同一种结构类型**，不是每种函数各发明一种新结构体」？——在 CPython 对 **`def` / `lambda` 用户函数**这条线上，**可以这么理解**。

**和内置函数区分**（与 **§1.4** 一致）：

- **`len`、`print`** 这类**内置**，在 CPython 里走 **`PyCFunctionObject` 等另一套路径**，Python 里常见类型名是 **`builtin_function_or_method`**，**不是**上面说的 **`PyFunctionObject`**。

**`lambda` 呢？**

- 仍是 **`function` 类型**；在 CPython 里同样按 **`PyFunctionObject`** 这一路处理（匿名只是 **`__name__` 常为 `'<lambda>'`** 等元数据差异）。

**类里定义的方法呢？（易混点）**

- 挂在**类命名空间**里的那个 **`def`**，通常仍是**用户函数对象**（`function` / CPython 里仍是 **`PyFunctionObject`** 这一路）。
- 从**实例**上访问时，你拿到的常常是 **`bound method`（绑定方法）**：它是**另一层包装**（把**函数**和 **`self`** 绑在一起），`type(obj.meth)` 一般是 **`method`**，**不要**和「类体里那份裸函数对象」混成同一个 Python 对象。

**终极一句（面试吹牛用，记得加「CPython 实现」限定）**：

**CPython 里，`def` / `lambda` 得到的用户函数，底层共用 `PyFunctionObject` 这一套结构；每个函数实例里装的字节码、名字、默认值等不同，内置函数则走别的 C 实现。**

**小结**：7-1 用最少代码说明——**函数在运行时是带元数据的对象**；**§1.5** 补一句 **CPython** 里「统一模板 + 分实例数据」的直觉。

## 二、示例 7-2：别名赋值 + 把函数交给 `map`

### 2.1 赋值：别名指向同一函数对象

```python
fact = factorial
assert fact(5) == factorial(5)  # 120
assert fact is factorial       # 通常为 True：同一对象
```

`fact` 与 `factorial` 是**两个名字**，绑定**同一个** `function` 实例。

### 2.2 `map(函数, 可迭代对象)`：高阶函数

**语义**：对可迭代对象里**每个元素**调用第一个参数（必须是 **callable**），在 Python 3 里返回**迭代器**，常用 **`list(...)`** 一次看全结果。

```python
list(map(factorial, range(6)))
# [1, 1, 2, 6, 24, 120]

list(map(factorial, range(11)))
# 0!..10! 共 11 个数，书中常用这一版
```

这就是**高阶函数**：**把别的函数当参数传进来**（此处把 `factorial` 当「规则」交给 `map`）。

**与列表推导式**：`[factorial(n) for n in range(6)]` 语义接近；日常代码里**推导式往往更直观**，`map` 在「已有一个现成的一元函数」时很干净。

## 三、函数对象上常见属性（查阅表）

| 属性 | 含义 |
|------|------|
| `__name__` | 定义时的名字（字符串） |
| `__doc__` | 文档字符串 |
| `__defaults__` | 默认参数值的元组（从左到右与参数对应） |
| `__kwdefaults__` | 仅限关键字参数的默认值字典，可能为 `None` |
| `__code__` | 代码对象（字节码、变量名等；深入 CPython 才常看） |
| `__annotations__` | 类型注解（含 `'return'` 时指返回值注解） |

这些描述的是**函数对象自身的元数据**，不改变「能不能用 `()` 调用」。

## 四、「可调用对象」（Callable）不止 `def` 一种

凡是可以写成 **`obj(...)`** 的，都是可调用对象，例如：

- **`def` / `lambda`** → 用户函数对象  
- **内置函数** → `len`、`print` 等  
- **类型（类对象）** → `MyClass()` 走构造逻辑  
- **实现了 `__call__` 的实例** → 实例可当「函数」调  
- **方法** → 绑定 / 未绑定方法等（本质仍是 callable）

判断：

```python
callable(factorial)  # True
callable(len)        # True
callable(42)         # False
```

## 五、与后续小节的关系

- **装饰器**：把函数对象交给另一个可调用对象再绑回去。  
- **闭包**：返回的仍是函数对象，但携带外层绑定。  
- **`functools`**：`partial`、`lru_cache` 等建立在「函数可当数据」之上。
