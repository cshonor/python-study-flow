## 一、示例 7-1：出处、代码、输出与概念（对照书与脚本）

### 1.0 出处与背景

- 来源：《流畅的 Python（第二版）》**第 7 章「函数是一等对象」**，**示例 7-1**（函数作为运行时对象及其属性）。
- 核心结论（书上要建立的直觉）：

> 在 Python 里，用 **`def` 定义的名字**绑定到运行时的 **`function` 类实例**；因此可以像对待普通对象一样读 **`__doc__`、`__name__`、`__class__`** 等属性——这与「**函数是数据 / 对象**」的设计一致，也是本章后续高阶函数、闭包、装饰器的共同前提。

### 1.1 完整可运行代码（示例 7-1）

```python
def factorial(n: int) -> int:
    """returns n!"""
    return 1 if n < 2 else n * factorial(n - 1)

# 检查函数对象的属性（与脚本 demo_example_7_1_function_as_object 一致）
print(factorial.__doc__)    # 文档字符串
print(type(factorial))      # 类型：function
print(factorial.__class__)  # 所属类：与用户定义函数一般与 type 结果相同
print(factorial.__name__)   # 定义时的函数名
```

### 1.2 典型输出（CPython，以你本机为准）

```text
returns n!
<class 'function'>
<class 'function'>
factorial
```

说明：`type(factorial) is factorial.__class__` 在 CPython 中对用户定义函数通常为 **`True`**（脚本里会打印这一行作为验证）。

### 1.3 关键概念（背要点）

1. **运行时对象**：`def` 不是「静态占位符」，而是**执行到该条 `def` 时**创建 **`function` 实例**并绑定到名称（如 `factorial`）。
2. **`__doc__`**：文档字符串；未写三引号文档时多为 **`None`**；交互环境可用 **`help(factorial)`** 查看整理后的说明。
3. **`type()` 与 `__class__`**：  
   - `type(factorial)`：对象的类型；  
   - `factorial.__class__`：对象所属类；  
   对用户定义函数，在 CPython 中二者通常都是 **`<class 'function'>`**。
4. **本质**：**用户定义函数是 `function` 类的实例**，因此可以像其他对象一样：**赋值、传参、作返回值、挂属性**（见本章 **§零** 与下文 **§二**）。

### 1.4 延伸：与用户定义函数相对的内置函数

```python
import builtins

print(type(builtins.len))  # 典型为 <class 'builtin_function_or_method'>
```

- **用户定义函数**（`def` / 多数 `lambda`）→ **`function`**  
- **内置函数**（如 `len`）→ **`builtin_function_or_method`**（具体名称随实现略有差异，但语义相同：**也是可调用的对象，类型不同**）

**小结**：示例 7-1 用最少代码证明——**函数在运行时是带元数据的对象**；与 **「一等对象」**（可传递、可绑定）合在一起记最省力。

## 二、示例 7-2：别名赋值 + 把函数交给 `map`

### 2.1 赋值：多一个名字指向同一个函数对象

```python
fact = factorial
assert fact(5) == factorial(5)  # 成立 → 别名与原名同一函数，5! 均为 120
```

`fact` 与 `factorial` 是同一函数对象的**两个引用**（一般 `fact is factorial` 为 `True`）。

### 2.2 `map(factorial, range(11))`：高阶函数

`map` 的语义是：**对可迭代对象里每一个元素，调用你传入的那个函数**，返回一个迭代器（在 Python 3 里需要 `list(...)` 才能一次性看到所有结果）。

```python
list(map(factorial, range(11)))
```

这里：

- 第一个参数必须是 **callable**（可调用对象）；
- `factorial` 作为「数据处理规则」被传进去——这就是**高阶函数**：接收别的函数当参数。

**和列表推导式的关系**：`[factorial(n) for n in range(11)]` 语义接近，Often 在 Python 里推导式更直观；`map` 在「已经有一个现成的一元函数」时很简洁。

## 三、函数对象上常见属性（查阅与元编程时会用到）

| 属性 | 含义 |
|------|------|
| `__name__` | 定义时的名字（字符串） |
| `__doc__` | 文档字符串 |
| `__defaults__` | 默认参数值的元组（仅位置/关键字默认值，与参数从左到右对应） |
| `__kwdefaults__` | 仅限关键字参数的默认值字典，可能为 `None` |
| `__code__` | 代码对象（字节码、变量名、常量表等；深入 CPython 才常看） |
| `__annotations__` | PEP 484 风格注解（参数名 → 类型对象；可能含 `'return'`） |

这些属性**不影响**「函数能不能调用」；它们描述的是**函数对象自身的元数据**。

---

## 四、「可调用对象」（Callable）不止 `def` 一种

凡是可以写成 `obj(...)` 的，都是可调用对象，常见来源包括：

- **`def` / `lambda`**：用户函数对象；
- **内置函数**：如 `len`、`print`（实现细节因解释器而异）；
- **类型对象（类本身）**：`MyClass()` 会调用类的调用逻辑并构造实例；
- **实现了 `__call__` 的实例**：实例像函数一样被「调用」；
- **方法**：绑定方法、未绑定方法等（本质仍是 callable）。

判断「能不能当函数用」可用内置函数 **`callable(obj)`**。

## 五、与后续小节的关系（本章后文会展开）

- **装饰器**：依赖「函数可作为参数/返回值」；
- **闭包**：内层函数仍是函数对象，但携带了外层作用域的绑定；
- **`functools`**：如 `partial`（基于已有函数造新函数）、`lru_cache`（在函数对象上挂缓存）。
