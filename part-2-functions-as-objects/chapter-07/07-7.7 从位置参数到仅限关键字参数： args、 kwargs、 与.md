# 7.7 从位置参数到仅限关键字参数：`*args`、`**kwargs`、`*` 与 `/`

这一节讲的不是“语法花活”，而是 **Python API 设计能力**：你如何用参数规则让调用更安全、更清晰、更不容易被误用。

本节围绕书里的 `tag` 函数（示例 7-9、7-10）讲清楚：

- `*content`：可变位置参数
- `class_`：仅限关键字参数 + 关键字冲突的工程解法
- `**attrs`：可变关键字参数
- `/`：仅限位置参数（Python 3.8+）

配套脚本：`07_tag_and_positional_only_demo.py`（含 **`demo_beginner_walkthrough`**：新手逐行注释 + 打印）。

## 大白话新手版（不讲术语顺序；与 **§零** 说的是同一套规则）

### 1、先认 4 个符号，记死

1. **`/`**：**斜杠左边**的参数，调用时**只能按顺序塞位置值**，不能写 **`名字=值`**。  
2. **裸 `*`**（后面不跟 `args` 这个名字也行）：**星号右边**的参数，调用时**必须写 `名字=值`**。  
3. **`*args`**：多出来的**位置**实参，全收进来，变成一个**元组**（名字不一定叫 `args`，习惯如此）。  
4. **`**kwargs`**：多出来的 **`名字=值`**，全收进来，变成一个**字典**；在定义里**永远排在最后**。

### 2、定义里参数排队顺序（新手先背骨架）

从左到右常见合法骨架是：

1. **只能按位置传的**（写在 **`/`** 的左侧）  
2. **普通参数**（位置、关键字两种写法一般都可以，除非被上面的规则夹住）  
3. **可选：`*args`** 收「多出来的位置」  
4. **`/` 与 `*args` 后面的形参**：对调用者来说往往变成**必须写 `名字=值`** 的区间（**仅限关键字**）  
5. 形参表**最后**写 **可变关键字参数**（习惯写成 ``**kwargs``）：收「多出来的关键字」  

不想用 **`*args`**、又想把后面一刀切成「只能关键字」时，就写个**裸 `*`** 挡在中间（见 **§零·二** 小例）。

### 3、最经典：`tag(name, *content, class_=None, **attrs)` 用人话拆

- **`name`**：第一个名字，比如 **`"p"`**、**`"img"`**。  
- **`*content`**：从第二个位置实参开始，**一串正文**都可以往里丢，函数里收到的是**元组**。  
- **`class_`**：想传 HTML 的 **`class`** 属性，调用时必须写 **`class_=...`**；参数名带下划线是因为 **`class` 是 Python 保留字**，不能直接当形参名。  
- **`attrs`**：其它所有 **`xxx=...`** 关键字，统统进**字典**。

### 4、什么叫「仅限关键字」

只要形参出现在 **裸 `*`** 或 **`*args`** 的**右边**，调用时一般就**不能只丢一个光秃秃的值**，而要写 **`形参名=值`**（具体以解释器报错提示为准）。

### 5、什么叫「仅限位置」（`/`）

```python
def f(a, b, /):
    pass


f(1, 2)        # 可以：按位置
# f(a=1, b=2)  # 不行：/ 左侧禁止关键字实参
```

用处（记方向）：**不想让用户依赖参数名**；以后你改形参名字，**按位置调用的人**不容易被你改挂。

### 6、新手必踩坑：生成器别拿来判断「有没有内容」

```python
attr_pairs = (f"{k}={v}" for k, v in attrs.items())
if attr_pairs:  # 错：生成器对象几乎恒为「真」，判不出里面有没有元素
    ...
```

应判断 **`if attrs:`**，或先 **`list(...)`** 再判断。正文 **§二**、**§零·六** 同一件事。

### 7、六条最简总结

1. **`/` 左边**：只能**按位置**传，不要写 **`名=值`**。  
2. **`*` 右边**：一般要 **`名=值`**。  
3. **`*args`**：多出来的位置进**元组**；**`kwargs`**：多出来的关键字进**字典**，且**在最后**。  
4. **`*args`（或裸 `*`）后面的形参**：调用侧通常就当**仅限关键字**记。  
5. **`class_`**：避开保留字 **`class`** 的常规写法。  
6. **生成器对象**：别当「空不空」的判断依据。

## 零、终极笔记（背诵 + 总览；与 **§一～§五** 对照）

### 零·一、本节核心一句话

**Python 用参数在签名里的位置 + 两个符号 `/` 与 `*`，约束「只能怎么传」。**  
再配合 **`*args` / `**kwargs`** 收「多出来的位置 / 关键字」。

| 符号 / 形态 | 含义（调用侧） |
|-------------|----------------|
| **`/`** | **其左侧**：**仅限位置**——不能写 `name=...` 这种关键字实参 |
| **裸 `*`**（后面没有名字） | **其右侧**：**仅限关键字**——必须写 `kw=...` |
| **`*args`** | 收「多出来的位置实参」→ **`tuple`** |
| **`**kwargs`** | 收「多出来的关键字实参」→ **`dict`** |

### 零·二、合法顺序（背骨架；`*args` 整段可省略）

[官方语法](https://docs.python.org/3/reference/compound_stmts.html#function) 大意如下（**`**kwargs` 必须最后**）：

```python
def func(
    a, b, /,              # 1 仅限位置
    c, d=0,              # 2 位置或关键字（可有默认值）
    *args,               # 3 可变位置（可整块省略）
    e, f=1,              # 4 仅限关键字（出现在 *args 之后）
    **kwargs,            # 5 可变关键字（最后）
) -> None:
    ...
```

**没有 `*args`、只想关键字区**时，用**裸 `*`** 隔开：

```python
def only_kw(a, b, *, c, d) -> None:
    ...

# only_kw(1, 2, 3, 4)     # TypeError：c、d 必须关键字
only_kw(1, 2, c=3, d=4)  # 合法
```

这就是书里 **`tag`** 一类 API 的底层规则。

### 零·三、`tag(name, *content, class_=None, **attrs)` 逐段（示例 7-9）

```python
def tag(name, *content, class_=None, **attrs):
    ...
```

| 段 | 含义 | 调用时注意 |
|----|------|------------|
| **`name`** | 普通形参 | 位置、关键字均可（除非你给前面加 `/`） |
| **`*content`** | 可变位置 | `tag("p", "a", "b")` 里 `"a"`,`"b"` 进 tuple；**它后面的形参默认变成仅限关键字** |
| **`class_`** | 给 HTML `class` 留的槽 | **必须** `class_=...`；下划线是因为 **`class` 是保留字** |
| **`**attrs`** | 其余关键字 → **`dict`** | `tag("img", src="x.png", title="...")` |

**实现细节**：仓库里的可运行版本在 **`07_tag_and_positional_only_demo.py`**（对 `attrs` 做了 `sorted`、换行拼接等，与书上打印格式可能略有差别，**参数形态一致**）。

### 零·四、`/` 与「裸 `*`」最小例

```python
def my_divmod(a, b, /):
    return (a // b, a % b)

my_divmod(10, 3)        # 合法
# my_divmod(a=10, b=3)  # TypeError：/ 左侧禁止关键字实参
```

### 零·五、参数类型速查表（背诵版）

| 语法 | 名字 | 调用特点 |
|------|------|------------|
| `def f(a, /):` | 仅限位置 | 只能 `f(1)`，不能 `f(a=1)` |
| `def f(a):` | 普通 | 位置、关键字均可 |
| `def f(*args):` | 可变位置 | 多出来的位置进 `tuple` |
| `def f(*, a):` 或 `def f(*args, a):` | 仅限关键字 | 必须 `f(a=1)`（在 `*` 或 `*args` 之后） |
| `def f(**kwargs):` | 可变关键字 | 多出来的关键字进 `dict`（总在最后） |

### 零·六、生成器别当「有没有内容」的判断（与 **§二** 同一坑）

```python
attr_pairs = (f'{k}="{v}"' for k, v in attrs.items())
if attr_pairs:  # 错：生成器对象恒为真，即使内部没有元素
    ...
```

应判断 **`attrs`** 是否为空，或先 **`list(...)`** 再判断。详见 **§二**。

### 零·七、三条结论（面试 / 写 API）

1. **`*content` 之后的形参**（如 `class_`）对调用者而言是**仅限关键字**（除非再引入更怪的 `/` 组合，书里 `tag` 不需要）。  
2. **`class_`** 是避开关键字 **`class`** 的常规写法，体内再写进 **`attrs["class"]`**。  
3. **签名里从前往后**：**仅限位置** → **位置或关键字** → **`*args`（可选）** → **仅限关键字** → **`**kwargs`（最后）**。

## 一、`tag`：一个“参数特性大全”的实战例子（示例 7-9）

目标：生成一个或多个 HTML 标签字符串。

函数签名（重点在参数形态）：

```python
def tag(name, *content, class_=None, **attrs):
    ...
```

你可以把它拆成四块理解：

| 形态 | 名字 | 含义 | 调用时怎么传 |
|---|---|---|---|
| 普通参数 | `name` | 标签名，如 `"p"`、`"img"` | 位置或关键字都可（除非你加 `/`） |
| `*content` | 可变位置参数 | 捕获多段内容，形成一个 tuple | `tag("p", "hello", "world")` |
| `class_` | 仅限关键字参数 | 专门处理 `class` 属性（避开关键字冲突） | 必须写 `class_="sidebar"` |
| `**attrs` | 可变关键字参数 | 捕获其余所有关键字参数，形成 dict | `tag("img", src="x.png", title="...")` |

### 1.1 `class_` 为什么要这么写

因为 `class` 是 Python 关键字，不能写成参数名：

- 不能写：`def tag(..., class=None):`（语法错误）
- 常见替代：`class_` / `klass` / `css_class`

然后函数内部把它转成真正的 HTML 属性名：

```python
attrs["class"] = class_
```

---

## 二、一个容易写错的点：生成器不能 “if attr_pairs”

有的人会写：

```python
attr_pairs = (f'{k}="{v}"' for k, v in attrs.items())
if attr_pairs:
    ...
```

这是个坑：`attr_pairs` 是生成器对象，**它本身永远为真**，即使里面没有元素。

正确做法是：

- 要么先构造成 list：`pairs = [...]`
- 要么判断 `attrs` 本身是否为空：`if attrs:`

配套脚本里用的是“先 list 再 join”的写法，逻辑最直观。

## 三、示例 7-10：调用方式为什么这么灵活

你会在 demo 里看到这些调用方式（都能工作）：

- `tag("h1", "hello")`：一个内容
- `tag("p", "hello", "world")`：多个内容（`*content`）
- `tag("p", "hello", class_="sidebar")`：仅限关键字参数
- `tag("img", src="x.png", title="Sunset")`：可变关键字参数
- `tag("img", **{"src": "x.png", "title": "Sunset"})`：`**` 解包

关键直觉：

> 你把参数形态设计好，调用者就能自然地用“位置/关键字/解包”的方式组合调用。

## 四、`/`：仅限位置参数（Python 3.8+）

语法：

```python
def f(a, b, /, c, d):
    ...
```

含义：

- `/` **之前**：只能用位置传参（positional-only）
- `/` **之后**：默认既可位置也可关键字（除非再加 `*`）

### 4.1 为什么要有它

两个常见原因：

1. **兼容内置函数风格**：很多内置函数历史上就不接受关键字实参；现在你也能在 Python 层写出同样的约束。
2. **参数名不想成为 API 契约**：有些参数名只是实现细节，未来可能改名；强制位置能避免用户依赖参数名。

### 4.2 最小例子（demo 里会跑）

```python
def divmod_like(a, b, /):
    return (a // b, a % b)

divmod_like(10, 3)      # 合法
# divmod_like(a=10, b=3)  # TypeError：/ 左侧禁止关键字实参
```

## 五、参数类型速查（把规则一次记清楚）

| 类型 | 语法 | 说明 |
|---|---|---|
| 仅限位置参数 | `def f(a, /):` | 只能 `f(1)`，不能 `f(a=1)` |
| 普通参数 | `def f(a):` | 可位置可关键字 |
| 可变位置参数 | `def f(*args):` | 捕获多余的位置参数为 tuple |
| 仅限关键字参数 | `def f(*, a):` / `def f(*args, a):` | 必须写 `a=...` |
| 可变关键字参数 | `def f(**kwargs):` | 捕获多余关键字参数为 dict |
