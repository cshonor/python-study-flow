# 7.4 匿名函数：`lambda` 表达式

`lambda` 用来在表达式里**临时**创建一个小的函数对象，语法是：

```text
lambda 参数列表: 表达式
```

整个 `lambda ...` 求值的结果就是一个**函数对象**（callable），通常立刻传给 `sorted(..., key=...)`、`map`、`filter` 等，而不是先起名字再调用。

配套脚本：`04_lambda_expressions_demo.py`。

---

## 一、核心限制：主体只能是**一个表达式**

- **不能**在 `lambda` 里写多条语句：没有 `;` 分隔的多语句块，也不能写 `while`、`for`、`try`/`except`、`with` 等**语句**（statement）。
- **不能**写普通赋值；Python 3.8+ 虽可在表达式里用海象运算符 `:=`，但塞进 `lambda` 往往更晦涩，**不推荐**作为“技巧”使用。
- “返回值”就是**那个表达式的值**，不需要也**不能**写 `return`。

如果你发现逻辑写不进一个清晰的表达式，说明该用 **`def`**。

---

## 二、示例 7-7：作为 `sorted` 的 `key`（最常见、最合理）

```python
fruits = [
    "strawberry", "fig", "apple", "cherry", "raspberry", "banana",
]
sorted(fruits, key=lambda word: word[::-1])
```

这与下面写法等价，但省去了一次顶层 `def`：

```python
def reverse(word: str) -> str:
    return word[::-1]

sorted(fruits, key=reverse)
```

何时用哪种：

- **只在这一次排序里用**：`lambda` 往往更短；
- **同一规则要在多处复用、要加类型注解/文档**：用 `def` 更清晰。

---

## 三、适合用 `lambda` 的场景（简短）

- 作为高阶函数的参数：`sorted`、`map`、`filter`、`min`/`max` 的 `key` 等；
- 逻辑真的是**一行表达式**就能说清。

## 四、不推荐的做法（反模式）

**不要把 `lambda` 绑定给变量**，例如：

```python
add = lambda x, y: x + y  # anti-pattern
```

应写成：

```python
def add(x: int, y: int) -> int:
    return x + y
```

原因：`def` 有名字、可挂文档字符串、栈回溯更清晰；用 `lambda` 冒充命名函数没有收益。

---

## 五、可读性：从 `lambda` 迁回 `def`（Fredrik Lundh 的四步）

当一段 `lambda` 让人读不懂时，可以按下面顺序重构（思想是：**先想清楚“这段逻辑叫什么”**）：

1. 用注释写下 `lambda` 在做什么；
2. 根据注释起一个**动词性**函数名；
3. 改成 `def`，把表达式挪到 `return`（或多行语句若需要）；
4. 删掉多余注释，让函数名承载含义。

---

## 六、`lambda` 与 `def` 对照（心智模型）

| 维度 | `lambda` | `def` |
|------|----------|-------|
| 名字 | 匿名（显示名为 `<lambda>`） | 显式函数名 |
| 函数体 | 单个表达式 | 任意合法语句块 |
| 文档 / 注解 | 不适用常规 docstring 写法 | 可写 docstring、注解，便于维护 |
| 典型用途 | 一次性、极短的高阶函数参数 | 复用逻辑、复杂逻辑、对外 API |
