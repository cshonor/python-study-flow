# 7.3 高阶函数（Higher-Order Function）

**高阶函数**的定义很窄：满足下面任一条件的函数即可称为高阶函数：

1. **接收至少一个函数作为参数**（例如 `sorted(..., key=f)`、`map(f, it)`）；
2. **返回值为函数**（例如工厂函数返回内层函数、装饰器返回包装函数）。

本节只展开第 1 类里最常用的内置用法，以及 Python 3 里更常见的**写法替代**（推导式、`sum` 等）。

配套脚本：`03_higher_order_functions_demo.py`。

## 零、总纲（整节 7.3 先记这一句）

**高阶函数 = 能把「函数」当参数传进去（或返回函数）的函数。**  
之所以能这么写，是因为第 7 章地基：**函数是一等对象**——可以绑定名字、**可以当参数**、可以当返回值、可以进容器。

### 零·一、两条判定（背这个就够）

满足 **任意一条** 即高阶：

| # | 条件 | 本章重点 |
|---|------|----------|
| 1 | **接收函数当参数** | **`sorted(..., key=)`、`map`、`filter`、`functools.reduce`** 等 |
| 2 | **返回函数** | 闭包、装饰器（后文）；本节主要讲 **第 1 类** |

### 零·二、四个最常碰的内置（大白话 + 最小例子）

| 名字 | 人话 | 最小例子（Python 3 里 `map`/`filter` 要先 `list(...)` 才一次看全） |
|------|------|------------------------------------------------------------------|
| **`sorted(..., key=函数)`** | 按你给的函数算「排序键」，再按键排 | `sorted(["apple", "fig", "banana"], key=len)` |
| **`map(函数, 可迭代对象)`** | 把函数**挨个**用到每个元素上 | `list(map(lambda x: x * x, [1, 2, 3]))` → `[1, 4, 9]` |
| **`filter(函数, 可迭代对象)`** | 只保留「函数返回真」的元素 | `list(filter(lambda x: x > 0, [-1, 0, 2]))` → `[2]` |
| **`functools.reduce(函数, 可迭代对象[, 初始])`** | 两两归并成**一个**结果 | `from functools import reduce; from operator import add; reduce(add, [1,2,3])` → `6` |

**为什么要把函数传给 `sorted` / `map`？**  
因为 **函数是对象**：`len`、`square`、lambda 都是**值**，可以像传 `int`、传列表一样传给别的函数——**能接收函数的函数**就是高阶函数；这也是第 7 章主线。

### 零·三、`map()` 不是字典（再钉一次）

- **`map(...)`**：内置**高阶函数**，本章讲的这个。  
- **`{"k": v}` / `dict`**：键值容器；Java/C++ 里口语常叫 **Map**，和 Python 的 **`map()` 毫无关系**。

详见 `02-示例 7-1 7-2：函数是对象、可赋值、可传给 map（高阶函数）.md` **§2.3**。

### 零·四、一句收束（复习用）

**第 7 章在搭台：函数是对象，可以当参数传给别的函数；`sorted`、`map`、`filter`、`reduce` 都是典型高阶用法。**

## 一、`sorted` + `key`：最常用的高阶场景（示例 7-3、7-4）

`sorted(iterable, key=func)` 里的 `key` 接收**单参数函数**：对每个元素先算一个“排序键”，再按键排序；**不改变原列表**，返回新列表。

### 1.1 示例 7-3：按长度排序

```python
fruits = [
    "strawberry", "fig", "apple", "cherry", "raspberry", "banana",
]
sorted(fruits, key=len)
```

这里 `len` 本身就是函数对象，传给 `key`。

### 1.2 示例 7-4：按“反向拼写”排序

```python
def reverse(word: str) -> str:
    return word[::-1]

sorted(fruits, key=reverse)
```

排序比较的是 `reverse(word)` 的字符串顺序，**不是**把单词真的反过来显示在结果里；列表元素仍是原来的单词。

## 二、`map` / `filter` 与推导式（示例 7-5）

**提醒**：此处的 **`map` 是内置函数**；与 **`dict` / 别语言 Map** 无关，见 **§零·三** 与 `02` **§2.3**。

在 Python 3 中，`map(f, it)`、`filter(pred, it)` 返回的是**惰性迭代器**（遍历到才计算），需要 `list(...)` 才能一次看到全部结果。

很多团队风格指南会建议：

- **逻辑简单、一眼能读完**：优先 **列表推导式** 或 **生成器表达式**；
- **已经有一个现成的、名字清晰的函数**：`map(f, it)` 仍然很合适（不必为教条而改写）。

典型对照：

| 风格 | 代码 |
|------|------|
| `map` | `list(map(factorial, range(6)))` |
| 列表推导 | `[factorial(n) for n in range(6)]` |
| `filter` + `map` | `list(map(factorial, filter(lambda n: n % 2, range(6))))` |
| 推导 + 条件 | `[factorial(n) for n in range(6) if n % 2]`（奇数 `n`） |

要点：`filter(lambda n: n % 2, ...)` 保留**奇数**（`n % 2` 为真）；与 `[n for n in ... if n % 2]` 同义。

## 三、`reduce` 与 `sum`（示例 7-6）

- Python 3 把 **`reduce` 从内置挪到了 `functools.reduce`**，用意是：**不要默认到处归约**，显式 `import` 再使用。
- **求和**这一类场景，优先用内置 **`sum(iterable)`**，不必写 `reduce(add, ...)`。

```python
from functools import reduce
from operator import add

reduce(add, range(100))  # 0+1+...+99
sum(range(100))          # 同上，更直接
```

对更复杂的归约（自定义二元函数、非加法语义），再考虑 `reduce`。

## 四、`all` / `any`：短路“逻辑归约”

- **`all(it)`**：所有元素为真则 `True`；**空可迭代对象**为 `True`（全称量化在空集上常为真）。
- **`any(it)`**：任一为真则 `True`；**空可迭代对象**为 `False`。

它们常用于快速检查条件，而不必手写 `for` 循环。

Python 2 曾有 `apply(fn, args, kwargs)`。Python 3 中请使用：
