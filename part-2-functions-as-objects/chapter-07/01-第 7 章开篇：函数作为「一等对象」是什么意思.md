# 第 7 章开篇：函数作为「一等对象」是什么意思

在 Python 里，**函数不是某种特殊语法魔法**，而是和普通值一样：**运行时在堆里存在的对象**，有类型、有身份、可以绑定到变量名上。

本章标题里的「一等对象」（first-class object / first-class citizen）来自程序语言研究里的习惯说法：某种实体如果在语言里**能像整数、字符串那样被同等对待**，就称它是一等的。

## 零、极简背诵版（考试 / 面试直接用）

约 **1 分钟**可过一遍；细节与扩展见下文 **§一～§六**。

### 0.1 什么是「一等对象」（4 条必背）

满足这 4 条，常被称为**一等对象**（first-class object / first-class citizen）：

1. **运行时创建**（程序跑着还能新建）
2. **能赋值给变量**（或放进容器）
3. **能作为参数传给函数**
4. **能作为函数的返回值**

**Python 里用户定义的函数对象满足以上四条 → 函数是一等对象。**

### 0.2 一句话总结

**函数在 Python 里就是普通对象，可以像数字、字符串一样绑定名字、传来传去。**

### 0.3 为什么重要？（支撑哪些特性）

因为函数是一等对象，下面这类写法才有根基：

- **高阶函数**（如 `sorted(fruits, key=len)`）
- **`lambda`**（小的函数对象）
- **闭包**（内层函数捕获外层名字，仍可当作返回值）
- **装饰器**（把函数对象交给另一个可调用对象再绑回去）
- **回调**（事件 / 异步里稍后调用）

### 0.4 关键误区（必背）

- **「一等函数 / 函数是一等对象」≠「高阶函数」**  
  - **一等**：强调**地位**——函数和数据一样，可创建、可绑定、可传参、可返回。  
  - **高阶**：强调**类型签名**——函数的**参数或返回值**里出现函数。  
- **Python 不是纯函数式语言**；只是**吸收了函数式里常用的一部分**（与 **§二** Guido 的表述一致）。

### 0.5 最小代码（四条各演示一角）

```python
# 1. 运行时创建（def / lambda 都会在运行时产生函数对象）
def f():
    return 1

# 2. 赋值给变量（名字绑定到同一函数对象）
g = f
assert g() == f()

# 3. 作为参数（高阶函数）
def call_func(func):
    return func()

assert call_func(f) == 1

# 4. 作为返回值
def return_func():
    return f

assert return_func() is f
```

完整演示见 **`01_first_class_functions_demo.py`**。

**能创建、能赋值、能传参、能返回 → 一等对象。**

## 一、「一等对象」通常说的四个条件

一种实体若满足下面四条，常被称作一等对象（不同书里措辞略有出入，但意思一致）：

1. **在运行时创建**：程序跑起来之后还能造出新的（函数可以 `def`、也可以运行时通过 `exec`/`types`/`lambda` 等方式构造；日常最常用的是 `def` 与 `lambda`）。
2. **能赋值给变量或放进数据结构**：可以把函数对象绑定给名字，或放进 `list`/`dict`/`tuple` 等容器。
3. **能作为参数传给其他函数**：这就是**高阶函数**（higher-order function）的基础：接收「函数」当入参。
4. **能作为函数的返回值**：闭包、工厂函数、装饰器都依赖这一点。

在 Python 中，**所有用户定义的函数对象**都满足这四条（内置函数、方法对象也大多可按同样方式处理）。所以人们常说「Python 里函数是一等对象」——有时也口语化成「一等函数」，但要注意：**不是说「只有一部分函数一等」**，而是**函数这种对象类型**在语言里被一视同仁。

---

## 二、Guido 的表述：有一等函数，不等于「Python 是函数式语言」

书中引用 Guido van Rossum 的观点，大意是：**虽然函数是一等对象，但不要把 Python 当成纯函数式语言**。

这很重要，因为它帮你选对工具：

- Python **吸收了**函数式编程里很有用的一部分：高阶函数、`map`/`filter`、推导式、`functools` 等。
- 但 Python **默认风格**仍是命令式 + 面向对象为主：可变状态、`for` 循环、类与方法是日常代码的主流。

所以：**你会写 `sorted(..., key=...)`，不等于你要用 Haskell 的方式写一切。**

---

## 三、为什么说「一等函数」容易误解

口语里「一等函数」听起来像「某些函数比较高级」。更准确的说法是：

> **函数值（callable）在语言里是一等公民：可以像数据一样传来传去。**

同理，**方法**、**内置函数**、以及实现了 `__call__` 的实例，也常常可以当作「可调用对象」参与同样的模式。

---

## 四、这些特性直接支撑哪些机制

当你接受「函数是对象」，下面这些就不再神秘：

- **高阶函数**：`sorted(fruits, key=len)` 里的 `len` 是函数对象；`key` 参数接收的是 callable。
- **`lambda`**：创建一个小的函数对象（表达式），常用于一次性回调。
- **闭包**：内层函数捕获外层变量，返回的仍是函数对象。
- **装饰器**：`@decorator` 本质是「把函数对象交给另一个可调用对象处理后再绑定回去」。
- **回调**：GUI、异步、事件系统里把函数注册进去稍后调用。
- **`__call__`**：让实例表现得像函数一样可调用。

## 五、最小示例：把四个条件各看一眼

下面每一段都对应「一等」的一个条件（完整可运行版本见 `01_first_class_functions_demo.py`）。

### 5.1 运行时存在：`def` 定义的就是一个函数对象

```python
def factorial(n: int) -> int:
    return 1 if n < 2 else n * factorial(n - 1)
```

`factorial` 这个名字绑定到一个函数对象上；你可以用 `type(factorial)`、`id(factorial)` 把它当普通对象观察。

### 5.2 赋值与容器：函数对象可以复制引用、可以进列表

```python
f = factorial
assert f(5) == factorial(5)  # 成立 → f 与 factorial 绑定同一函数，调用结果相同

registry = [len, str.lower]
```

### 5.3 作为参数：高阶函数

```python
fruits = ["strawberry", "fig", "apple", "cherry"]
sorted(fruits, key=len)
```

### 5.4 作为返回值：工厂返回函数

```python
def make_pow(n: int):
    return lambda x: x**n

square = make_pow(2)
assert square(3) == 9  # 成立 → 工厂返回的平方函数：3**2==9
```
