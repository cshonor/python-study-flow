# 序列的 `+` / `*`：新对象 vs 引用共享（以及二维列表初始化陷阱）（§2.8）

`+` 和 `*` 看起来是“拼接/重复”，但在可变对象参与时，它们最容易引发一个经典大坑：**引用共享**。

这篇你只需要牢牢记住一句话：`+` / `*` 通常会创建**新的外层容器**，但**不会自动深拷贝里面的元素**。当里面的元素是可变对象（例如子列表）时，你就可能“一改全改”。

---

## 一、核心规则：`+` 与 `*` 产生新对象（不修改原操作数）

- **`+` 拼接**：返回新序列
- **`*` 重复**：返回新序列

```python
assert [1, 2] + [3, 4] == [1, 2, 3, 4]  # 成立 → + 拼接为新 list
assert "ab" * 3 == "ababab"  # 成立 → str 重复拼接
```

注意：这里说的“新对象”是指**外层容器**是新的；内部元素是否共享引用，取决于你重复的是什么（下一节是关键）。

---

## 二、嵌套列表陷阱：`[obj] * n` 只是复制引用（浅拷贝）

当 `obj` 是可变对象（如 `list/dict/set`），`[obj] * n` 不会创建 `n` 个独立副本，而是创建 `n` 个**指向同一对象**的引用。

```python
row = []
board = [row] * 3

board[0].append("X")
assert board == [["X"], ["X"], ["X"]]  # 成立 → 三行共享同一 list，一改全改
```

### 为什么会这样？

你可以用 `is` 验证“是否同一个对象”：

```python
board = [[]] * 3
assert board[0] is board[1] is board[2]  # 成立 → 三槽同一子 list 对象
```

### 2.1 井字棋经典对照：`[['_'] * 3] * 3` vs 列表推导式

| 写法 | 代码 | 本质 |
| :--- | :--- | :--- |
| ✅ | `board = [["_"] * 3 for _ in range(3)]` | 循环每次**新建一行**列表，三行是三个不同对象 |
| ❌ | `weird = [["_"] * 3] * 3` | `["_"] * 3` 只算出一行，外层的 `* 3` 把**同一行引用**贴了 3 次 |

错误写法的效果与「外层 `[row] * 3`」相同：改一格等于改同一行对象，三行一起变。

```python
# 错误：改 [1][2] 时三行同步（因为三行是同一个 list）
weird_board = [["_"] * 3] * 3
weird_board[1][2] = "O"
# [['_', '_', 'O'], ['_', '_', 'O'], ['_', '_', 'O']]

# 正确：只改目标格
board = [["_"] * 3 for _ in range(3)]
board[1][2] = "X"
# [['_', '_', '_'], ['_', '_', 'X'], ['_', '_', '_']]
```

等价于错误写法的循环（同样是 3 次追加**同一个** `row`）：

```python
row = ["_"] * 3
bad = []
for _ in range(3):
    bad.append(row)  # ❌ 不要这样初始化“多行”
```

若已有一行模板、又要独立副本，可对**行**做拷贝（仅一层嵌套时够用）：

```python
board = [row.copy() for _ in range(3)]  # 或 [list(row) for ...]
```

更深层的嵌套若需完全独立对象，再用 `copy.deepcopy`（见标准库 `copy`）。

### 2.2 单层 `['_'] * 3` 为何没问题？

这里重复的是**不可变**字符串 `'_'`，得到 3 个槽位各放一个 `str`；问题出在**外层**对**可变子列表**做 `* n` 时只复制引用。若子元素本身是 `list`/`dict`，仍按 §二 的规则处理。

---

## 三、正确做法：用列表推导式创建独立子列表

以 3×3 井字棋盘为例：

```python
board = [["_"] * 3 for _ in range(3)]
board[1][2] = "X"
assert board == [["_", "_", "_"], ["_", "_", "X"], ["_", "_", "_"]]  # 成立 → 每行独立，只改一格
```

对比：

| 写法 | 结果是否独立 | 适用场景 |
| :--- | :--- | :--- |
| `[[]] * 3` / `[['_']*3] * 3` | ❌ 不独立（共享引用） | 几乎不该用在可变子元素 |
| `[["_"] * 3 for _ in range(3)]` | ✅ 独立 | 初始化二维表/矩阵最常用写法 |

多维嵌套可继续套推导式，例如 3×3×3：

```python
cube = [[["_" for _ in range(3)] for __ in range(3)] for ___ in range(3)]
```

---

## 四、避坑清单（工程口径）

- **元素不可变**（数字、字符串、元组等）：`[x] * n` 通常安全（重复的是不可变对象引用）。
- **元素可变**（列表、字典、集合等）：不要用 `[*] * n` 初始化嵌套结构，改用 listcomp。
- 不要只看打印出来像 `[..., ..., ...]` 就以为是 3 份数据；用 `is` 或 `id()` 验证引用关系。
- **多层嵌套**且要完整复制时，浅拷贝不够再用 `copy.deepcopy`。
