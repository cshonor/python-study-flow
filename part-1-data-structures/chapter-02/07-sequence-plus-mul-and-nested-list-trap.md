# 序列的 `+` / `*` 运算与嵌套列表陷阱（Fluent Python 2.8）

> **本篇定位**：掌握序列的拼接（`+`）与重复（`*`）语义，以及 `[*] * n` 在**可变元素**场景下的引用共享陷阱；学会用 listcomp 正确初始化二维列表（井字棋盘）。  
> **相关**：切片与就地修改见 `06-slicing.md`；推导式与生成器见 `03-listcomps-and-genexps.md`。

---

## 一、核心规则：`+` 与 `*` 产生新对象（不修改原操作数）

- **`+` 拼接**：返回新序列
- **`*` 重复**：返回新序列

```python
assert [1, 2] + [3, 4] == [1, 2, 3, 4]
assert "ab" * 3 == "ababab"
```

注意：这里说的“新对象”是指**外层容器**是新的；内部元素是否共享引用，取决于你重复的是什么（下一节是关键）。

---

## 二、嵌套列表陷阱：`[obj] * n` 只是复制引用（浅拷贝）

当 `obj` 是可变对象（如 `list/dict/set`），`[obj] * n` 不会创建 `n` 个独立副本，而是创建 `n` 个**指向同一对象**的引用。

```python
row = []
board = [row] * 3

board[0].append("X")
assert board == [["X"], ["X"], ["X"]]  # 一改全改
```

### 为什么会这样？

你可以用 `is` 验证“是否同一个对象”：

```python
board = [[]] * 3
assert board[0] is board[1] is board[2]
```

---

## 三、正确做法：用列表推导式创建独立子列表

以 3×3 井字棋盘为例：

```python
board = [["_"] * 3 for _ in range(3)]
board[1][2] = "X"
assert board == [["_", "_", "_"], ["_", "_", "X"], ["_", "_", "_"]]
```

对比：

| 写法 | 结果是否独立 | 适用场景 |
| :--- | :--- | :--- |
| `[[]] * 3` | ❌ 不独立（共享引用） | 几乎不该用在可变子元素 |
| `[["_"] * 3 for _ in range(3)]` | ✅ 独立 | 初始化二维表/矩阵最常用写法 |

---

## 四、避坑清单（工程口径）

- **元素不可变**（数字、字符串、元组等）：`[x] * n` 通常安全（重复的是不可变对象引用）。
- **元素可变**（列表、字典、集合等）：不要用 `[*] * n` 初始化嵌套结构，改用 listcomp。
- 不要只看打印出来像 `[..., ..., ...]` 就以为是 3 份数据；用 `is` 或 `id()` 验证引用关系。
