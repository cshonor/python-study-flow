## 一、本章主题：Python 的序列类型体系

本章核心是把 Python 里的 **「序列」（sequence）** 放在一起理解，包括：

- 最常见的 **列表（`list`）**、**元组（`tuple`）**
- **字符串（`str`）**、**字节序列（`bytes` / `bytearray`）**
- 更偏场景的 **`array.array`**、**`collections.deque`** 等

Python 继承自 ABC 等语言「**序列统一处理**」的思路：只要是序列形态，往往在**迭代、切片、排序、拼接**等操作上有共通体验——这也是日常写代码时觉得「顺手」的来源之一。

## 二、本章核心骨架（《流畅的 Python》第 2 章：一页纸串联）

下面按**书里的讲法**把知识点压成一条线，便于和 **CPython 对象头 / `PyObject*`**（见 `02`、`04`）对照。细节与交叉表以 **`03-sequence-flat-container-mutable-immutable-abc.md`** 为准。

### 1. 对象底层：为什么 `array.array('d')` 往往比「装一堆 `float` 的 `tuple`」更省？

任意 Python 对象在 CPython 里都有**对象头**；以单独的 `float` 为例：

```c
typedef struct {
    PyObject_HEAD;   /* 引用计数、类型指针 ob_type 等 */
    double ob_fval;
} PyFloatObject;
```

- **`array.array('d', [9.46, 2.08, 4.29])`**：载荷区紧排 **`double`**，一般**没有**「每个元素一个完整 `PyFloatObject`」的开销。
- **`(9.46, 2.08, 4.29)`**：每个元素常为**独立的 `PyFloatObject`**（头 + 值），总内存往往大得多。

### 2. 序列的两套分类（可交叉）

**按可变性**

|  | 代表 | 要点 |
| :--- | :--- | :--- |
| **可变** | `list`、`bytearray`、`array.array`、`collections.deque` | 可原地改元素或增删（视类型） |
| **不可变** | `tuple`、`str`、`bytes` | 不能改已有元素/不能增删槽位（`bytes` 不可变；`bytearray` 可变） |

**按存储（容器 vs 扁平）**

|  | 代表 | 底层直觉 | 要点 |
| :--- | :--- | :--- | :--- |
| **容器序列** | `list`、`tuple`、`deque` | `PyObject*` 槽位 | 可异构、可嵌套 |
| **扁平序列** | `str`、`bytes`、`bytearray`、`array.array` | 连续缓冲中的**同构原始数据** | 紧凑；**外层仍有序列对象头**，「扁」的是载荷区（见 `02`、`03`） |

### 3. `collections.abc`：统一接口

- **`Collection`**：`__contains__`、`__len__`、`__iter__`
- **`Sequence`**：序列语义（如 `__getitem__`），以及 `index`、`count` 等混入实现
- **`Reversible`**：`__reversed__`
- **`MutableSequence`**：在 `Sequence` 上增加 `append`、`pop`、`extend`、`insert` 等

内置类型多为 ABC 的**虚拟子类**，故可写：

```python
from collections import abc

issubclass(tuple, abc.Sequence)           # True
issubclass(list, abc.MutableSequence)    # True
```

### 4. 设计思想（三条）

1. **容器序列**：为动态类型与异构服务，用**指针槽位**换通用性。  
2. **扁平序列**：为同构大批量数据服务，用**连续原始缓冲**换密度与速度。  
3. **可变 / 不可变**：在**一致性、可哈希、共享安全**与**原地修改效率**之间分工。

### 5. 与 `PyObject*` 串起来（复习钩）

- **容器序列**每个槽位是一个 **`PyObject*`**，指向独立对象（如 `PyFloatObject`、`PyListObject`）。  
- **扁平序列**的载荷是**原始编码**，不是「每格一个任意对象的指针」；与「完全无对象头」不是一回事。  
- **ABC** 为不同实现提供**同一套协议**，便于 `isinstance` 与鸭子类型并存。

## 三、本章重点内容预告（按书内脉络）

1. **列表推导式与生成器表达式**
   - 用紧凑语法从可迭代对象构造列表
   - 与生成器表达式在**内存占用、惰性求值**上的差异

2. **元组的双重身份**
   - 当作**不可变列表**（与 `list` 对照）
   - 当作**记录 / 轻量结构**（坐标、行数据等，`namedtuple` 与之相关）

3. **序列拆包与模式**
   - `a, b, *rest = seq` 等写法
   - 用 `*` 处理**可变长度**，让接口更 Pythonic

## 四、为什么这一章很重要？（结合进阶与量化场景）

- **数据处理**：价格序列、指标序列常用 `list` / `tuple`；理解 **`list` vs `array`** 等差异，有助于在批量计算里控制内存与速度。
- **批量转换**：列表推导、生成器表达式是**清洗 / 变换**时的主力写法，能明显减少样板循环。
- **轻量结构化数据**：元组作「记录」可在小脚本里**少写类**，又与第 1 章 `namedtuple` 思路衔接。

下面顺序**由易到难、由常用到细节**，可按此推进，减少绕路：

| 优先级 | 内容 | 理由 |
|--------|------|------|
| 1 | 列表推导式（基本形式 + 条件过滤） | 日常最高频，立刻能简化循环 |
