# 第 2 章「丰富的序列」开篇：主题与学习路线

> 《流畅的 Python》第 2 章：**系统认识 Python 的序列类型**，以及列表推导、生成器表达式、元组的双重角色、拆包等。本文是**开篇拆解 + 学习优先级**，便于你高效推进。

---

## 一、本章主题：Python 的序列类型体系

本章核心是把 Python 里的 **「序列」（sequence）** 放在一起理解，包括：

- 最常见的 **列表（`list`）**、**元组（`tuple`）**
- **字符串（`str`）**、**字节序列（`bytes` / `bytearray`）**
- 更偏场景的 **`array.array`**、**`collections.deque`** 等

Python 继承自 ABC 等语言「**序列统一处理**」的思路：只要是序列形态，往往在**迭代、切片、排序、拼接**等操作上有共通体验——这也是日常写代码时觉得「顺手」的来源之一。

---

## 二、本章重点内容预告（按书内脉络）

1. **列表推导式与生成器表达式**
   - 用紧凑语法从可迭代对象构造列表
   - 与生成器表达式在**内存占用、惰性求值**上的差异

2. **元组的双重身份**
   - 当作**不可变列表**（与 `list` 对照）
   - 当作**记录 / 轻量结构**（坐标、行数据等，`namedtuple` 与之相关）

3. **序列拆包与模式**
   - `a, b, *rest = seq` 等写法
   - 用 `*` 处理**可变长度**，让接口更 Pythonic

---

## 三、为什么这一章很重要？（结合进阶与量化场景）

- **数据处理**：价格序列、指标序列常用 `list` / `tuple`；理解 **`list` vs `array`** 等差异，有助于在批量计算里控制内存与速度。
- **批量转换**：列表推导、生成器表达式是**清洗 / 变换**时的主力写法，能明显减少样板循环。
- **轻量结构化数据**：元组作「记录」可在小脚本里**少写类**，又与第 1 章 `namedtuple` 思路衔接。

## 五、学习优先级建议（高效推进）

下面顺序**由易到难、由常用到细节**，可按此推进，减少绕路：

| 优先级 | 内容 | 理由 |
|--------|------|------|
| 1 | 列表推导式（基本形式 + 条件过滤） | 日常最高频，立刻能简化循环 |
| 2 | 生成器表达式（与推导式对照） | 搞清**惰性**与内存，后面迭代器章节更顺 |
| 3 | 元组：不可变列表 + 拆包 | 与 `list` 对照，拆包贯穿全书 |
| 4 | `*` 与扩展拆包、`deque` / `array` 等 | 在已熟悉推导与元组后再上专项容器 |
