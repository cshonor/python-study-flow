# Python 对象模型总览：以 `a = 123` 串起来（CPython）

> **本篇定位**：把「名字绑定到对象」落到 CPython 的实现直觉：**对象头（`ob_refcnt`/`ob_type`）+ 类型载荷**，以及 `ob_type → PyTypeObject` 如何支撑动态分派。  
> **本篇不负责**：详讲扁平/容器的序列选型（去 `02-container-vs-flat-sequences.md`），也不详讲可变/不可变与 hashable（去 `05-mutability-open-api-and-hash.md`），ABC 与虚拟子类（去 `03-sequence-flat-container-mutable-immutable-abc.md`）。

---

## 一、核心前提：CPython 与「一切皆对象」

日常默认用的 **Python** 多半是 **CPython**（C 实现的解释器）：把 Python 语义落到 **C 里的数据结构与控制流**上，可类比「Java 代码跑在 JVM 上」这种**实现层**直觉。

在 Python **语义**里有一条总纲：**一切皆对象**——`int`、`str`、`list`、函数、类型对象本身等，在运行时都有**对象模型**；在 **CPython** 里，这些对象通常对应堆上的 **C 结构体**实例（带统一的对象头），由解释器分配与回收。

---

## 二、以 `a = 123` 理清：名字、引用与堆上对象

### 1. 例子里的概念

- 语句：`a = 123`
- **`a`**：源码里的**名字**（标识符），不是「装着 123 的小盒子」这种 C 局部变量直觉。
- **`123`**：字面量，求值结果是一个 **`int` 对象**。

### 2. 名字绑定（教学里常说成「变量 = 引用」）

执行赋值的效果是：**把名字 `a` 绑定到「表示 123 的那个 `int` 对象」**。  
在 **CPython** 的 C 层，处理用户可见对象时，习惯用 **`PyObject*`** 这种**统一的指针类型**指过去（见 `02` 第四章）。

**名字 `a` 存在哪里？**  
实现上多在**当前栈帧的局部名字空间**（快速槽位等）或**模块全局字典**里登记 **`a` → 某对象地址**，**不是**硬要说成「C 语言栈帧里有一个名叫 `a` 的 `PyObject*` 自动变量」才算对——但记笔记时可以用一句二分法：**对象实体一般在堆；名字在帧/表里解析后得到指向该对象的引用**。

### 3. 堆上的整数对象：`PyLongObject`（CPython 3）

**注意**：CPython 3 的整数是**任意精度**，**不是**「结构体里一个 C `long ob_val` 就结束」。真实布局更接近（示意，细节以源码为准）：

```c
/* 示意：完整定义见 CPython Include/cpython/longint.h */
typedef struct {
    PyObject_VAR_HEAD   /* 含 ob_refcnt、ob_type、以及与长度相关的字段 */
    digit ob_digit[1]; /* 实际为变长 digit 序列，共同编码整个整数 */
} PyLongObject;
```

- **`ob_refcnt`**：引用计数（整数类型，不是指针）。
- **`ob_type`**：指向该对象的**类型对象**（此处与 `int` 类型对应，如 `PyLong_Type`）。
- **数值 `123`**：编码在 **`ob_digit` 等载荷**里，而不是「`a` 这个字里存 123」。

小整数常被**驻留/复用**（`-5`…`256` 一类），因此多个名字可能指向**同一块** `PyLongObject`，`ob_refcnt` 随之增减——这是实现优化，不改变「绑定到对象」的语义。

### 4. 关系小结

1. **名字 `a`**：在命名空间/栈帧里参与解析，得到**对某对象的引用**（C 层常记为 `PyObject*`）。
2. **堆上的 `PyLongObject`**：`123` 这一值的**载体**，含对象头与**数值编码**。
3. **`PyObject*` 的通用性**：具体类型体（`PyLongObject`、`PyListObject`…）都在开头与 **`PyObject` / `PyVarObject` 头**布局兼容，故可统一当作 `PyObject*` 传递，**具体类型看目标地址处的 `ob_type`**。

### 5. 易混点（一次说清）

- **不是**「`a` 里存着 123 的比特」，而是「`a` 绑定到**存着 123 的对象**」。
- **不是**「`PyObject*` 自己携带类型标签」，而是**被指向的结构体头里的 `ob_type`** 决定类型。
- **名字 `a` 不是对象**；**堆上那块带对象头的结构体**才是（该语境下的）** Python 对象**。

## 三、通用规则：对象头（`PyObject_HEAD` / `PyObject_VAR_HEAD`）

### 1. 对象头是什么

CPython 里，**用户可见对象**对应的结构体，**开头**必须与 **`PyObject` 或变长对象的 `PyVarObject` 头**布局一致。教学里统称**对象头**：保证「任意对象」都能从同一段头部读出引用计数与类型。

### 2. 对象头里常记的两样

1. **`ob_refcnt`**（引用计数）  
   - 记录**有多少条引用路径**指向该对象；降到 0 时，CPython 可立即回收（循环引用另由 **GC** 处理）。  
   - 例：`a = 123` 若独占该对象，计数至少为 1；`b = a` 则同一块 int 的计数增加（若 `b` 也绑定到同一对象）。

2. **`ob_type`（类型指针）**  
   - 类型为 **`PyTypeObject*`**，指向该实例所属的**类型对象**（如 `PyLong_Type` 对应 `int`）。运行时分派 `+`、`len` 等操作，最终都依赖「沿 `ob_type` 找到类型」这条链路。  
   - 更细的展开见下一小节。

### 3. `ob_type` 与 `PyTypeObject`（实现视角）

**`ob_type`** 是 CPython 里 **`PyObject` 头**的核心字段之一：**每个实例对象**头部都有 **`PyTypeObject* ob_type`**，指向**描述该类型的那个对象**（类型本身也是堆上的对象）。

**核心作用（笔记级）**

- **标识类型**：标明实例属于 `int`、`list`、`dict` 还是某个自定义类。
- **支撑多态**：解释器通过类型对象里登记的**方法槽（slot）**与元数据，决定 `obj + 1`、`len(obj)`、`repr(obj)` 等走哪段 C 实现。
- **接上元类**：类型对象由 **`type`（及元类）** 创建；可记一条链：**实例 → `ob_type` → 类型对象**；**类型对象**自己也有 `ob_type`，通常指向 `type` 或你的元类，形成「对象 → 类型 → 元类」层级（细节见元类章节即可）。

**内存布局与源码（简化）**

- 与 **`ob_refcnt`** 一起构成最小 **`PyObject`** 头。Release 版常见写法如下（调试版可能还有额外宏字段，读源码时注意 `Include/object.h`）：

```c
/* Include/object.h — 概念摘录，以你本机源码为准 */
typedef struct _object {
    Py_ssize_t ob_refcnt;
    PyTypeObject *ob_type;
} PyObject;
```

**`PyTypeObject` 里常扫一眼的槽位（记名字即可）**

类型对象是 **`struct _typeobject`**（即 `PyTypeObject`），里面是一大组函数指针与标志位，相当于**「该类型的行为表」**。例如：

| 槽位（示意） | 作用（笔记级） |
| :--- | :--- |
| `tp_name` | 类型名字符串（如 `"list"`），调试、`repr` 等会用到 |
| `tp_doc` | 文档字符串 |
| `tp_call` | 若类型可调用（如函数、类对象），对应「调用」语义 |
| `tp_str` / `tp_repr` | 与 `str()` / `repr()` 相关的展示逻辑 |
| `tp_hash` | 与 `hash()` 可哈希语义相关 |
| `tp_getattro` / `tp_setattro` | 与属性访问、赋值相关（常和 `__getattribute__` 等协议衔接） |

不必背全表；建立直觉：**`ob_type` 指向的那块 `PyTypeObject` 就是「这类对象该怎么被解释器摆弄」的登记表**。

**与日常 Python 的关系**

- 写 Python **不必**碰 `ob_type`；`type(x)`、`x.__class__` 表达的就是「实例的类型对象」这一层概念。
- 写 **C 扩展**、做 **底层调试** 时，常在 C 里读 `Py_TYPE(obj)`（通常即 `obj->ob_type`）判断对象种类。

> **一句话**：**`ob_type` 是每个实例的「类型身份证」，指向描述其行为的类型对象（`PyTypeObject`）。**

### 4. 为什么统一对象头很重要

统一对象头 → **`PyObject*`** 可通用于所有实例 → **容器序列**的槽位只要存 **`PyObject*`**，就能指向**任意类型** → 动态类型的实现基础之一。

## 四、序列：只做“挂钩式”回顾（不展开）

### 1. 容器序列（Container Sequence）

**定义（实现视角）**：序列对象体里主要是 **`PyObject*` 槽位数组**（如 `PyListObject` / `PyTupleObject` 的 `ob_item` 思路）；**元素**是**指针**，指向堆上**各自独立**的对象。

**典型**：`list`、`tuple`、`collections.deque`（块内槽位同理）。

**特点**：

- **异构**：`[123, 'python', 3.14, [1, 2]]` 中，每个元素槽位是一个 `PyObject*`，分别指向 `PyLongObject`、`PyUnicodeObject`、`PyFloatObject`、`PyListObject` 等。
- **代价**：多一层间接；元素常为完整 Python 对象，有头有载荷，**内存与缓存**通常不如大块同构数据友好。

**例**：`t = (123, 'test')` → `PyTupleObject` 里两个槽位，两个 **`PyObject*`**，分别指向 int 与 str 的对象体。

### 2. 扁平序列（Flat Sequence）

**定义（实现视角）**：对象体管理一段**连续缓冲**，里面是**按类型编码的原始数据**（定宽数、`0~255` 字节、Unicode 的内部紧凑表示等），**不是**「一格一个 `PyObject*` 指向子对象」的通用列表。

**典型**：`str`、`bytes`、`bytearray`、`array.array`。

**特点**：

- **同构**：如 `array.array('d', …)` 只能按 `double` 语义紧排；`bytes` 按字节值。
- **注意**：**扁平 ≠ 没有对象头**。外层仍有 `PyUnicodeObject`、`array.array` 的对象头等；**「扁」的是载荷区**，不是「每个元素再包一层完整 Python 标量对象」那种模式。

### 3. 对照表（够用版）

| 对比维度 | 容器序列 | 扁平序列 |
| :--- | :--- | :--- |
| 载荷层存什么 | **`PyObject*`，指向独立对象** | **原始编码数据（同构）** |
| 元素类型 | 可异构 | 单一元素语义 / 类型码 |
| 内存与访问 | 分散、多间接（概念上） | 连续、少「每元素一对象」开销 |
| 典型场景 | 通用业务、嵌套结构 | 文本、二进制、大规模同构数 |

## 五、整体底层逻辑（背诵版）

> 1. **名字**经帧/命名空间解析，绑定到**堆上的对象**（C 层常记 **`PyObject*`**）。  
> 2. **对象体** = **对象头**（如引用计数、类型指针）+ **类型相关载荷**。  
> 3. **统一对象头** → `PyObject*` 可通吃 → **容器槽位可指向任意类型**。  
> 4. **容器序列**：槽位里是 **`PyObject*`** → 灵活、通用。  
> 5. **扁平序列**：载荷是**连续原始数据** → 紧凑、受限、适合同构与 IO/数值。
