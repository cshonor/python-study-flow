一、类属性 vs 实例属性
 
1. 类属性
 
- 定义位置：类内部、所有方法外面
- 特点：
- 所有实例共用同一份
- 修改类属性，所有实例都会跟着变
- 既可以通过类名访问，也可以通过实例访问
 
示例代码
 
```python
  
class FrenchDeck:
    # 类属性
    ranks = [str(n) for n in range(2, 11)] + list('JQKA')
    suits = ['spades', 'diamonds', 'clubs', 'hearts']
 
```

类属性怎么使用
 
```python
  
# 通过类名访问
print(FrenchDeck.ranks)
print(FrenchDeck.suits)

# 通过实例访问
deck1 = FrenchDeck()
print(deck1.ranks)
print(deck1.suits)
 
``` 

 
 
2. 实例属性
 
- 定义方式： self.属性名 ，一般在  __init__  里初始化
- 特点：
- 每个实例自己单独一份，互不干扰
- 只属于当前这个实例

 
示例代码
 
``` python
  
class FrenchDeck:
    ranks = [...]
    suits = [...]

    def __init__(self):
        # 实例属性：每个牌堆自己的 52 张牌
        self._cards = [Card(rank, suit) for suit in self.suits
                                        for rank in self.ranks]
 
 
```
 
 
3. 两个实例之间能不能互相使用对方的属性？
 
结论：
 
可以！
只要你拿到了另一个实例对象，就可以用  实例.属性  访问它的实例属性。
 
示例
 
```python
  
deck1 = FrenchDeck()
deck2 = FrenchDeck()

# deck1 访问 deck2 的属性
print(deck2._cards[0])

# deck2 访问 deck1 的属性
print(deck1._cards[-1])


```
 
一句话理解
 
- 实例属性不是“谁都不能碰”
- 而是：谁拿到这个实例，谁就能点它的属性
 
 
 
二、嵌套两个 for 的列表推导式
 
对应代码：
 
```python
  
self._cards = [Card(rank, suit) for suit in self.suits
                                for rank in self.ranks]
 

```
1. 含义
 
这是两层 for 循环的简写，顺序和普通循环完全一致：
 
1. 第一个 for：遍历花色（外层）

2. 第二个 for：遍历牌面（内层）


 
2. 等价普通循环
 
```python
  
self._cards = []
for suit in self.suits:
    for rank in self.ranks:
        self._cards.append(Card(rank, suit))
 
```
3. 作用
 
- 4 种花色 × 13 种牌面
- 一次性生成 52 张牌，存入  self._cards 
 
 
 
三、魔法方法的执行顺序
 
FrenchDeck 里用到三个关键魔法方法：
 __init__ 、 __len__ 、 __getitem__ 
 
1. 固定执行顺序
 
1.  __init__  第一个执行
- 创建实例  deck = FrenchDeck()  时自动运行
- 负责初始化  self._cards ，没有它后面都用不了
2.  __len__  按需执行
- 调用  len(deck)  时触发
- 返回牌的总数
3.  __getitem__  按需执行
- 用下标  deck[0] 
- 用切片  deck[:5] 
- 用 for 循环  for card in deck 
都会触发它
 
2. 整体执行流程
 
1. 新建对象 → 自动跑  __init__ 
2. 求长度  len(deck)  → 自动跑  __len__ 
3. 索引/切片/循环 → 自动跑  __getitem__ 
 
3. 关键点
 
-  __init__  必须最先执行
- 其他魔法方法都是“用到才跑”，没有固定先后顺序
