# Python 列表、元组、数组 详解

## 0. 前言
在 Python 中，很多人会把 **list** 口语称为“数组”，但它和严格意义上的数组并不完全一样。
本文清晰区分：
- `list`：列表，最常用、最灵活
- `tuple`：元组，不可变序列
- `array.array`：数值数组，同类型、高效存储
- `namedtuple`：具名元组，带字段名的元组

---

## 1. list（列表）
### 定义
列表是 Python 内置的**可变、有序、可重复**的序列容器，使用方括号 `[]` 表示。

### 特点
- 可随时增、删、改、查
- 可存放任意类型数据（int、str、list、对象等）
- 有序，支持索引、切片
- 长度动态可变

### 示例
```python
# 创建列表
nums = [1, 2, 3, 4]
mix = [10, "hello", True, [1, 2], None]

# 访问
print(nums[0])    # 1

# 修改
nums[1] = 200

# 增加
nums.append(5)
nums.insert(0, 0)

# 删除
nums.pop()
nums.remove(3)

# 切片
print(nums[1:3])
```

### 适用场景
业务逻辑、数据收集、临时存储、接口返回、配置列表、量化回测数据等**几乎所有日常场景**。

---

## 2. tuple（元组）
### 定义
元组是**不可变、有序**的序列，使用小括号 `()` 表示。

### 特点
- 一旦创建，**不能修改、添加、删除元素**
- 但可以访问、切片、遍历
- 比 list 略快，更安全，可作为字典的 key
- 常用于固定结构、函数多返回值

### 示例
```python
# 创建元组
point = (10, 20)
info = ("张三", 25, "engineer")

# 访问
print(point[0])   # 10

# 切片
print(info[1:])

# 错误示例：不能修改
# point[0] = 100  # 报错
```

### 适用场景
- 不希望被意外修改的配置
- 函数返回多个值
- 作为字典键
- 结构化轻量数据

---

## 3. array（数组，array 模块）
### 定义
Python 标准库 `array` 提供的**同类型数值数组**，更接近 C 语言数组。

### 特点
- 只能存储**同一类型数值**（int、float 等）
- 内存紧凑、效率高
- 支持大部分 list 操作
- 日常开发极少用，多用于科学计算、大量数值处理

### 示例
```python
from array import array

# 'i' = signed int
arr = array('i', [1, 2, 3, 4, 5])

print(arr[0])
arr.append(6)
arr[1] = 20
```

### 适用场景
- 大量数值数据处理
- 低内存占用场景
- 与底层 C 接口交互
**普通后端、量化脚本基本不用，优先 list / numpy**

---

## 4. namedtuple（具名元组）
### 定义
`collections.namedtuple` 是**带字段名的元组**，本质还是 tuple（不可变），但可以像对象一样访问字段。

### 特点
- 不可变
- 可读性强
- 比手动定义类更轻量
- 常用于结构化数据、DTO、数据记录

### 示例
```python
from collections import namedtuple

Point = namedtuple("Point", ["x", "y"])
p = Point(10, 20)

print(p.x)    # 10
print(p.y)    # 20
print(p[0])   # 10
```

### 适用场景
- 轻量级数据结构
- 替代简单类
- 回测中的 K 线、订单、持仓记录

---

## 5. 核心对比总结

| 类型         | 符号 | 可变 | 数据类型 | 性能 | 常用场景                 |
|------------|------|------|----------|------|--------------------------|
| list 列表    | []   | 是   | 任意     | 常规 | 绝大多数业务场景          |
| tuple 元组   | ()   | 否   | 任意     | 稍快 | 固定结构、安全数据、返回值 |
| array 数组   | 无   | 是   | 仅限数值 | 高   | 大量数值、底层交互        |
| namedtuple  | 无   | 否   | 结构化   | 高   | 轻量级对象、数据记录      |

### 一句话记忆
- **日常开发用 list**
- **不想被改用 tuple**
- **大量数值用 array 或 numpy**
- **要字段名又不想写类用 namedtuple**
