Python 类与魔法方法核心知识点总结（完整版）
 
一、魔法方法（双下划线方法 / dunder 方法）
 
1. 定义：以  __  开头和结尾的预定义特殊方法，是 Python 内置语法钩子，用于让自定义类适配原生语法与函数（ len() 、索引、遍历、打印等）。

2. 是否强制实现：不强制，按需重写
- 大部分魔法方法（ __len__ 、 __getitem__  等）无默认实现，不写则无法使用对应功能，调用会直接报错；
- 少数魔法方法（ __init__ 、 __repr__ ）有默认实现，不重写则使用 Python 自带逻辑。

3. 使用规则：想要类具备某内置功能，必须主动重写对应魔法方法，重写后覆盖默认实现。
 
二、类中方法的第一个参数规则（重点）
 
核心结论
 
- 不加任何装饰器 → 默认是 实例方法 → 第一个参数必须写  self 
- 加  @classmethod  → 类方法 → 第一个参数为  cls 
- 加  @staticmethod  → 静态方法 → 无  self  /  cls 
 
详细说明
 
1. 实例方法（魔法方法 + 普通自定义方法）
- 无装饰器，默认类型；
- 第一个参数必须是  self ，代表当前实例对象；
- 调用时无需传  self ，Python 自动绑定。
2. 类方法
- 必须加  @classmethod  装饰器；
- 第一个参数固定为  cls ，代表类本身。
3. 静态方法
- 必须加  @staticmethod  装饰器；
- 不需要  self  和  cls ，就是放在类里的普通函数。
 
三、普通方法定义
 
- 属于实例方法，无装饰器；
- 第一个参数必须为  self ；
- 方法名无  __  特殊格式，用于实现业务逻辑；
- 通过  实例.方法名()  调用，可访问实例属性与其他方法。
 
四、关键误区
 
1. 魔法方法不是类必备，不需要对应功能可以不写；
2. 不重写魔法方法 = 使用默认实现（或无此功能）；
3. 只要没加  @classmethod  /  @staticmethod ，一律默认实例方法，必须写  self 。
