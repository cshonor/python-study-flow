# 特殊方法（魔术方法）：谁调用、怎么触发

> 《流畅的 Python》中关于**特殊方法**的核心提炼：它们是解释器与你的类之间的**约定接口**——通常只**实现**，由 Python 在合适场景**自动调用**。

---

## 1. 特殊方法的调用方式

特殊方法（如 `__len__`、`__getitem__`）主要由 **Python 解释器**在语法层面触发，而不是日常代码里“点出来”调用。

- **推荐**：`len(my_object)`
- **不推荐**（除非元编程等少数场景）：`my_object.__len__()`

当你写 `len(my_object)` 时，解释器会：

- **内置类型**（如 `list`、`str`）：可走高度优化的内部路径（不等价于总是 Python 层的一次普通方法调用）。
- **自定义类**（如 `FrenchDeck`）：若实现了 `__len__`，会调用你写的 `__len__`。

---

## 2. 隐式调用（常见例子）

很多特殊方法在写语法时**不会写出方法名**，但背后会走到对应钩子：

| 你写的语法 | 背后常涉及的能力（示意） |
|------------|--------------------------|
| `x[i]`、`x[a:b]` | `__getitem__` |
| `x[i] = v` | `__setitem__` |
| `len(x)` | `__len__` |
| `for item in x` | `__iter__`，或退化为 `__getitem__` 的迭代协议 |
| `item in x` | `__contains__`，或退化为遍历 |
| `repr(x)` / 交互式显示 | `__repr__` |
| `str(x)` | `__str__`（无则常回退 `__repr__`） |

在 `FrenchDeck` 里：`deck[0]` 会**隐式**调用 `__getitem__(0)`。

---

## 3. 编写代码时的原则

1. **日常业务代码**：优先使用 **`len()`、`[]`、`for … in`** 等语法，而不是直接调 `__xxx__`。
2. **自定义类**：在类里**实现**需要的特殊方法，让对象表现得像内置类型。
3. **常见例外**：`__init__` 在子类里经常通过 **`super().__init__(...)`** 显式调用，用于协作初始化。

---

## 4. 一句话总结

特殊方法是解释器与自定义类之间的**约定接口**：你实现它们，Python 在合适时机**自动**调用，从而让对象支持 `len`、索引、迭代、`in` 等与内置类型一致或相近的行为。

---

## 5. 常见特殊方法对照表（快速查阅）

| 特殊方法 | 调用方式 / 触发场景 | 作用 |
|----------|---------------------|------|
| `__len__(self)` | `len(obj)` | 返回长度（元素个数） |
| `__getitem__(self, key)` | `obj[key]`（含切片等） | 按索引/键取值 |
| `__setitem__(self, key, value)` | `obj[key] = value` | 按索引/键赋值 |
| `__contains__(self, item)` | `item in obj` | 成员检测（未实现时可能退化为遍历） |
| `__iter__(self)` | `for x in obj` / `iter(obj)` | 返回迭代器（可迭代对象） |
| `__next__(self)` | `next(it)` | 迭代器协议：下一项 |
| `__init__(self, ...)` | `Cls(...)` 构造实例时 | 初始化实例状态 |
| `__str__(self)` | `str(obj)` / `print(obj)` | 面向用户的字符串 |
| `__repr__(self)` | `repr(obj)` / 交互式回显对象 | 面向开发者、宜无歧义 |
| `__add__(self, other)` | `obj + other` | 加法（另有 `__radd__` 等反向算子） |
| `__eq__(self, other)` | `obj == other` | 相等比较 |
| `__call__(self, ...)` | `obj(...)` | 实例像可调用对象 |

更完整的列表见 [Python 数据模型](https://docs.python.org/3/reference/datamodel.html)；本仓库 Part 1 的 `08`、`09`、`10` 等笔记含 `__getitem__`、`__contains__`、`__setitem__` 相关 demo。

## 6. 按三类整理（系统记忆）

### 6.1 数据模型与对象表现

对象“是什么、怎么显示、多大规模”等：

| 方法 | 触发 | 作用 |
|------|------|------|
| `__init__` | `Cls()` | 初始化 |
| `__repr__` / `__str__` | `repr` / `str`、`print` | 字符串表示 |
| `__len__` | `len` | 长度 |

（扩展：`__bool__`、`__hash__`、`__format__` 等也属于广义数据模型。）

### 6.2 运算符与比较（重载）

| 方法 | 触发 | 作用 |
|------|------|------|
| `__add__` 等 | `+`、`-`、`*`… | 算术 |
| `__eq__`、`__lt__` 等 | `==`、`<`… | 比较与排序 key 的基础 |

（扩展：`__radd__`、`__iadd__` 等反向/增量赋值算子。）

### 6.3 迭代与类序列 / 容器

| 方法 | 触发 | 作用 |
|------|------|------|
| `__iter__` | `for`、`iter` | 可迭代 |
| `__next__` | `next` | 迭代器步进 |
| `__getitem__` | `obj[i]` | 序列式访问；无 `__iter__` 时可能参与旧式迭代 |
| `__setitem__` | `obj[i] = v` | 可变序列写回 |
| `__contains__` | `in` | 成员判断（可优化实现） |
