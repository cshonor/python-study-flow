# 第 4 章「Unicode 文本与字节序列」开篇：先把 `str` / `bytes` 的边界讲清楚

Unicode/编码这块，新手最常见的痛点不是“概念多”，而是**边界不清**：什么时候你手里是文本（`str`），什么时候你手里是字节（`bytes`）？什么时候该 `encode`，什么时候该 `decode`？

这一章你只要先建立一个稳定的心智模型：**`str` 是人类文本（码点序列），`bytes` 是存储/传输用的字节序列**。两者之间的转换必须显式，并且要写清楚编码规则。

---

## 一、核心纲领

> 文本给人类阅读，字节序列供计算机处理。  
> ——Esther Nam 和 Travis Fischer（书中引用）

- **`str`**：Unicode **字符序列**（逻辑上是**码点**序列），**不绑定**某一具体「磁盘上的字节形态」。  
- **`bytes` / `bytearray`**：**字节**的不可变 / 可变序列，是「物理存储与传输」层面的数据。  
- **编码（encode）/ 解码（decode）**：在「文本」与「某套编码规则下的字节」之间**显式**转换；Python 3 **取消**了 Python 2 式 **`str`/`unicode` 与字节混用时的隐式转换**，从源头减少乱码类 bug。

---

## 二、本章知识地图（结构化）

| 模块 | 核心知识点 | 学习价值 |
| :--- | :--- | :--- |
| **字符、码点、字节** | 字符与 **Unicode 码点**；**UTF-8** 等编码的角色 | 读懂「为何同一字符会有不同字节」 |
| **二进制序列类型** | **`bytes`**、**`bytearray`**、**`memoryview`** | 网络、文件、缓冲区的高效处理 |

## 三、为什么要精读本章

- **高频问题**：爬虫与 HTTP、读写文件、数据库与 JSON、子进程与管道——凡涉及「字节进、文本出」之处，编码错误都会表现为**乱码**或**异常**。  
- **工程能力**：国际化文本、二进制协议、加密与压缩，都要求你**自觉**处理 **`str` 与 `bytes` 的边界**。  
- **平台差异**：**默认编码**依赖 **locale** 与 **Python 版本**（例如文本模式打开文件时的默认、标准流编码），**不要**在代码里假设「永远是 UTF-8」或「永远是某一种中文编码」；**显式指定**最稳。

## 四、ASCII / Unicode / UTF-8… 一层怎么记（大白话）

### 一句总结构

- **`ASCII`**：**最小的「字符集」**，并且**自带**「每个字符怎么变成 1 个字节」的规则（**0–127**，常用说法「7 位」）。  
- **`Unicode`**：**超大的字符集**——给全世界字符**分配码点 `U+xxxx`**；**本身不负责**「磁盘上几个字节怎么排」。  
- **`UTF-8` / `GBK` / `UTF-16`…**：**编码（encoding）**——把 **Unicode 码点**按各自规则**翻译成 `bytes`**；其中 **UTF-8** 是 **Unicode 的一种**具体字节编码。

### 纯文本层级图（一眼分清）

```text
第一层：字符集（“有哪些字、编号多少”）
├── ASCII      ：128 个字符，码点 U+0000～U+007F
└── Unicode    ：收录远多于 ASCII；前 128 个码点与 ASCII 对齐 → 纯英文 UTF-8 与 ASCII 字节常一致

第二层：编码（“码点怎么变成字节”）
├── ASCII      ：上面已自带“1 字节”规则
├── UTF-8      ：变长，Unicode 码点 → 1～4 字节（常用）
├── GBK/GB2312 ：中文区历史双字节方案（与 Unicode 并行演化，工程里常显式写 gbk）
├── UTF-16     ：常 2 字节/码点（BMP），更高码点可出现 4 字节（代理对）
└── latin-1 等 ：单字节 0～255 映射，排查时易“假解码成功”（见 `05`）
```

### 三个最容易混的点（直接斩断）

1. **「Unicode 是一种编码」？**  
   **不准确。** 日常说「Unicode 编码」多指 **UTF-8**；严格讲 **Unicode 是字符集（码点表）**，**UTF-8/UTF-16/GBK** 才是「怎么写成字节」。

2. **「UTF-8 和 Unicode 打架」？**  
   **不打架。** **UTF-8** 是 **Unicode 标准里规定的**一种**字节编码形式**。

3. **「ASCII 和 Unicode 没关系」？**  
   **有关系。** **Unicode 前 128 个码点与 ASCII 一一对应**，所以 **ASCII 文本**在 **UTF-8** 下通常**字节级兼容**。

### 终极背诵口诀

**ASCII 小号全集自带字节；Unicode 大号全集只发码点；UTF-8/GBK 才是把码点压进磁盘的翻译规则。**
