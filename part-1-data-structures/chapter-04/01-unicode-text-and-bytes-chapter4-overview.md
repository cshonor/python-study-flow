## 一、核心纲领

> 文本给人类阅读，字节序列供计算机处理。  
> ——Esther Nam 和 Travis Fischer（书中引用）

- **`str`**：Unicode **字符序列**（逻辑上是**码点**序列），**不绑定**某一具体「磁盘上的字节形态」。  
- **`bytes` / `bytearray`**：**字节**的不可变 / 可变序列，是「物理存储与传输」层面的数据。  
- **编码（encode）/ 解码（decode）**：在「文本」与「某套编码规则下的字节」之间**显式**转换；Python 3 **取消**了 Python 2 式 **`str`/`unicode` 与字节混用时的隐式转换**，从源头减少乱码类 bug。

| 模块 | 核心知识点 | 学习价值 |

## 三、为什么要精读本章

- **高频问题**：爬虫与 HTTP、读写文件、数据库与 JSON、子进程与管道——凡涉及「字节进、文本出」之处，编码错误都会表现为**乱码**或**异常**。  
- **工程能力**：国际化文本、二进制协议、加密与压缩，都要求你**自觉**处理 **`str` 与 `bytes` 的边界**。  
- **平台差异**：**默认编码**依赖 **locale** 与 **Python 版本**（例如文本模式打开文件时的默认、标准流编码），**不要**在代码里假设「永远是 UTF-8」或「永远是某一种中文编码」；**显式指定**最稳。

---

## 四、学习顺序建议

1. **先固定心智模型**：`str` = 人类可读文本抽象；`bytes` = 可传输、可存储的**原始字节**；二者转换必须**显式**并**指明编码**（或依赖明确、文档化的默认）。  
2. **再练 `encode`/`decode`**：包括**错误处理策略**，能读懂 traceback 中的 encode/decode 位置。  
3. **最后进阶**：规范化、排序、Unicode 数据库与库 API 的 **`str`/`bytes`** 分支。

## 五、面试速记（极简）

- **`str` vs `bytes`**：前者 Unicode 文本；后者 0–255 字节序列。  
- **码点**：字符在 Unicode 中的编号（如 `U+4E2D`）。  
- **UTF-8**：变长编码，兼容 ASCII，是 Web 与许多系统的首选。  
- **原则**：**decode 靠近边界**（读入字节后尽快变成 `str`）；**encode 靠近边界**（写出、发送前再变成 `bytes`）。  
- **常见异常**：**`UnicodeDecodeError`**（把字节当某种编码解释失败）；**`UnicodeEncodeError`**（把 `str` 用某编码写成字节失败）。
