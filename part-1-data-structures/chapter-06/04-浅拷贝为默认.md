# 6.4 默认做浅拷贝：为什么“复制了容器”还是会互相影响（以及何时必须 `deepcopy`）

这一节要解决的困惑通常是这种一句话：

> “我明明 copy 了，怎么改了一个，另一个也跟着变？”

答案几乎总是：**你做的是浅拷贝（shallow copy）**。

浅拷贝只会“复制最外层容器壳子”，容器内部仍然指向原来的元素对象（尤其是可变对象）。因此只要内部有可变对象，就可能发生“共享引用 → 互相影响”的隐形 bug。

配套脚本：`04_shallow_copy_and_deepcopy_demo.py`（6.4：`l1/l2` 复现 + `Bus` 的 copy/deepcopy + 循环引用 deepcopy）。

---

## 一、浅拷贝到底拷贝了什么

### 1.1 一句话定义（请背下来）

> **浅拷贝：创建一个新容器，但容器里的元素引用不变（仍指向原来的元素对象）。**

结果：

- 外层容器：新对象（`outer1 is outer2` 为 False）
- 内层元素：通常还是同一批对象（比如 `outer1[1] is outer2[1]` 可能为 True）

### 1.2 三种常见浅拷贝写法（语义等价：只拷一层）

```python
l2 = list(l1)   # 构造函数：通用
l2 = l1[:]      # 切片：序列专用（简洁）
l2 = l1.copy()  # 方法：语义最明确（“我要浅拷贝”）
```

这些方式的共同点：**只复制最外层结构**。

---

## 二、浅拷贝的致命坑：外层独立、内层共享

用书里最经典的结构（一个列表里既有列表也有元组）：

```python
l1 = [3, [66, 55, 44], (7, 8, 9)]
l2 = list(l1)  # 浅拷贝
```

你需要分开看两类修改：

### 2.1 修改外层容器（互不影响）

```python
l1.append(100)
```

解释：

- `l1`、`l2` 是两个不同的外层 list 对象
- 改 `l1` 的长度，不会影响 `l2`

### 2.2 修改内层可变对象（会互相影响）

```python
l1[1].remove(55)
```

解释：

- `l1[1]` 和 `l2[1]` 指向同一个内部 list 对象（这是浅拷贝的典型结构共享）
- 你修改的是“内部 list 对象本体”，因此 `l2` 也会看到变化

---

## 三、`+=` 的双重含义：就地修改 vs 重新绑定（浅拷贝场景里最容易误判）

同样是 `+=`，在不同类型上语义可能完全不同：

### 3.1 `list += [...]`：通常是“就地修改”（in-place）

```python
l1[1] += [33, 22]
```

这往往等价于：

- `l1[1].extend([33, 22])`

因为内部 list 是共享的，所以 `l2[1]` 也会“跟着变”。

### 3.2 `tuple += (...)`：不可变 → 只能创建新对象并重新绑定

```python
l1[2] += (10, 11)
```

tuple 不可变，所以它无法就地扩展，只能：

- 创建一个新 tuple
- 把 `l1[2]` 重新绑定到新 tuple

这时：

- `l2[2]` 仍指向旧 tuple，不会变

结论：

> 在浅拷贝场景里，判断“会不会互相影响”，关键看操作是不是**就地修改**同一个可变对象。

---

## 四、浅拷贝 vs 深拷贝：你到底想要“隔离到哪一层”

### 4.1 对比表（把差异记在“结构”上）

| 维度 | 浅拷贝（shallow） | 深拷贝（deep） |
|---|---|---|
| 复制层级 | 只复制外层容器 | 递归复制内部结构 |
| 内存开销 | 较小（共享内部对象） | 较大（内部对象也复制） |
| 内部可变对象 | **共享引用**，容易互相影响 | **互不影响**（隔离更彻底） |
| 常用方式 | `x.copy()` / `copy.copy(x)` / `x[:]` | `copy.deepcopy(x)` |

你真正要问的是：

> “我希望复制后，哪些层级必须独立？”

如果只要求外层独立（内部不改），浅拷贝足够；如果内部也会改，考虑深拷贝或更精确的“按层级复制”。

---

## 五、实战：`copy.copy` vs `copy.deepcopy`（Bus 示例）

这个例子非常适合理解“对象里装着可变属性”时会发生什么：

- `Bus.passengers` 是一个 list（可变）
- 浅拷贝只会复制 `Bus` 外壳，但 passengers 仍可能共享
- 深拷贝会把 passengers 也复制一份

配套脚本会打印 `id(bus.passengers)` 来给你证据。

---

## 六、循环引用：为什么 `deepcopy` 不会无限递归

如果结构里出现循环引用（对象间互相引用，或容器把自己装进去），`deepcopy` 会：

- 用一个 `memo` 字典记录“已经拷贝过的对象”
- 遇到重复对象时直接复用“拷贝出来的那个”，避免无限递归

这就是为什么 `deepcopy` 能正确处理“自包含列表”之类结构。
