# 6.4 默认做浅拷贝：为什么“复制了容器”还是会互相影响（以及何时必须 `deepcopy`）

这一节要解决的困惑通常是这种一句话：

> “我明明 copy 了，怎么改了一个，另一个也跟着变？”

答案几乎总是：**你做的是浅拷贝（shallow copy）**。

浅拷贝只会“复制最外层容器壳子”，容器内部仍然指向原来的元素对象（尤其是可变对象）。因此只要内部有可变对象，就可能发生“共享引用 → 互相影响”的隐形 bug。

## 大白话速通（与 **§一·1.3**、脚本 **§7** 同一套例子）

下面用 **`a = [1, 2, [3, 4]]`**、**`b = copy.copy(a)`** 把「浅拷贝」钉死。

### 1）`a` 和 `b` 是什么关系？

- **外层列表**：**不是同一个对象**（**`a is b`** 为 **`False`**），各自有自己的三个槽位。  
- **内层列表**：**往往是同一个对象**（**`a[2] is b[2]`** 通常为 **`True`**），所以**改内层**会一起看见。

### 2）改外层槽位：一般各改各的

只给 **`b[0]`** 重新赋值时，改的是 **`b` 第 0 个槽里存的是谁**，**不动** **`a[0]`** 指向的 **`1`**：

```python
b[0] = 99
# a → [1, 2, [3, 4]]
# b → [99, 2, [3, 4]]
```

（正文 **§一·1.3** 用 **`99` / `999`** 演示同一逻辑。）

### 3）改内层（嵌套的可变对象）：常常两边一起变

**`b[2][0] = 99`** 改的是 **`a[2]` 与 `b[2]` 共用的那个 list**，不是换 **`b[2]`** 这个槽的指向：

```python
import copy

a = [1, 2, [3, 4]]
b = copy.copy(a)
b[2][0] = 99
# a → [1, 2, [99, 4]]，b → [1, 2, [99, 4]]（同一子 list 被就地改掉）
```

一句话：**浅拷贝只「换容器壳」**，**壳里装的引用**对嵌套可变对象来说**常常照旧**。

### 4）内存直觉（好记版）

```
a: [槽0] [槽1] [槽2]───┐
                         ├──→ 同一个子 list 对象 [3, 4]
b: [槽0] [槽1] [槽2]───┘
```

### 5）想内层也完全独立：用深拷贝

```python
import copy

a = [1, 2, [3, 4]]
b = copy.deepcopy(a)

b[2][0] = 99
# a 仍是 [1, 2, [3, 4]]；b 为 [1, 2, [99, 4]]（内层 list 也已复制为新对象）
```

### 6）两句口诀（背）

- **`copy.copy`**：**新外壳**，**里层引用常共享**（嵌套可变要小心）。  
- **`copy.deepcopy`**：**递归新对象**，**内层也隔离**（更慢、更占内存，但更安全）。

**赋值 / 浅拷贝 / 深拷贝对照**：见脚本 **`demo_shallow_copy_catalog_and_assignment_contrast`**（输出里含 **`outer new`**、**`inner list shared`**、以及先改外层再改内层后的 **`src` / `shallow`**、**`deepcopy` 后互不影响**）。

## 一、浅拷贝到底拷贝了什么

### 1.1 一句话定义（请背下来）

> **浅拷贝：创建一个新容器，但容器里的元素引用不变（仍指向原来的元素对象）。**

结果：

- 外层容器：新对象（`outer1 is outer2` 为 False）
- 内层元素：通常还是同一批对象（比如 `outer1[1] is outer2[1]` 可能为 True）

**权威表述**（与 Python 文档一致）：浅拷贝构造一个新的复合对象，再把**原对象里各元素的引用**插进去；深拷贝则**递归**为新结构插入**子对象的拷贝**。详见 [The `copy` module — Python 3 documentation](https://docs.python.org/3/library/copy.html)。

### 1.2 常见浅拷贝方式（速查，语义等价：只拷一层）

#### 1）`copy.copy(x)`（通用）

`copy` 模块提供的**通用**浅拷贝，多数实现了拷贝协议的类型都能用。

```python
import copy

a = [1, 2, [3, 4]]
b = copy.copy(a)
```

#### 2）序列切片 `x[:]`（仅序列）

适用于 **`list` / `tuple` / `str` 等序列**：`x[:]` 得到**新序列**，元素仍是原引用。列表里最常用。

```python
a = [1, 2, [3, 4]]
b = a[:]
```

对 **`str`**、元素为不可变对象的 **`tuple`**，浅拷贝与深拷贝在「是否共享内部」上往往**看不出区别**（没有可变内层可改）。

#### 3）工厂函数 `list(x)` / `dict(x)` / `set(x)` / `tuple(x)` …

用构造器得到**新容器**，元素仍来自 `x` 的**同一批引用**（浅拷贝语义）。

**例 A：`list(...)`（与官方「浅拷贝」表述一致；对 `list` 与 `copy.copy` 通常等价）**

[The `copy` module](https://docs.python.org/3/library/copy.html) 对 **shallow copy** 的定义大意是：**新容器 + 原容器里各元素引用原样插入**。对 **`list`** 而言，**`list(a)`** 正是这种行为。

- **`b = list(a)`**：新建一个**外层 list**（**`a is b`** → **`False`**）。  
- **只复制「第一层」**：槽里装的是**引用**，**不递归复制**子对象。  
- **内层可变对象共享**：嵌套 list 时 **`a[2] is b[2]`** 通常为 **`True`**（同一子列表）。

**可运行自证**（复制即跑）：

```python
a = [1, 2, [3, 4]]
b = list(a)

print(a is b)       # False：两个不同 list 对象
print(a[2] is b[2])  # True：同一子列表

b[0] = 99
print(a)  # [1, 2, [3, 4]]
print(b)  # [99, 2, [3, 4]]

b[2][0] = 99
print(a)  # [1, 2, [99, 4]]
print(b)  # [99, 2, [99, 4]]
```

**对同一 `a`，常见四种「列表浅拷贝」写法效果同类**（外层新、内层常共享；以 `is` / `a[2] is b[2]` 自证即可）：

```python
import copy

a = [1, 2, [3, 4]]
b1 = list(a)
b2 = a[:]
b3 = a.copy()
b4 = copy.copy(a)
```

**例 B：`dict(...)`（新外壳，值仍是原引用）**

```python
d1 = {"x": [1, 2]}
d2 = dict(d1)
# d1 is not d2；但 d1["x"] is d2["x"] 通常为 True（同一个内层 list）
```

上面两段**互不依赖**，只是各演示一种「工厂浅拷贝」。

#### 4）容器自带的 `.copy()`（若类型提供）

- `list.copy()`、`dict.copy()`、`set.copy()` 等：**新外壳，元素引用共享**（与上面同类）。

```python
a = [1, 2, [3, 4]]
b = a.copy()
```

以上写法的共同点：**只复制最外层容器**；嵌套的可变对象仍可能**共享引用**。

### 1.3 最小背诵例子：改外层 vs 改内层

```python
import copy

a = [1, 2, [3, 4]]
b = copy.copy(a)

b[0] = 99       # 只改 b 的外层槽位 → a[0] 不变
b[2][0] = 999   # 改的是「a 与 b 共用的那个内层 list」→ a、b 都变

# a → [1, 2, [999, 4]]   b → [99, 2, [999, 4]]
```

### 1.4 对比记忆（赋值 / 浅拷贝 / 深拷贝）

| 写法 | 结果直觉 |
| :--- | :--- |
| `b = a` | **同一对象**：`a is b` 为 True，改任一层都可能牵连对方 |
| `copy.copy(a)`、`a[:]`、`list(a)`、`a.copy()`、`dict(d)` … | **外壳新、内层常共享**：适合「只动外层」或内层不可变 |
| `copy.deepcopy(a)` | **递归新对象**：内层可变对象也隔离（开销更大；见 **§四、§五**） |

### 1.5 三种情况怎么记：一眼图（赋值 / 浅拷贝 / 深拷贝）

变量是**名字**，对象是**内存里的东西**；下面用箭头表示「名字指向谁」。

**① 赋值：`b = a`（别名，同一外壳）**

```text
  a ──┐
      ╰──→  [ list: 槽0→1, 槽1→2, 槽2→子list[3,4] ]
  b ──┘              （只有一个 list 对象）
```

改 **`b[0]`** 或 **`b[2][0]`**，**`a` 都看得见**（因为就是同一个 **`list`**）。

**② 浅拷贝：`b = list(a)`（两副外壳，共用一个内层）**

```text
  a ──→  [ list A: 槽2 ───────┐
                              ├──→ 子 list [3, 4]  （只有一份）
  b ──→  [ list B: 槽2 ───────┘
```

改 **`b[0]`**：只动 **B 的槽 0**，**A 的槽 0** 仍指向原来的 **`1`**。  
改 **`b[2][0]`**：动的是**共用的子 list**，**`a`、`b` 里看到的内层一起变**。

**③ 深拷贝：`b = copy.deepcopy(a)`（外壳、内层各复制一份）**

```text
  a ──→  [ list A ] ──→ 子 list A [3, 4]
  b ──→  [ list B ] ──→ 子 list B [3, 4]   （两套独立对象）
```

改 **`b[2][0]`** 只动 **B 的内层**，**`a` 不受影响**。

## 二、浅拷贝的致命坑：外层独立、内层共享

用书里最经典的结构（一个列表里既有列表也有元组）：

```python
l1 = [3, [66, 55, 44], (7, 8, 9)]
l2 = list(l1)  # 浅拷贝
```

你需要分开看两类修改：

### 2.1 修改外层容器（互不影响）

```python
l1.append(100)
```

解释：

- `l1`、`l2` 是两个不同的外层 list 对象
- 改 `l1` 的长度，不会影响 `l2`

### 2.2 修改内层可变对象（会互相影响）

```python
l1[1].remove(55)
```

解释：

- `l1[1]` 和 `l2[1]` 指向同一个内部 list 对象（这是浅拷贝的典型结构共享）
- 你修改的是“内部 list 对象本体”，因此 `l2` 也会看到变化

## 三、`+=` 的双重含义：就地修改 vs 重新绑定（浅拷贝场景里最容易误判）

同样是 `+=`，在不同类型上语义可能完全不同：

### 3.1 `list += [...]`：通常是“就地修改”（in-place）

```python
l1[1] += [33, 22]
```

这往往等价于：

- `l1[1].extend([33, 22])`

因为内部 list 是共享的，所以 `l2[1]` 也会“跟着变”。

### 3.2 `tuple += (...)`：不可变 → 只能创建新对象并重新绑定

```python
l1[2] += (10, 11)
```

tuple 不可变，所以它无法就地扩展，只能：

- 创建一个新 tuple
- 把 `l1[2]` 重新绑定到新 tuple

这时：

- `l2[2]` 仍指向旧 tuple，不会变

结论：

> 在浅拷贝场景里，判断“会不会互相影响”，关键看操作是不是**就地修改**同一个可变对象。

## 四、浅拷贝 vs 深拷贝：你到底想要“隔离到哪一层”

### 4.1 对比表（把差异记在“结构”上）

| 维度 | 浅拷贝（shallow） | 深拷贝（deep） |
|---|---|---|
| 复制层级 | 只复制外层容器 | 递归复制内部结构 |
| 内存开销 | 较小（共享内部对象） | 较大（内部对象也复制） |
| 内部可变对象 | **共享引用**，容易互相影响 | **互不影响**（隔离更彻底） |
| 常用方式 | `copy.copy(x)` / `x.copy()` / `x[:]` / `list(x)` / `dict(x)` … | `copy.deepcopy(x)` |

你真正要问的是：

> “我希望复制后，哪些层级必须独立？”

如果只要求外层独立（内部不改），浅拷贝足够；如果内部也会改，考虑深拷贝或更精确的“按层级复制”。

## 五、实战：`copy.copy` vs `copy.deepcopy`（Bus 示例）

这个例子非常适合理解“对象里装着可变属性”时会发生什么：

- `Bus.passengers` 是一个 list（可变）
- 浅拷贝只会复制 `Bus` 外壳，但 passengers 仍可能共享
- 深拷贝会把 passengers 也复制一份

配套脚本会打印 `id(bus.passengers)` 来给你证据。

## 六、循环引用：为什么 `deepcopy` 不会无限递归

如果结构里出现循环引用（对象间互相引用，或容器把自己装进去），`deepcopy` 会：

- 用一个 `memo` 字典记录“已经拷贝过的对象”
- 遇到重复对象时直接复用“拷贝出来的那个”，避免无限递归

这就是为什么 `deepcopy` 能正确处理“自包含列表”之类结构。
