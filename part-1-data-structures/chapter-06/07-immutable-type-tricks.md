# 6.7 Python 对不可变类型施加的把戏：复用、驻留与“千万别用 `is` 比值”

这一节讲的是一种“看起来像魔法、但其实是优化”的现象：

> **对于不可变对象，解释器经常选择“复用旧对象”而不是“创建新对象”。**

因为不可变对象的值永远不会变，复用并不会破坏语义，反而能省内存、提速度。

但这会产生一个非常危险的误解：

> “既然 `a is b` 有时是 True，那我是不是可以用 `is` 比较值？”

答案是：**绝对不可以**。这类复用/驻留属于实现细节，不是 Python 语言规范承诺的行为。

## 一、元组（tuple）的复用：`tuple(t)`、`t[:]` 可能直接返回原对象

### 1.1 你会看到的现象

```python
t1 = (1, 2, 3)
t2 = tuple(t1)
t3 = t1[:]

print(t2 is t1)  # 可能 True
print(t3 is t1)  # 可能 True
```

直觉解释：

- 元组不可变，复制一个“完全一样且永不改变”的对象没有意义
- 所以解释器可能直接把原对象返回

### 1.2 重要边界：不要把它当成“保证”

这不是承诺：

- 不同 Python 版本/实现（CPython/PyPy 等）行为可能不同
- 某些情况下（例如你传入的不是 tuple，而是 tuple 子类或别的可迭代对象），`tuple(x)` 当然会创建新 tuple

所以你只能把它当成“可能发生的优化”。

## 二、`frozenset` 的复用：不可变集合的 `.copy()` 常常返回自身

`frozenset` 不可变，因此：

```python
fs1 = frozenset({1, 2, 3})
fs2 = fs1.copy()
print(fs2 is fs1)  # 可能 True
```

道理同 tuple：复制不可变值通常没意义。

## 三、字符串驻留（interning）：相同字符串可能复用同一对象

### 3.1 为什么会有驻留

字符串在 Python 里极其常见，尤其是：

- 标识符
- 属性名
- 关键字
- 小而频繁的字面量

驻留可以让这些字符串复用同一对象，省内存、加速比较。

### 3.2 你能观察到的现象（但不要依赖）

```python
s1 = "ABC"
s2 = "ABC"
print(s1 is s2)  # 很可能 True（实现细节）
```

但一旦涉及“运行时拼接/构造”，就不稳定了：

```python
s3 = "AB" + "C"      # 可能在编译期折叠成字面量
s4 = "".join(["A", "B", "C"])  # 运行时构造，通常不会自动驻留
```

你会看到 `s1 is s3` 可能 True，而 `s1 is s4` 往往 False。

### 3.3 `sys.intern()`：你可以手动驻留（但这是优化手段，不是日常写法）

```python
import sys

a = sys.intern("a_b_c")
b = sys.intern("a_" + "b_" + "c")
print(a is b)  # True
```

用途：

- 只有在非常明确的性能瓶颈（大量重复字符串比较/存储）时才考虑
- 业务代码一般不需要刻意 intern

## 四、小整数缓存：`-5..256` 常见被缓存（但仍然别用 `is` 比值）

很多人会在 CPython 里观察到：

```python
a = 10
b = 10
print(a is b)  # 很可能 True
```

这常见来自小整数缓存。但你必须把它当作实现细节：

- 缓存范围可能不同（实现与编译选项相关）
- 不同解释器行为不同
- 你写的代码形式也可能影响是否复用

因此：

> **比较数值相等永远用 `==`，不要用 `is`。**

---

## 五、这一节真正想教你的：`is` 的边界

### 5.1 `is` 的正确用途（硬规则）

- 判断 `None`：`x is None` / `x is not None`
- 判断是否为同一个对象（别名检测、哨兵对象、单例）

### 5.2 `==` 才是“值比较”

业务代码关心的是“值是否相等”，不关心“是否碰巧复用了同一对象”。

所以：

- `a == b`：值相等（可重载）
- `a is b`：同一对象（不可重载）
