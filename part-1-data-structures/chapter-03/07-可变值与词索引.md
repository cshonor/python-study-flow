# 字典里放“可变值”（list 等）时，怎么优雅地“缺了就建、然后 append”？

新手写“分组/索引”时最常见的需求是这个：

- key（比如一个单词）第一次出现：你要给它建一个空列表 `[]`
- 之后再出现：你要在同一个列表里 `append(...)`

难点不在 `append`，而在“缺键时怎么初始化”。

这篇用一个经典例子解释清楚：**单词 → 出现位置列表**（word index）。我们会对比 3 种写法：

- `get` + 写回
- `setdefault` 一行
- `defaultdict(list)`

它们都能做出正确结果，但“可读性、是否原地写入、是否额外分配对象”的差异很大。

**标题（大字号主文案）**  
字典里放可变值（list）｜优雅实现：缺了就建 + append

**核心需求**

- 第一次出现 key → 创建空列表 `[]`
- 再次出现 key → 直接在列表 `append`
- 难点：缺键时如何优雅初始化

---

### 三种实现方案（含代码）

#### 方案 1：`get` + 写回（基础版）

```python
d = {}
key = "a"
lst = d.get(key, [])
lst.append(1)
d[key] = lst  # 必须写回
```

- **优点**：易懂、不抛 `KeyError`  
- **缺点**：**3 行冗余**；`get(key, [])` **每次都会对 `[]` 求值**，键已存在时多造一个随即丢弃的空列表（见 **§四**）

#### 方案 2：`setdefault`（一行版）

```python
d = {}
key = "a"
d.setdefault(key, []).append(1)
```

- **优点**：**一行搞定**，无需手动写回  
- **缺点**：`setdefault(key, [])` **每次调用仍先对 `[]` 求值**；键已存在时同样多分配丢弃的 `[]`；且**不走** `defaultdict` 的 **`default_factory` / `__missing__`** 那条链（与「仅缺键才 `list()`」不是一回事，见 **`06-dict-defaultdict与OrderedDict对照.md`** **§五**）

#### 方案 3：`defaultdict(list)`（优雅版，推荐）

```python
from collections import defaultdict

d = defaultdict(list)
key = "a"
d[key].append(1)  # 缺键自动 list()，再 append
```

- **优点**：**极简**；**仅缺键时**才 `list()`，键已存在时无多余 `[]` 分配；高频分组/索引省心  
- **缺点**：需 `from collections import defaultdict`；类型为 **`defaultdict`**（多数场景可当 `Mapping` 用；与裸 `dict` 的细微差别见 **`06`**、**`08`**）

**单词索引里**把 `d` / `key` 换成 `index` / `word`，把 `1` 换成 `location` 即可与 **§三** 对齐。

---

### 直观对比表（写法 × 行数）

| 写法 | 代码行数 | 核心特点 |
| :--- | :---: | :--- |
| `get` + 写回 | 3 | 需要写回；`[]` 每次都求值 |
| `setdefault` | 1 | 一行简洁；`[]` 每次仍求值 |
| `defaultdict(list)` | 1 | 仅缺键创建 `list()`；无上述浪费 |

---

### 原理深度解析（卡片级）

#### 1. `defaultdict` 工作机制

- 继承自 **`dict`**，在 **`d[k]` 缺键**时配合 **`default_factory`**（内部与 **`__missing__`** 语义衔接）。  
- 初始化时传入**工厂**（如 **`list`**、`int`、`set`）。  
- **缺键访问 `d[k]`** 时：调用工厂 → 写入 `k` → 返回该默认值。  
- **关键**：**`.get(k)` 不触发** `default_factory` / `__missing__`**；**`.setdefault(...)` 同样不触厂**（要「按工厂自动补」请用 **`d[k]`**）。

#### 2. 为什么常比「`setdefault(k, [])`」更省心？

- **`setdefault(k, []).append(x)`**：每次调用都会**先求值**实参 **`[]`**；键已存在时仍会多造一个丢弃的列表。  
- **`defaultdict(list)` + `d[k].append(x)`**：**仅缺键**时调用 **`list()`**；键已存在时直接取已有列表，无上述浪费。

---

### 避坑指南（高频易错）

#### 1. 禁止 `dict.fromkeys` 共享可变默认

```python
# 错：所有键共享同一个列表对象
d = dict.fromkeys(["a", "b"], [])
d["a"].append(1)
# d → {'a': [1], 'b': [1]}

# 对：推导式，每键独立新列表
d = {k: [] for k in ["a", "b"]}
```

原因：`fromkeys` 的默认值是**同一对象引用**；可变对象会「一串全动」。与 **`get(key, [])`「每调一次、先算一次 `[]`」** 的对照（**求值几次**）见 **§四** `dict.fromkeys` 小节。

#### 2. `defaultdict` 与 `get` 混用

```python
from collections import defaultdict

d = defaultdict(list)
d["a"].append(1)

print(d["b"])  # []（缺键 → 工厂写入）
print(d.get("c"))  # None（不触厂、不插键）
```

**规则**：**只有 `d[k]`** 走 **`default_factory`/`__missing__`**；**`get` / `setdefault` 都不触厂**（与 **`06` §零.5、§五** 一致）。

#### 3. 选型原则（照抄）

- **普通 `dict`（3.7+）**：只要**插入顺序**、无缺键自动建容器 → **首选 `dict`**。  
- **`defaultdict(list)`**：**高频分组 / 追加**，缺键要自动 **`list()`** → **推荐**。  
- **`setdefault`**：**偶尔**补一项、**不想导入** `collections` → 够用。

---

### 选型速查表（一句话决策）

| 场景 | 最优选择 | 代码形态 |
| :--- | :--- | :--- |
| 键是否存在未知，但要**追加列表** | **`defaultdict(list)`** | `d[key].append(x)` |
| 小脚本、**不想导入**模块 | **`setdefault`** | `d.setdefault(key, []).append(x)` |
| **只读**、缺键也不要插入 | **`get`** | `d.get(key, [])` |
| 批量建「每键独立空列表」 | **字典推导式** | `{k: [] for k in keys}` |

---

### 总结（卡片收束）

**字典里放可变值（`list`），缺键初始化优先：`defaultdict(list)`**  
核心价值：**缺键自动建列表，一行 `d[key].append(value)`**。  
避坑关键：**区分 `d[k]` 与 `get`；可变默认别共享引用**。

## 一、场景

- **输入**：多行文本。  
- **输出**：`dict[str, list[tuple[int, int]]]`，键为词，值为 **`(行号, 列号)`** 列表（列号从 1 起与书中一致时，对 `match.start()` **+1**）。  
- **难点**：词**首次**出现要挂**新列表**；之后在同一列表上 **`append`**。

---

## 二、示例 3-4 风格：`get` + 写回

```python
occurrences = index.get(word, [])
occurrences.append(location)
index[word] = occurrences
```

### 优点

- 易读；不会 `KeyError`。

### 缺点

- **三步**：取列表 → 改列表 → **写回**（键已存在时写回仍执行）。  
- **默认实参求值**：`get(word, [])` 在**每次调用**时都会先对第二个实参求值，**新建一个 `[]`**；若键已存在，`get` 返回已有列表，**刚建的那个空列表会被丢弃**。  
- 同理：**`setdefault(word, [])` 里的 `[]` 也会在每次调用时被求值**（见 §四与脚本）。  
- 若希望「**仅缺键才分配**」：`defaultdict(list)`（§五）。

---

## 三、示例 3-5 风格：`setdefault` 一行（书中最常推荐的写法）

```python
WORD_RE = re.compile(r"\w+")
index: dict[str, list[tuple[int, int]]] = {}

with open(sys.argv[1], encoding="utf-8") as fp:
    for line_no, line in enumerate(fp, 1):  # 行号从 1 起，便于人类阅读
        for match in WORD_RE.finditer(line):
            word = match.group()
            column_no = match.start() + 1
            location = (line_no, column_no)
            index.setdefault(word, []).append(location)

for word in sorted(index, key=str.upper):
    print(word, index[word])
```

### 与「显式分支」等价（语义上）

```python
if word not in index:
    index[word] = []
index[word].append(location)
```

`setdefault` 把「缺则建、再取引用、再改」收进**一次方法调用**，省去显式 **`index[word] = …` 写回**，也比 §二 少一步。

### 相对 §二 的优势

- **无单独写回**：`get` 三行里第三步在键已存在时仍是多余赋值。  
- **代码一行**：适合词索引、分组累加等高频路径。

## 四、性能与「查找次数」（教学向）

下表是**阅读代码时的直觉步数**，不是 CPython 字节码级微基准；解释器可能对 `in`、下标等做优化。下面先用白话把 **`[]` 求值** 拆清楚，再对照表看「直觉步数」。

### 「实参先求值」：为什么 `get` / `setdefault` 里的 `[]` 每次都会跑？

Python 调用**任意**可调用对象（含 **`dict.get`**、**`dict.setdefault`**）时，固定顺序是：

1. **先把括号里写着的各个实参表达式逐个求值**，得到一组具体的对象（引用）。  
2. **再**把这些对象按规则绑定到形参，**进入**函数体。  
3. 函数体里才去查字典、决定返回值。

因此 **`lst = d.get(key, [])`** 的一次执行可以理解为：

1. 求值 **`key`** → 例如得到 `"a"`。  
2. 求值 **`[]`** → **立刻**在堆里 **new** 出一个空 `list`，得到它的引用（可想象成「某个地址上的新对象」）。  
3. 调用 **`get("a", 那个新列表引用)`**；若 **`"a"` 已在 `d` 里**，`get` 返回的是**字典里旧的那份**列表，**第二步造出来的那份根本没被用上**，很快成为待回收的短命对象。

**结论**：只要 **`[]` 写在实参位置上**，你**每调用一次** `get` / `setdefault`，**就会先执行一次 `[]`**，与 `key` 最后是否真的用得上这份默认无关——这就是「**对 `[]` 求值**」和「**键已存在时多造一个**」的**语言层原因**。

**一眼看差异的脚本**：`07_zen_word_index_demo.py` 第 **2)**、**2b)** 节（`fresh_list()` / `tracked_empty()` 计数）。

### 「`[]` 求值」白话拆解

#### 1. 先搞懂：`[]` 在 Python 里是什么？

`[]` 是**字面量**：一写出来，Python 就要**新建一个空列表对象**并给出引用。  
所以：**只要表达式里出现 `[]`，就会发生一次「造新列表」**——笔记里说的「对 `[]` 求值」，指的就是这件事。

#### 2. 方案 1、2 为什么「每次」都可能白造一个 `[]`？

**方案 1：`d.get(key, [])`**

```python
lst = d.get(key, [])
lst.append(1)
d[key] = lst
```

- `[]` 写在 **`get` 的第二个实参**上 → 属于上节「**实参先求值**」里的第二步，**每次调用必跑**。  
- 若 **`key` 已在字典里**：`get` 返回**已有**列表，**刚造出来的那个空列表没人要**，随即被丢弃。

**方案 2：`d.setdefault(key, []).append(1)`**

- 同理：`[]` 是 **`setdefault` 的第二个实参**，**每次调用都会先求值**。  
- **`key` 已存在**时：`setdefault` 返回旧列表，**新造的 `[]` 仍被丢弃**。

#### 3. 方案 3 `defaultdict(list)` 为什么通常没有「白造 `[]`」？

```python
from collections import defaultdict

d = defaultdict(list)
d[key].append(1)
```

- **`defaultdict(list)`** 这一行里，**`list` 是工厂函数对象本身**，**不会**在构造时替你调用 **`list()`** 造列表。  
- **只有 `key` 缺失、走到 `d[key]` 缺键分支时**，才会 **`list()` → 新 `[]`**。  
- **`key` 已存在**：直接取已有列表，**不会再调用工厂**——「造列表」从「**每次调用前的实参**」推迟到了「**键真的缺的那一次**」，自然就没有 `get`/`setdefault` 那种「热循环里反复造短命 `[]`」的问题。

#### 4. 比喻（寄信）

- **方案 1 / 2**：每封信都**先裁好一个备用空信封（`[]`）**再投递；收件人若在，备用信封**直接扔**；不在才用上。  
- **方案 3**：**确认收件人不在**才临时做一个新信封（**`list()`**）；在的话**沿用旧信封**，不重复做。

#### 5. 这算不算「大问题」？

- 小程序里往往**可忽略**。  
- **大字典 + 超长热循环**（百万级）时，多余分配会叠成可见的 **CPU / 分配器**压力；写法上也不如 `defaultdict` 干净。  
- **可运行对比**：`07_zen_word_index_demo.py` 第 **4)** 节（键已存在、`N` 次 `append` 的粗略计时；**绝对秒数因机器而异**，主要看 **`defaultdict` 相对更省**的趋势）。

### 直觉步数对照表（与上节互补）

| 方案 | 典型代码形态 | 直觉 |
| :--- | :--- | :--- |
| `if key not in d` + 赋值 + `append` | 显式分支 | 多次名字查找 / 方法调用 |
| `get` + `append` + 写回 | §二 | `get` 一次 + **写回**一次 |
| `setdefault(k, []).append(…)` | §三 | **一次** `setdefault`（内部完成缺键插入或返回值） |
| `defaultdict(…); d[k].append` | §五 | **`d[k]`** 一次；缺键时才跑工厂 |

**重要**：`setdefault(k, [])` 与 `get(k, [])` 一样，**每次调用都会对第二个实参 `[]` 先求值**；键已存在时仍会多分配一个随即丢弃的空列表。若极端在意分配次数，优先 **`defaultdict(list)`**（工厂仅在缺键时调用，见 **`07_zen_word_index_demo.py`** 第 **3)**、**4)** 节）。

### `dict.fromkeys(keys, [])`：第二个实参**只求值一次**，全键**共享**同一对象

`dict.fromkeys` 用来**批量建新字典**：所有给定 key 先存在，**值**都来自你传入的**同一个**默认对象。

```python
d = dict.fromkeys(["a", "b", "c"], [])
```

这里的 **`[]` 只在这行代码执行时求值一次**：先 **new** 出一个空列表，再把这个引用**赋给每一个 key**。因此 **`d["a"] is d["b"] is d["c"]`** 为真——**改一个键下的列表，全字典跟着变**：

```python
d = dict.fromkeys(["a", "b", "c"], [])
d["a"].append(1)
assert d == {"a": [1], "b": [1], "c": [1]}
```

**和 `get` / `setdefault` 的对比（把「求值几次」说透）**

| 写法 | `[]`（或默认表达式）何时求值 | 结果 |
| :--- | :--- | :--- |
| **`d.get(key, [])`** | **每调用一次 `get` 就先求值一次**第二个实参 | 键已存在时常造「短命」空列表 |
| **`d.setdefault(key, [])`** | **每调用一次 `setdefault` 就先求值一次** | 同上 |
| **`dict.fromkeys(keys, [])`** | **整句 `fromkeys` 执行时只求值一次**第二个实参 | **所有 key 共用**那**一个**列表（可变默认时极坑） |
| **`{k: [] for k in keys}`** | **推导式循环体每轮**都会对 `[]` 求值 | **每键一个**新列表，互不干扰 |

**结论**：`fromkeys` **不是**「给每个 key 各建一个空列表」的语法糖；要做「每键独立 `list`」，用**字典推导式** `{k: [] for k in keys}` 或 **`defaultdict(list)`**。

**脚本**：`07_zen_word_index_demo.py` 第 **5)** 节（`fromkeys` 只调用一次工厂 vs 多次 `get` 各求值一次）。

## 五、`defaultdict` 版本

```python
from collections import defaultdict

index: defaultdict[str, list[tuple[int, int]]] = defaultdict(list)
# ...
index[word].append(location)
```

- **缺键**时由 `defaultdict` 插入 `list()`；**键已存在**时不再调用工厂。  
- 后续章节（书中 **3.5** 等）会对 `collections` 做系统展开；此处作为 §3.4.3 的**自然延伸**。

## 六、避坑与选型

### 1. 不要共用同一个可变默认值

```python
# 错：所有键共享同一列表对象
shared: list[int] = []
d.setdefault(k, shared).append(v)

# 对：每次用新列表（`[]` 字面量）或 `defaultdict(list)`
d.setdefault(k, []).append(v)
```

**`dict.fromkeys(keys, [])`** 是「**一次求值、全键共享**」的另一种典型坑（与 **`setdefault(k, shared)`** 同类），和 **`get`/`setdefault` 里每次调用都先算 `[]`** 正好形成对照；**求值模型与对照表**见 **§四** `dict.fromkeys` 小节。

### 2. `get` vs `setdefault`

- **只读**、不想插入键：用 **`get`**。  
- **要改可变值**（追加、原地更新）：用 **`setdefault`** 或 **`defaultdict`**。

### 3. `setdefault` vs `defaultdict`

- 已有普通 **`dict`**、偶尔需要默认值：**`setdefault`**。  
- **大量**缺键、固定用 `list`/`int` 等工厂：**`defaultdict`** 往往更清晰，且可避免 §四所述的 **`[]` 反复求值**（在键已存在时）。

## 七、实现细节

- **`enumerate(fp, 1)`**：行号从 **1** 起；若与 §二 其它示例用 `enumerate(fp)`（从 0 起）混用，注意**同一项目内统一**。  
- **`sorted(index, key=str.upper)`**：传入 **`str.upper` 方法对象**，排序键为单词的大写形式，实现**大小写不敏感**的字典序；函数作一等值传递详见原书后续章节。  
- **快速失败**：`d[k]` 缺键抛 **`KeyError`**；`get`/`setdefault`/`defaultdict` 是不同默认策略。
