## 一、可哈希（能当 `dict` 键 / 进 `set`）

- **三条件（工程口径）**：**`hash` 与生命周期内语义一致**；能 **`==`**；**`a == b` ⇒ `hash(a) == hash(b)`**。  
- **好记**：可变容器、含可变元素的 `tuple` → **通常不可哈希**；**`str` / `bytes` / 数字 / `frozenset` / 元素均可哈希的 `tuple`** → 通常可。  
- **自定义类**：重写 **`__eq__`** 须同步 **`__hash__`** 或 **`__hash__ = None`**。  
- **详文**：[`05-Mapping抽象与可哈希.md`](05-Mapping抽象与可哈希.md) **§三**。

---

## 二、三种映射怎么选（照抄背）

1. **绝大多数**、只要**插入顺序** → **`dict`**（**3.7+**）。  
2. **`d[k]` 缺键**就要列表/集合/计数初值 → **`defaultdict`**。  
3. **`move_to_end`、FIFO `popitem`、LRU** → **`OrderedDict`**。

**详文**：[`06-dict-defaultdict与OrderedDict对照.md`](06-dict-defaultdict与OrderedDict对照.md) **§零.4**。

---

## 三、`defaultdict` 与 `get` / `setdefault`

- **只有 `d[k]` 缺键**才走 **`default_factory` / `__missing__`**。  
- **`d.get(k)`、`d.setdefault(...)` 都不触发工厂**（与「自动补默认」不是一条路）。  
- **`setdefault` vs `defaultdict`**：偶尔、显式 → `setdefault`；循环里大量分组/计数 → `defaultdict`。  
- **详文**：同 **`06`** **§零.3、§五、§七**。

---

## 四、`dict.fromkeys` 与可变默认

- **`dict.fromkeys(keys, [])`**：全键**共享同一列表**。  
- **改**：**`{k: [] for k in keys}`**。  
- **详文**：**`06`** **§零.5、§七**；脚本 **`06_mapping_types_three_way_demo.py`** 第 **3)** 节。

---

## 五、`OrderedDict` 何时才值得上

- **3.7+ `dict` 已有插入顺序**；仅遍历顺序**不需要** `OrderedDict`。  
- **要** `move_to_end`、`popitem(last=False)`、LRU 式顺序 API → **`OrderedDict`**。  
- **详文**：**`06`** **§零.5、§四**；变体并列见 **`10-OrderedDict-ChainMap-Counter.md`**。

---

## 六、`popitem` / `|`·`|=` 一句

- **`popitem`**：**`dict` / `defaultdict`**（**3.7+**）**LIFO**；**`OrderedDict`** 可 **FIFO**（`last=False`）；空映射 **`KeyError`**。  
- **`| / |=`**：**3.9+**（[PEP 584](https://peps.python.org/pep-0584/)）；**`|`** 新 dict，**`|=`** 原地；**右侧覆盖左侧**同名键；**≤3.8** 用 **`{**a, **b}`**。  
- **详文**：**`06`** **§四**（含 **`### 0` 四句**与 **`### 7` 一页表**）。

---

## 七、`ChainMap`（多层「叠在一起查」）

- **直觉**：多个映射**按序叠成一条查找链**，**不改**底层 dict 就得到「合并视图」；**写**通常落在**第一层**。  
- **详文**：[`10-OrderedDict-ChainMap-Counter.md`](10-OrderedDict-ChainMap-Counter.md)；合并语法对照见 [`03-映射拆包与字典合并.md`](03-映射拆包与字典合并.md)。
