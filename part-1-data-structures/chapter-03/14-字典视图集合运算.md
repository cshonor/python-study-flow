# 字典视图的集合运算：`dict_keys` / `dict_items` vs `frozenset`（§3.12）

`d.keys()` 和 `d.items()` 这两种视图对象，有一些“很像集合”的能力：你可以对它们用 `&`、`|`、`-`、`^` 这些集合运算符。

但它们又**不是** `set/frozenset`：有些具名方法没有、有些运算会受“动态视图”影响、`values()` 还完全不具备集合语义。

这篇就是把这些边界一次性讲清楚，避免你在代码里写出“看起来能跑，但逻辑不稳”的集合操作。

配套脚本：`14_dict_view_setops_demo.py`。

---

## 一、核心结论（务必区分「运算符」与「具名方法」）

- **`dict_keys`**、**`dict_items`**（在键、键值对可哈希等前提下）实现 **`collections.abc.Set`** 的语义，支持 **集合运算符**：**`&` `|` `-` `^`**，以及**关系比较**（如 **`<=`** 表示子集关系，以当前对象类型组合为准）。  
- **在 CPython 中**，这两类视图对象上**通常没有** **`frozenset`** 那样的 **`intersection()` / `union()` / `difference()` / `issubset()`** 等**具名方法**（对 **`dict_keys`** 做 **`dir()`** 可见公开 API 主要是 **`isdisjoint`**、**`mapping`** 等）。需要「多参数 / 可迭代对象并参与运算」时，往往用 **运算符**，或先 **`set(...)`** / **`frozenset(...)`**。  
- **`dict.values()`**：值**可重复**，**不**构成集合语义，**不支持**上述集合运算符（**`&`** 等会 **`TypeError`**）。  
- **动态视图 vs 快照**：视图**不拷贝**整张表，随原 **`dict`** 变；**`frozenset(d.keys())`** 是**静态快照**，且**可哈希**（在元素可哈希时）。

---

## 二、为何 `keys` / `items` 可以、`values` 不行？

- **键**在字典中**唯一** → **`dict_keys`** 的元素可视为集合成员。  
- **`dict_items`** 的元素是 **`(k, v)`**；在 **键唯一** 的前提下，每个 **`(k, v)`** 在视图中唯一；若参与 **`items()`** 的集合运算，需注意 **(键, 值) 元组须可哈希**（值不可哈希时无法参与）。  
- **值**可重复 → **`dict_values`** 不满足集合的「元素互异」直觉，故不提供集合运算。

---

## 三、与 `frozenset` 的对照（能力取向）

| 能力 | **`dict_keys` / `dict_items`** | **`frozenset`** |
| :--- | :--- | :--- |
| **集合运算符** | **`&` `|` `-` `^`** 等（与另一兼容对象） | 同左 |
| **多参数 `intersection(a,b,...)` 等** | **通常无**（见 §一） | **有**，且可传入**可迭代对象** |
| **动态同步** | **是**（随 dict） | **否**（快照） |
| **可哈希** | **否** | **是**（元素可哈希时） |
| **`reversed()`** | **是**（按 **3.7+ dict** 插入顺序逆序） | **`TypeError`**（无序类型） |

书表 3-5 的逐项勾叉以你使用的 **Python 版本**为准；**以 REPL / `14_dict_view_setops_demo.py` 验证最稳**。

---

## 四、常用写法示例

```python
d1 = {"a": 1, "b": 2, "c": 3}
d2 = {"b": 2, "c": 3, "d": 4}

d1.keys() & d2.keys()           # 共同键（结果类型依版本/运算对象，常为 set）
d1.items() & d2.items()         # 键值对完全一致才在交集中
d1.keys() - d2.keys()         # 仅在 d1 的键
d1.keys() | d2.keys()         # 全部键
```

**`d1.values() & d2.values()`**：不要写，会报错。

---

## 五、实战选型

- **只要对比键 / 键值对、且希望零拷贝、随 dict 更新**：用 **视图 + 运算符**。  
- **需要 `intersection(*many)`、或要把结果当 dict 键、或要嵌套进 `set`**： **`frozenset(d.keys())`** 等**物化**为不可变集合。  
- **需要反复做「多可迭代对象」参与同一套运算**：**`set`** / **`frozenset`** 的方法族通常更顺手。
