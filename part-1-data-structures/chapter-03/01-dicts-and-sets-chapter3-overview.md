## 一、开篇核心观点：字典是 Python 的基石

书中有句著名比喻：**Python 是包裹在语法糖里的字典**——指的不是「所有代码都写成 dict」，而是：**大量语言机制在实现层面依赖「映射」语义**。

- **语言机制**：类与实例的 `__dict__`、模块命名空间、函数调用时的**关键字参数绑定**，本质都是「名字 → 对象」的映射思路。
- **运行时环境**：内置名可通过 `builtins` 模块访问；教学里常提的 `__builtins__.__dict__` 用来直观看到**内置环境是一张映射表**（具体形态可能因实现细节略有差异，但**命名空间即 dict 类思想**不变）。
- **性能**：CPython 对 `dict` 做了深度优化；**平均**时间复杂度上，查找 / 插入 / 删除常视为 **O(1)**（哈希冲突与扩容时会有额外成本，第 3 章后文会讲哈希表）。

---

## 二、集合（`set` / `frozenset`）的定位

- **`set`**：可变、元素唯一，支持**并 / 交 / 差 / 对称差**等集合运算；适合**去重**、**成员关系**、**把 O(n²) 嵌套循环**换成集合操作。
- **`frozenset`**：不可变、**可哈希**（在元素都可哈希的前提下），可作为 **`dict` 的键**或 **`set` 的元素**；`set` 自身**不可哈希**，不能作为 `dict` 键。

## 四、关键概念提前（读本章前）

### 1. 哈希表（直觉）

- `dict` / `set` 底层依赖**哈希表**：用键的哈希值把元素散列到桶中，从而**摊还**意义下的快速查找。
- **约束**：映射的键、集合的元素必须是**可哈希**的（`__hash__` 与 `__eq__` 约定一致）。

### 2. 可哈希性（Hashable）

- **可哈希**：对象在生命周期内**哈希值稳定**，且可与同类对象比较相等；`dict` 键、`set` 元素要求可哈希。
- **常见**：`str`、`int`、`float`、`frozenset`、元素均可哈希的 `tuple`；**不可**：`list`、`dict`、可变 `set`。
- **自定义类**：若实现 `__eq__` 且未妥善定义 `__hash__`，实例可能**不可哈希**；不可变数据模型可用 `@dataclass(frozen=True)` 等保证可哈希。

### 3. `set` vs `frozenset`

| 特性 | `set` | `frozenset` |
| :--- | :--- | :--- |
| 可变性 | 可变（`add` / `discard` 等） | 不可变 |
| 可哈希 | 否 | 是（元素可哈希时） |
| 典型用途 | 去重、集合运算 | 作键、集合的嵌套元素 |

### 4. 「有序」与「无序」

- **Python 3.7+**：`dict` **保留插入顺序**（语言规范保证）；**不要**把它与「按键排序」混淆。
- 教学上仍会说哈希结构「不保证有序遍历」的**历史直觉**；写代码时以**官方语义**为准即可。

## 五、本章学习意义（为何值得精读）

1. **理解运行时**：命名空间、属性字典与 `dict` 同构，有助于后续读类、模块、元类。
2. **Pythonic**：用集合运算与 `defaultdict` / `Counter` 替代手写循环与 `if`。
3. **避坑**：迭代中修改 `dict`、可变对象作键、误判「相等」与「同一」等。
4. **性能**：在合适场景用 `set` 做成员检测 / 去重，往往比反复扫列表更省时间。

## 七、面试速记（极简背诵版）

- **`dict` vs `list`**：按键查找，平均 O(1) 级；列表按索引 O(1) 但按键名需线性搜。
- **`set` vs `list` 去重**：`set` 成员测试平均 O(1)；列表为 O(n)。
- **什么能当 `dict` 键**：可哈希、不可变语义（或 `frozenset` 等）；**list/dict 不能**。
- **`get` vs `[]`**：`d[k]` 缺键抛 `KeyError`；`d.get(k, default)` 不抛。
- **`setdefault` / `defaultdict`**：前者单次键上「有则取、无则设」；后者为**整类键**提供默认工厂。
- **`dict` 顺序**：3.7+ 插入顺序；**不是**按键排序。
- **`frozenset` 用途**：需要「集合当键」或嵌套进 `set` 时用。
