# 集合论与实现：`set` / `frozenset`（§3.10–§3.11）

集合（`set`）解决的核心问题只有一个：**成员是否存在**（membership test）和 **集合运算**（并/交/差/对称差）。

它之所以快，是因为底层使用哈希表；但也因此引出一些“必须记住的规则”：

- 元素必须是 **hashable**（可哈希）
- 集合不按“插入顺序”提供稳定语义（不要依赖遍历顺序）
- `set` 可变、`frozenset` 不可变，后者因此能当 `dict` 的 key 或另一个集合的元素

配套脚本：`13_set_theory_demo.py`。

---

## 零、三魂、易错坑、四句口诀（总览）

### 1）三个「核心灵魂」

1. **底层本质**  
   **`set` / `frozenset`** 都是**哈希表**思路的实现；**成员测试**在平均意义下可当作**摊还 O(1)**。硬性约束：**元素必须可哈希**（与 **`dict` 键**同一套规则）。

2. **可变与不可变分界**  
   - **`set`**：可变、**不可哈希** → **不能**作 **`dict` 键**或**另一个 `set` 的元素**。  
   - **`frozenset`**：不可变、在元素均可哈希时**可哈希** → **可嵌套**（如 **`set` 里放 `frozenset`**）、**可作映射键**。

3. **两大高频场景**  
   - **普通去重**：**`set(序列)`**，**不保序**（勿依赖遍历顺序）。  
   - **保序去重（3.7+）**：**`list(dict.fromkeys(序列))`**，保留**首次出现**次序。  
   - **集合运算**：**运算符**（**`& | - ^`**）两侧须为**集合类型**，**不能**把 **`list`** 直接拿来和 **`&`** 配；**具名方法**（如 **`intersection`**、**`union`**）往往**可直接吃列表等可迭代对象**（细节见 **§五**、脚本）。  
   - 注：**`set` 与 `frozenset`** 之间用 **`& |`** 等**通常合法**；「不能混」指的是**不要和任意非集合可迭代对象混用运算符**。

### 2）必记易错坑

1. **空集合**只能写 **`set()`**；**`{}` 是空字典**。  
2. **集合不提供稳定遍历顺序**——永远不要依赖打印或循环顺序。  
3. **`{1, 2} & [2, 3]`** → **`TypeError`**；要交集请 **`{1, 2}.intersection([2, 3])`** 或先 **`set(...)`**。  
4. **嵌套集合、把集合当 `dict` 键**——只有 **`frozenset`**（在元素可哈希前提下）这条路稳。

### 3）极简背诵口诀

1. **集合哈希打底，元素必可哈希**  
2. **`set` 可变不能键，冻集不可可嵌套**（「冻集」= **`frozenset`**）  
3. **去重 `set` 乱序，字典键保序**（指 **`dict.fromkeys`** 那条保序去重惯用法）  
4. **运算符号要集合型，方法兼容任意迭代**

脚本末尾 **`demo_one_liners()`** 为每条口诀各配**一行可运行示例**（与 **§一–§五** 正文对照）。

## 一、`set` 与 `frozenset`

| | **`set`** | **`frozenset`** |
| :--- | :--- | :--- |
| 可变性 | 可变（`add` / `discard` / 原地运算等） | 不可变 |
| 可哈希 | **否**（不能作 `dict` 键或 `set` 元素） | **是**（元素均可哈希时） |
| 典型用途 | 去重、动态增删、集合运算 | 常量集合、**嵌套**、作映射键 |

---

## 二、构造与常见陷阱

- **非空字面量**：`{1, 2, 3}` 是 **`set`**。  
- **空集合**：**必须**写 **`set()`**；**`{}` 是空 `dict`**。  
- **`frozenset`**：无专用字面量，用 **`frozenset(iterable)`**。  
- **集合推导式**：`{f(x) for x in it if cond}`（与字典推导式区分：无 `:` 即集合）。

---

## 二附、`frozenset` 速查清单（可复制）

（与 **§一** 表格互补：这里专收 **`frozenset`** 的构造与「能当键」场景。）

### 构造与字面量

- **无专用字面量**：没有类似 **`{1, 2, 3}`** 那种「一眼就是冻集」的写法；**`{1, 2, 3}` 在 Python 里是 `set`**。  
- **空 `{}`**：仍是**空 `dict`**；**空 `set`** 须 **`set()`**；**空 `frozenset`** 须 **`frozenset()`**。  
- **唯一构造入口**：**`frozenset(iterable)`**，参数为任意**可迭代对象**（**`list` / `tuple` / `str` / `set`** 等）。

```python
fs1 = frozenset([1, 2, 3])
fs2 = frozenset("abc")  # 三个单字符元素
fs3 = frozenset()       # 空 frozenset
```

### 不可变、可哈希、嵌套

- **不可变**：创建后**不能**原地 **add/remove**；元素须**可哈希**。  
- **可哈希**（在元素均可哈希时）：**可作 `dict` 键**、**可放进 `set`**（普通 **`set`** 不能当键或当元素）。

```python
d = {frozenset({1, 2}): "ok"}
s = {frozenset({3, 4})}
```

### 只读集合运算

- **`| & - ^`**、**`<=` `<` 等**与 **`set`** 语义类似；**`frozenset` 没有** **`|=` / `&=`** 等原地更新（那是 **`set`** 专属）。  
- 二元运算**产生新的 `frozenset`**（或 **`set`**，以运算符两侧类型与 CPython 规则为准）。

### 与 `set` 对比（速记）

| 特性 | **`set`** | **`frozenset`** |
| :--- | :--- | :--- |
| 非空字面量 | **`{1, 2, 3}`** | ❌ 无 |
| 空实例 | **`set()`**（**勿用 `{}`**） | **`frozenset()`** |
| 可变性 | 可变（**`add` / `discard` / `|=`** 等） | 不可变 |
| 可哈希 | ❌ | ✅（元素均可哈希时） |

---

## 三、去重与保序去重

- **`set(seq)`**：去重，**不**保留原序列顺序（集合**无序**；遍历顺序受实现与哈希随机化等影响，**勿依赖**）。  
- **保序去重（3.7+）**：**`list(dict.fromkeys(items))`** 利用 `dict` 插入顺序保留**首次**出现次序。

---

## 四、成员测试与「交集计数」

- **`x in s`**：在平均情况下可视为**摊还 O(1)**（哈希表）；远优于扫列表的 **O(n)**。  
- 求「两个容器里共同元素个数」时，**`len(needles & haystack)`** 在二者已是 **`set`** 时很直接；否则常用 **`len(set(needles) & set(haystack))`** 或 **`len(a.intersection(b))`**（**方法**接受**任意可迭代对象**，不必先 `set`）。

---

## 五、集合运算：运算符 vs 方法

- **运算符**（如 **`&` `|` `-` `^`**）：两侧通常需为 **`set`** / **`frozenset`**（等兼容类型）；**`{1,2} & [2,3]`** 会 **`TypeError`**。  
- **方法**（如 **`intersection`**、**`union`**）：可把**列表等可迭代对象**当参数，内部再参与运算，更灵活。  
- **关系**：**`<=` `<` `>=` `>`**、**`issubset`** / **`issuperset`**、**`isdisjoint`**。  
- **原地更新**（仅 **`set`**）：**`|=`**、**`&=`**、**`-=`**、**`^=`** 与 **`update`**、**`intersection_update`** 等。

---

## 六、实现直觉（CPython）

- **`set` / `frozenset`** 基于**哈希表**；元素须**可哈希**且互比一致（与 **`dict` 键**同一套约束）。  
- **无序**：**不要**依赖打印或遍历顺序；需要顺序时 **`sorted(s)`** 或显式规则。  
- **扩容 / 哈希种子**：表重建时，遍历顺序可能变化；**仍勿依赖顺序**。

---

## 七、`collections.abc` 中的位置（概念）

- **`collections.abc.Set`**：只读集合协议；**`MutableSet`** 扩展可变语义。  
- 内置 **`frozenset`** 可视为**不可变集合**的典型实现；**`set`** 为**可变**实现。具体 **`isinstance`** 以当前版本为准。

---

## 八、与 `dict_keys` 的对比（易混点）

- **`d.keys()`** 返回的 **`dict_keys`** 在键可哈希时可参与 **`&` `|`** 等，语义类似集合，但类型仍是**视图**，且与 **`set`** 互操作规则以文档为准（见 **`12-字典视图.md`**）。  
- 需要**独立快照**或保证为 **`set`** 时，用 **`set(d.keys())`**。  
- **§3.12** 级对照（运算符 vs **`frozenset`** 具名方法、**`items()`**、**`values()`**）见 **`14-字典视图集合运算.md`**。
