# 映射拆包与合并：`{**d1, **d2}`、`d1 | d2`、`d1.update(d2)` 到底怎么选？

你在 Python 里合并字典，最常见就是这几种写法：

- `merged = {**d1, **d2}`（新字典，后者覆盖前者）
- `merged = d1 | d2`（Python 3.9+，新字典，右覆盖左）
- `d1 |= d2` / `d1.update(d2)`（原地修改左边）
- `ChainMap(d1, d2, ...)`（不是“合并”，而是“按顺序查找的视图”）

很多新手会被“看起来都能合并”搞晕：到底差在哪？什么时候会踩坑？

这篇笔记只讲 4 件事，讲透你就能选对写法：

1. **重复键怎么处理（谁覆盖谁）**  
2. **会不会修改原字典**（新对象 vs 原地）  
3. **是不是浅合并**（嵌套 dict 的坑）  
4. **`ChainMap` 不是合并，它是“查找链”**

---

## 零、选型口诀（背这四条就够）

1. **要新字典、不动原来的**：`{**d1, **d2}`（**3.5+**）或 **`d1 | d2`（3.9+）**。  
2. **要原地改、省一份拷贝**：`d1.update(d2)`（全版本）或 **`d1 |= d2`（3.9+）**；`update` 在 Python 3 里**返回 `None`**，别误当返回值用。  
3. **多层配置只想按优先级查、不物化合并**：`ChainMap`（**不是拷贝合并**，是查找视图）。  
4. **普通合并全是浅合并**：嵌套 `dict` **不会递归融合**，同名顶层键上**整段子 dict 被替换**。

---

## 一、与字典推导式衔接（3.2.1 回顾）

- 可把「键值对」可迭代对象直接交给 **`dict(...)`** 构造。  
- 复杂变换用**字典推导式**更顺手（对调、过滤、`.upper()` 等），详见 `02-字典推导式与集合推导式.md`。

**注意**：若 `country_dial` 已是「国名 → 区号」的 `dict`，要按**国名排序**再推导，应对 **`items()`** 排序，而不是 `for code, country in sorted(country_dial)`（`sorted(dict)` 只给出**键**，不能拆成两个变量）：

```python
# 需先有「国名 → 区号」的 country_dial（见 02）
example_by_country = {
    country.upper(): code
    for country, code in sorted(country_dial.items())
    if code < 70
}
```

（与 `02` 中「以区号为键、国名为大写」的写法只是**键值角色不同**，按需求选用。）

## 二、3.2.2 映射拆包

Python 3.5 起，`**` 可用于：

### 1. 函数调用：多个 `**` 与关键字实参混用

- 可将**多个**映射拆成关键字参数传入；**靠后的覆盖靠前**的同名键。  
- 键须是合法的关键字参数名（通常为字符串键；细节见语言参考）。

```python
def dump(**kwargs):
    return kwargs

dump(**{"x": 1}, y=2, **{"z": 3})  # {'x': 1, 'y': 2, 'z': 3}
```

这段你可以按“从左到右把键值对倒进 kwargs 里”来理解：后面倒进来的同名键会覆盖前面的值。

### 2. 字典字面量：`{**a, **b}`

- 在 `{}` 里可写多个 `**mapping`，**后面的键覆盖前面的**。  
- 可与普通 `键: 值` 项混写，顺序决定优先级。

```python
{"a": 0, **{"x": 1}, "y": 2, **{"z": 3, "x": 4}}
# {'a': 0, 'x': 4, 'y': 2, 'z': 3}   # 右侧的 x=4 覆盖左侧的 x=1
```

把它读成“拼装一个新 dict”更好理解：

- `{...}` 本身就会创建新对象  
- `**mapping` 的效果等价于把 mapping 的每个键值对“拷贝进来”（但注意是浅层拷贝：只拷贝引用）

## 三、3.2.3 使用并集运算符合并（Python 3.9+）

### 1. 二元 `|`：新字典

- 返回**新** `dict`，**不修改**左右操作数（除非左操作数是子类且重载了行为，一般按内置语义理解即可）。  
- 键冲突时，**以右操作数为准**。

```python
d1 = {"a": 1, "b": 3}
d2 = {"a": 2, "b": 4, "c": 6}
d1 | d2   # {'a': 2, 'b': 4, 'c': 6}
```

如果你会读数学里的“并集”，那就把它当成“两个映射的并集”：键冲突时右边获胜。

### 2. `|=`：原地更新

- **修改**左侧字典，并入右侧映射；同样**右覆盖左**。  
- 右侧可为任意**映射**对象（见 PEP 584）。

```python
d1 = {"a": 1, "b": 3}
d2 = {"a": 2, "b": 4, "c": 6}
d1 |= d2
# d1 现为 {'a': 2, 'b': 4, 'c': 6}
```

这和 `list.sort()` 一样，属于“改自己”的操作：你应该把结果看作在 `d1` 里，而不是期待它返回新对象。

### 3. 版本与兼容

- **`|` / `|=`** 需 **Python 3.9+**。  
- 3.8 及更早：用 **`{**d1, **d2}`** 做一级浅合并，或 `d1 = {**d1, **d2}` / `copy` + `update`。

## 四、对照表（极简）

| 写法 | 最低版本 | 产生新字典？ | 原地改左侧？ | 键冲突 |
| :--- | :--- | :--- | :--- | :--- |
| `{**d1, **d2}` | 3.5+ | ✅ | ❌ | **后**写的映射覆盖**前**面的同名键 |
| `d1 \| d2` | 3.9+ | ✅ | ❌ | **右**覆盖**左** |
| `d1.update(d2)` | 2+ | ❌（返回 `None`） | ✅ 改 `d1` | `d2` 覆盖 `d1` |
| `d1 \|= d2` | 3.9+ | ❌ | ✅ 改 `d1` | **右**覆盖**左** |

对**两个** `dict` 来说，「后覆盖前」与「右覆盖左」只是写法不同，**冲突结果一致**；差别主要在**版本**与**是否原地**。

## 五、实用拓展

### 1. 兼容 3.5–3.8

```python
merged = {**d1, **d2}
```

### 2. 嵌套字典

上述写法都是**浅合并**：同名键上的**内层 dict**会被**整段替换**，不会递归合并子键。

这点非常关键。新手最容易误以为“合并会把子字典也合并”。实际上不会：

```python
d1 = {"cfg": {"timeout": 1, "retry": 2}}
d2 = {"cfg": {"timeout": 10}}
res = {**d1, **d2}
assert res == {"cfg": {"timeout": 10}}  # 成立 → 子 dict 整段替换，内层 retry 不会与 d2 的 timeout 做字段级合并
```

如果你真的要“递归合并”，那已经是另一类问题了（需要你自己写规则：子字典怎么合？列表怎么合？冲突怎么处理？）。文末 **§九** 给了一个最小可用的递归合并示例，可直接抄走再按项目规则改。

### 2.1 场景化怎么选（照抄）

1. 要兼容 **3.8 及以下**、且**不**改原字典 → **`{**d1, **d2}`**。  
2. 环境 **3.9+**、只读合并、写法要短 → **`d1 | d2`**。  
3. 数据大、**允许**改左边、不想多占一份顶层 dict → **`update` / `|=`**。  
4. 用户配置 > 项目 > 默认，**按优先级查找**、不想拷贝 → **`ChainMap`**。  
5. 嵌套 **深度融合** → **手写递归**（或第三方如 `deepmerge` 库），原生合并语法做不到。

### 3. 其它模式

- **`collections.ChainMap`**：多映射**按顺序查找**的视图，**不是**把多个 dict **拷贝合并**成一个新 dict。  
- **`dict.update`**：原地更新；在 Python 3 中**返回 `None`**。

你可以把 `ChainMap` 当成“配置覆盖”的最常用工具：

- 第一层：用户配置（最优先）  
- 第二层：环境配置  
- 第三层：默认配置（兜底）

查找时从前往后找；**写入**时默认落在**链里最前**的那个映射上，不会帮你把各层物化成一层（这在 `10_dict_variants_demo.py` / `ChainMap` 例子里也演示过）。

## 八、课后题参考答案（可复制对照）

### 题 1：`{"x": 1}` 与 `{"x": 2}`

```python
d1 = {"x": 1}
d2 = {"x": 2}

m_unpack = {**d1, **d2}
assert m_unpack == {"x": 2} and d1 == {"x": 1}  # 成立 → 新 dict，d1 不变

m_pipe = d1 | d2
assert m_pipe == {"x": 2} and d1 == {"x": 1}  # 成立 → 新 dict，d1 不变

d1_copy = {"x": 1}
assert d1_copy.update(d2) is None  # 成立 → update 返回 None
assert d1_copy == {"x": 2}  # 成立 → 原地改了 d1_copy
```

### 题 2：浅合并（子 dict 整段覆盖）

见 **§五 · 2** 的 `cfg` 示例；结论：`retry` 会丢，因为 `d2["cfg"]` **整块**替换了合并结果里的 `cfg`。

### 题 3：多层查找

**用户配置 > 项目配置 > 默认配置**：用 **`ChainMap(user, project, defaults)`**，只建查找链、不拷贝合并；读键时从左到右命中即停。

## 九、可选：递归合并嵌套 `dict`（最小工具）

原生 `{**a,**b}` / `|` / `update` **都不会**递归合并子字典。下面是一个**最小**递归合并（仅处理「值仍是 dict」的情况；列表等需另定规则）：

```python
def deep_merge(a: dict, b: dict) -> dict:
    out = dict(a)
    for k, v in b.items():
        if k in out and isinstance(out[k], dict) and isinstance(v, dict):
            out[k] = deep_merge(out[k], v)
        else:
            out[k] = v
    return out


base = {"cfg": {"timeout": 1, "retry": 2}}
over = {"cfg": {"timeout": 10}}
assert deep_merge(base, over) == {"cfg": {"timeout": 10, "retry": 2}}  # 成立 → 子键融合而非整块替换
```
